### 7-Stage Lifecycle Model Established

| Stage | Code | Description | Is Endpoint? |
|-------|------|-------------|---------------|
| 等待态 | W | Default start/stop. No new signals, no action, maintain current frozen boundaries. | ✅ Yes (default endpoint) |
| 轻响应态 | L | Caliber/term/query-only inputs. Does not enter assessment. | ❌ No (intermediate) |
| 评估态 | A | Boundary unclear / trigger pending / anomaly pending. Not yet prepared. | ❌ No (intermediate) |
| 准备态 | P | Near next-step conditions. Not yet entering RUN-next. | ❌ No (intermediate) |
| 升级态 | E | Needs manual decision or confirmation. Not auto-advance. | ❌ No (intermediate) |
| 回复吸收态 | R | Received reply, waiting for min write-back. Not auto-transition. | ❌ No (intermediate) |
| 回收停点 | S | This cycle ends. Back to W / stop at A / stop at P / stop (end). | ✅ Yes (this cycle endpoint) |

### Key Prohibited Auto-Transitions (Critical)
- Any state → RUN-next: MUST have manual confirmation, V7 baseline frozen
- 等待 → 评估: MUST have new signal trigger
- 评估 → 准备: MUST meet preparation conditions
- 升级 → 回复: MUST receive actual reply
- 回复 → 继续: MUST receive confirmation

### Common Misconceptions Clarified (All Addressed)
- 等待态 ≠ 没有事情发生 (waiting is default stop/start)
- 进入评估 ≠ 必然升级 (clear boundaries don't need escalation)
- 进入准备 ≠ 已经运行 (preparation ≠ RUN-next execution)
- 补材料 ≠ 否决 (supplementary ≠ rejection)
- 暂不推进 ≠ 永久不做 (pause ≠ permanent)
- 不同意本次 ≠ 永久否决 (this time ≠ always)
- 收到回复 ≠ 自动进入下一状态 (reply → classify → stop, not auto)
- 升级后 ≠ RUN-next已触发 (escalation = request for judgment, not RUN-next trigger)
- 阅读路径 = 生命周期路径 (reading path = understand system, lifecycle path = how system actually moves — NOT the same)
