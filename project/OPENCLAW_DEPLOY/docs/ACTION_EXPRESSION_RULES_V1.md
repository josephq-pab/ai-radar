## 一、核心概念更新

**V1 新增：动作类型三分法**

在 V0"主动作/观察项/待核实"三分类基础上，V1 新增"动作类型"维度：

| 类型 | 定义 | 是否占主动作名额 | 本月是否产生业务动作 |
|------|------|----------------|-------------------|
| **立即执行** | 本月必须有人推动，完成后直接影响业务结果 | ✅ 占 | ✅ 产生 |
| **研究准备** | 本月需要搞清楚一件事，结果给下月提供判断依据 | ❌ 不占 | ❌ 不产生 |
| **建立机制** | 本月启动但本月完不成，效果在下个月之后体现 | ❌ 不占主动作名额 | ⚠️ 本月有动作但无结果 |

## 二、"可部署"的定义更新

**V1 更严格的可部署定义**：

一条建议，只有同时满足以下四个条件，才算"可部署"：
1. **动作类型明确**：读者知道这是"本月立即要做"，还是"先研究搞清楚"，还是"建个机制"
2. **动作明确**：读者知道要做什么（不是"加强关注"，而是"本周完成XX客户的XX动作"）
3. **责任到位**：读者知道由谁来做（不是"建议相关部门"，而是"分行客户管理部门"）
4. **可验证**：读者知道怎么做算成功了（不是"持续关注"，而是"当XX超过X%时预警"）

---

## 三、"研究准备"与"立即执行"的区分标准

**最核心的区分问题**：

> "这条动作做完，会在本月产生一个可以直接推动的业务动作吗？"

- **能** → 立即执行
- **不能（只能产出结论）** → 研究准备

**典型错误对照表（V0 样稿中的问题）**：

| V0 写法（错误）| V1 写法（正确）| 错误原因 |
|--------------|--------------|---------|
| "获取华夏银行存款利率截面数据" | "研究准备：获取华夏银行存款利率" | 获取数据本身不是业务动作，是研究动作 |
| "分析华夏增速是否可持续" | "研究准备：形成华夏增速是否趋势性判断（预计产出：下月结论）" | 分析是研究不是执行 |
| "建立招行贷存款联动监测" | "立即执行：做一次招行对公贷款利率监测（监测完才算执行了一次）" | 建立机制≠立即执行 |
| "摸排我行客户异动" | "立即执行：联系≥5000万存款客户，确认到期续做意向" | 摸排是研究动作，联系客户才是执行动作 |

**研究准备的正确格式**：
```
▸ 研究准备：[研究什么]
    研究目的：[搞清楚这个问题是为了支持什么判断]
    建议责任人：[战略研究/企金部门；不是业务执行部门]
    预计产出：下月初可形成结论（是/否 + 理由）
    注意：此动作本月不产生直接业务动作
```

## 四、"建立机制"与"立即执行"的区分标准

**核心问题**：
> "这条动作本月做完，能看到一个具体的结果吗？"

- **能** → 立即执行
- **不能（本月只是启动）** → 建立机制（不占主动作名额，但可以写进主动作）

**"建立机制"类动作的处理方式**：
- 如果本月要"建一个机制"，要写清楚"本月建到什么程度"
- 示例："建立招行贷存款联动监测" → 改为"本月启动一次招行贷存款联动监测，把本月数据记录清楚，作为下月对比基准"
- **不占主动作名额，但可以作为主动作2的有效内容**

## 五、观察项的新要素（V1 必填四项）

V0 的观察项只有三项（观察什么/频率/预警信号），V1 新增"执行人/部门"：

| 要素 | 说明 | 是否必填 |
|------|------|---------|
| 观察什么 | 具体盯哪个指标/行为/事件 | ✅ 必填 |
| 执行人/部门 | 谁来执行这个观察 | ✅ 必填（V1新增）|
| 观察频率 | 每周/每两周/每月 | ✅ 必填 |
| 预警信号 | 出现什么情况需要上报/升级 | ✅ 必填 |

**观察项的格式示例**：
```
▷ 观察项A：外币存款流入是否持续
    观察什么：平安银行外币存款月增量（是否持续>100亿）
    执行人/部门：分行客户管理部门（每月检查一次）
    观察频率：每月
    预警信号：连续2个月>100亿且我行外币存款流失>10%，升级为主动作
```

## 六、泛建议升级标准（V1 更新）

| 泛建议（不可部署）| 升级为"立即执行"的写法 | 升级为"研究准备"的写法 |
|----------------|----------------------|----------------------|
| "摸排客户异动" | "联系5000万以上存款客户，确认到期续做意向（截至本周末）" | "获取我行大额存款客户分层名单（给下月摸排提供依据）" |
| "关注招行动向" | "记录本月招行对公贷款利率变化，填入监测表" | "分析近3个月招行存贷款联动策略（为下月判断提供依据）" |
| "加强客户维护" | "本周完成重点客户的大额存单到期提醒（客户覆盖率100%）" | — |

---

## 七、"研究准备"的验证指标写法

研究准备的验证指标，不能写成业务结果，要写成"研究产出的质量指标"：

| 研究准备动作 | 错误写法（业务结果）| 正确写法（研究质量）|
|------------|-----------------|------------------|
| 获取华夏利率数据 | "华夏银行存款增长加速" | "拿到华夏3年/5年定存利率数据，填入分析表" |
| 分析招行策略 | "招行挖走了我行客户" | "招行近3个月对公贷款利率策略变化形成一页摘要" |

---

## 八、V1 检查表（5问）

写完一条建议后，用以下5问自检：

1. **动作类型是否明确？** → 不能把研究准备写成立即执行
2. **动作是否具体？** → 不能用"加强/关注/做好"开头
3. **责任是否到部门/团队？** → 不能只写"建议相关部门"
4. **是否有可验证指标？** → 不能写"持续跟踪"（研究准备则写"研究产出"指标）
5. **是否区分了"立即执行/研究准备/建立机制"？** → 三类动作不能混在一个格式里

## 九、版本变更说明（V0 → V1）

| 变更点 | V0 | V1 | 变更原因 |
|--------|----|----|---------|
| 动作分类 | 主动作/观察项/待核实 | 立即执行/研究准备/建立机制/观察项/待核实 | V0-05/V0-09问题未解决，核心改变 |
| 可部署定义 | 三条件 | 四条件（增加"动作类型明确"）| 解决"研究动作混入主动作" |
| 研究准备边界 | 未定义 | 明确"做完不能产生业务动作" | V0最核心问题 |
| 建立机制边界 | 未定义 | 区分"建机制≠立即执行" | 解决"主动作3不清晰" |
| 观察项要素 | 三项（缺执行人）| 四项（增加执行人/部门）| 解决"每月看了但没人负责" |
| 研究准备的验证指标 | 未定义 | 写"研究质量指标"而非"业务结果" | 防止研究准备变成业务动作 |
