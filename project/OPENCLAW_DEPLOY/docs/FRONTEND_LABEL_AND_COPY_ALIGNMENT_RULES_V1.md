## 二、停止条件语法

### 2.1 主链路/决策类页面

**标准**：`[已完成动作] → [结论] → 即可停止`

**示例**：
- single-chain-ops：`等待态正常 → 值班结果已给出 → 即可停止`
- ops-decision：`判断已完成 → [执行/等待/升级] 结论已给出 → 即可停止`
- ops-evidence：`理解冻结依据后 → [依据成立/不成立/待补充] 结论已给出 → 即可停止`
- ops-brief：`快报结论"等待态正常" → 读完即可停止`
- config-status：`核对的配置项 → 确认该项当前值是否正常 → 即可停止`

### 2.2 支援页（查到即回）

**标准**：`查到[对象] → [确认动作] → 即可返回`

**示例**：
- ops-glossary：`查到定义 → 理解口径 → 即可返回`
- ops-registry：`查到真源 → 确认权威版本 → 即可返回`
- ops-routes：`选路完成 → 确认第一条要看的页 → 即可返回`

**禁止用法**：
- ❌ "即可离开"（支援页应统一用"即可返回"）
- ❌ "回到引用的源页面继续"（非标准停止语法）

## 五、支援页支援化表达标准

支援页的核心约束：**查到即回，不得停留诱导**

| 支援页 | 标准支援化表达 |
|--------|-------------|
| ops-glossary | "查到定义即可返回" / "查到定义 → 理解口径 → 即可返回" |
| ops-registry | "查到真源即可返回" / "查到真源 → 确认权威版本 → 即可返回" |
| ops-routes | "选路完成即可返回" / "选路完成 → 确认第一条要看的页 → 即可返回" |

**禁止出现**：
- ❌ "不需要把四类角色全看完"（暗示有更多内容可看）
- ❌ "通读全页"（暗示需要完整阅读）
- ❌ "回到引用的源页面继续"（非标准支援页停止语）

## 六、同义不同写法拉齐

| 含义 | 标准写法 | 已废止写法 |
|------|---------|---------|
| 支援页停止 | 即可返回 | 即可离开、回到原工作流继续 |
| 主链路停止 | 即可停止 | 即可离开、读完即可返回 |
| 证据充分 | 依据成立/不成立/待补充 | 证据充分、证据不足（待补充）|
| 决策结论 | 执行/等待/升级 | 触发/等待/升级（decision 页为执行）|
