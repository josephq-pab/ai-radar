## 一、核心概念：动作闭环三角

每一个动作建议，都必须回答三个问题才算完整闭环：

1. **谁来做？** → 责任人（部门 + 角色）
2. **做完交给谁？** → 接收方（仅研究准备动作需要）
3. **什么条件下要接着做？** → 转动作条件（仅研究准备动作需要）

**立即执行动作**只需要回答第1问，闭环终点是"完成/未达到验证指标"。
**研究准备动作**需要回答全部三问，闭环终点是"接收方读完结论并决定是否触发下一步"。

---

## 二、动作类型及闭环要求（V3 完整版）

| 类型 | 定义 | 闭环终点 | 是否需要接收方 | 是否需要转动作条件 |
|------|------|---------|--------------|-----------------|
| **立即执行** | 本月必须有人推动，完成后直接影响业务结果 | 完成后确认验证指标（达到/未达到）| ❌ 不需要 | ❌ 不需要 |
| **研究准备** | 本月需要搞清楚一件事，结果给下月提供判断依据 | 研究结论被接收方阅读并决定是否触发动作 | ✅ 需要 | ✅ 需要 |
| **建立机制** | 本月启动但本月完不成，效果在下个月之后体现 | 机制进入常态运行，下月由谁运行待定义 | ⚠️ 视情况 | ⚠️ 视情况 |

## 三、研究准备动作的标准格式（V3 完整版）

V3 研究准备格式相比 V2，新增**组织差异降调写法**（Section 九）：

```
▸ 研究准备：[研究主题]
    研究目的：[搞清楚这个问题是为了支持什么判断]
    研究责任人：[企金部（主办）+ 战略研究部（支持）] ← V3新增联合主办格式
    角色建议：企金部行业研究员（主办）+ 战略研究部宏观研究员（支持）
    研究内容：
      1. [具体要拿到的证据/数据/结论 1]
      2. [具体要拿到的证据/数据/结论 2]
    产出形式：一页书面判断报告（3~5个要点）
    接收方：[部门] / [角色]
    转动作条件：
      若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    预计完成时点：XXXX年XX月XX日
    注意：此动作本月不产生直接业务动作
```

## 四、责任归口要求（V3 完整版）

### 4.1 粒度要求（V2已定义，V3不变）

**V3 责任归口必须到"部门 + 角色"两级**，仅写部门级视为不完整。

### 4.2 联合行动写法（V2已定义，V3新增主办+支持格式）

```
研究责任人：企金部（主办）+ 战略研究部（支持）
角色建议：企金部行业研究员（主办）+ 战略研究部宏观研究员（支持）
```

### 4.3 组织差异降调写法（V3新增）

当责任归属在各行之间存在差异时，使用降调写法：

```
建议责任人：企金部 / 对公业务部（注：若无专门研究支持，可由企金部产品经理兼任）
```

详见 Section 九。

## 五、接收方的写法规则（V2已定义，V3不变）

**接收方定义**：能决定"这个结论要不要触发下一步动作"的人，一般是部门负责人或分行长级别。

**典型写法**：分行公司部负责人（或分行长）

---

## 六、转动作条件的写法规则（V2已定义，V3不变）

每个转动作条件必须包含三要素：触发事实 + 判断门槛 + 对应动作。

**标准格式**：
```
若[触发事实]达到[判断门槛]，则下月启动[对应动作]
```

## 七、立即执行动作的格式（V2已定义，V3新增双门槛表达）

V3 的立即执行动作，相比 V2 增加了**双门槛试行表达**（适用于客户排查类动作）：

```
▶ 主动作1（本 月 必 做）：[动作名称]
    目的：[做这件事是为了防范什么/抓住什么机会]
    建议责任人：[部门] / [角色]
    验证指标：
        重点排查：联系5000万以上存款到期客户，确认续做意向
        快速扫描：排查1000万以上存款客户有无异常信号（重点是有没有异常，不必全部联系）
        预警阈值：[超出多少需要立即上报]
    完成时点：XXXX年XX月XX日
```

详见 Section 八。

## 八、双门槛试行表达规则（V3 新增）

### 8.1 什么是双门槛

**单一门槛的问题（来自Q2反馈）**：
- 当前只有"5000万以上"一个门槛
- Q2反馈：5000万偏高，应降低到1000万
- 直接替换为1000万：执行量扩大5倍，影响未知

**双门槛的折中逻辑**：
- 5000万以上：重点排查，必须联系，防挖转
- 1000万以上：快速扫描，不需要全部联系，重点是摸底有没有异常信号
- 两层覆盖不同目的，不是简单的"降门槛"

### 8.2 适用场景

| 动作类型 | 是否适用双门槛 |
|---------|-------------|
| 客户挖转风险排查 | ✅ 适用（排查有轻重，天然分两层）|
| 客户到期续做提醒 | ✅ 适用（淡旺季触达量差异大）|
| 招行联动客户扫描 | ❌ 不适用（这是建立基准，不需要排查）|
| 建立机制类动作 | ❌ 不适用（机制类不是本月完成）|

### 8.3 双门槛格式标准

```
验证指标：
    重点排查：联系[门槛A]以上存款到期客户，确认续做意向；已有转投迹象的，立即上报
    快速扫描：排查[门槛B]以上存款客户有无异常信号（不必全部联系，重点是有无异常）
    预警阈值：[超出多少需要立即上报]
```

### 8.4 试行加注要求

双门槛是试行表达，必须加注：

```
【注】：双门槛为试行表达（V3）。执行层实际触达量取决于各行存量客户数量。
试行期间若发现[门槛B]扫描范围过宽或过窄，下月反馈中注明，模板据此调整。
```

### 8.5 双门槛试行反馈机制

**触发调整的条件**：
- 执行层反馈"1000万扫描量太大，每月做不完" → 下月收紧为"快速扫描：只查有异常信号的客户"
- 执行层反馈"5000万重点排查太粗，漏掉了2000~5000万区间的风险" → 下月考虑扩大到2000万

**调整路径**：执行层反馈 → 记录到下一轮REAL_FEEDBACK_LOG → 模板参数调整 → V4体现

## 九、研究准备归口调整规则（V3 新增）

### 9.1 触发来源

Q1反馈（FB-P2-20260410-001）：
- 原模板：战略研究部 / 行业研究员
- 反馈：企金部（对公业务部）更合适

### 9.2 V3调整：不替换，联合主办

**不直接替换的原因**：
- Q1反馈来自单家银行企金部，不代表所有银行
- 企金部和战略研究部在研究准备上的能力边界不同银行差异大
- "企金部主办"是否具有跨机构普适性，需要下一轮反馈验证

**V3写法：联合主办**
```
建议责任人：企金部（主办）+ 战略研究部（支持）
角色建议：企金部行业研究员（主办）+ 战略研究部宏观研究员（支持）
```

**这样写的含义**：
- 主办=一线业务视角主导（企金部最先感知竞品动态）
- 支持=宏观分析能力兜底（战略研究部提供框架）
- 这是试行写法，不代表所有银行都用这个组合

### 9.3 组织差异降调写法

若某银行没有战略研究部，或企金部明确表示不承接研究准备：

```
建议责任人：企金部 / 对公业务部（注：若无专门研究支持，可由企金部产品经理兼任）
```

### 9.4 归口调整反馈机制

**触发进一步调整的条件**：
- 执行层反馈"企金部不承接研究准备，仍应由战略研究部主办" → V4回退为战略研究部主办
- 执行层反馈"企金部主办是对的，但战略研究部支持没必要" → V4简化为企金部单独主办
- 执行层反馈"企金部+战略研究部联合主办，沟通成本高" → V4改为明确主办方

## 十、什么叫"试行表达"（V3 新增定义）

### 10.1 试行表达的三原则

**原则一：不是最终答案**
试行表达是基于已有反馈的当前最优解，不等于最终正确答案。

**原则二：必须标注"试行"**
所有试行表达必须在文档中明确标注"试行"，包括：试行门槛、试行归口、试行粒度。

**原则三：有反馈触发调整机制**
试行表达必须在下一轮收到反馈时主动确认是否调整，不是"写死了就不动"。

### 10.2 试行表达的判断标准

| 判断维度 | 非试行（固化）| 试行（不固化）|
|---------|------------|------------|
| 反馈数量 | ≥2份独立来源一致 | 仅1份 |
| 跨机构普适性 | 经过多机构验证 | 单机构样本 |
| 执行量影响 | 已评估 | 未评估 |
| 反馈层位 | 领导+执行双重确认 | 仅单一层位确认 |

### 10.3 当前V3的试行表达清单

| 试行内容 | 类型 | 原因 |
|---------|------|------|
| 双门槛（1000万+5000万）| 试行 | 仅Q2一份反馈，执行量影响未知 |
| 企金部主办+战略研究部支持 | 试行 | 仅Q1一份反馈，跨行差异未验证 |
| 企金部/对公业务部降调写法 | 试行 | 同上 |

## 十二、V3 检查表（10问）

写完一条动作后，用以下10问自检：

1. **动作类型是否明确？** → 不能把研究准备写成立即执行
2. **动作是否具体？** → 不能用"加强/关注/做好"开头
3. **责任是否到部门+角色？** → 不能只写部门
4. **研究准备是否有主办+支持双角色？** → 不能只写单一部门
5. **是否有可验证指标？** → 不能写"持续跟踪"
6. **是否区分了"立即执行/研究准备"？** → 两类动作不能混格式
7. **研究准备是否有接收方？** → 不能留空
8. **研究准备是否有转动作条件？** → 不能留空
9. **双门槛是否加注"试行"？** → 不加注则视为固化，不符合V3原则
10. **试行表达是否有反馈触发机制？** → 试行内容下月必须主动确认
