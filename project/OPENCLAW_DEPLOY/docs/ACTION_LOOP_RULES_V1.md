# ACTION_LOOP_RULES_V1.md — 动作闭环规则 V1

> 文档版本：v1.0（新建）
> 编制日期：2026-04-09
> 对应阶段：P2-3（动作闭环升级 + 责任归口收窄）
> 用途：定义研究准备动作的完整闭环要素，以及研究→执行的触发条件

---

## 一、动作类型及闭环要求

### 1.1 三类动作的闭环要求对比

| 动作类型 | 本月目的 | 闭环起点 | 闭环终点 | 是否有"下月接续" |
|---------|---------|---------|---------|----------------|
| **立即执行** | 推动一件具体的事 | 执行人在本月完成这件事 | 完成后验证结果（达到/未达到）| 否（本月闭环）|
| **研究准备** | 搞清楚一件事，给下月提供判断依据 | 研究人启动研究 | 研究结论被接收方阅读，并决定是否触发执行动作 | ✅ 是 |
| **建立机制** | 启动一个持续运行的机制 | 责任人在本月完成初始化 | 机制进入常态运行，下月由谁运行待定义 | 是（但本月无结果）|

## 二、研究准备动作的最小闭环要素

**研究准备动作必须包含以下 7 个要素，缺一不可：**

| # | 要素 | 定义 | 典型问题（若缺失）|
|---|------|------|----------------|
| 1 | **研究主题** | 要搞清楚的具体问题 | 不知道要研究什么 |
| 2 | **研究目的** | 搞清楚这个问题是为了支持什么判断 | 研究完了不知道怎么用 |
| 3 | **研究责任人** | 谁来做这个研究（部门+角色）| 没人接活 |
| 4 | **研究内容** | 具体要拿到什么证据/数据/结论 | 研究产出不对路 |
| 5 | **产出形式** | 结论以什么形式呈现（报告/表格/备注）| 结论散乱无法传递 |
| 6 | **接收方** | 研究完成后，谁来看这份产出 | 研究完了没人看 |
| 7 | **转动作条件** | 什么条件下，这个研究结论会触发下月的执行动作 | 研究结论永远停在研究结论 |

**若缺失第6项（接收方）**：研究完成后没人看，研究白做。
**若缺失第7项（转动作条件）**：研究结论永远停在结论，不会转化为下月动作。

## 三、"接收方"的定义与写法

### 3.1 什么是"接收方"

研究准备动作的产出（报告/表格/备注）完成后，需要交给下一个角色阅读并做出判断。这个"下一个角色"就是接收方。

**接收方不等于研究责任人**。研究责任人负责做研究，接收方负责看完结论后决定下一步。

### 3.2 接收方的判断逻辑

研究准备动作的产出，会流向哪类角色？

| 研究主题类型 | 研究产出接收方（典型）|
|------------|-------------------|
| 存款竞争策略研究 | 分行公司部负责人（或分行长）|
| 贷款利率策略研究 | 资产负债部/计财部负责人 |
| 客户挖转风险评估 | 分行客户管理部门负责人 |
| 宏观政策对存款影响 | 战略研究部/企金部负责人 |
| 单一客户异动研究 | 客户所属业务部门负责人 |

### 3.3 接收方的写法

```
接收方：分行公司部负责人（或分行长）
```

**禁止写法**：
- ❌ "交由相关部门"（太泛）
- ❌ "留存备查"（等于没人看）
- ❌ 不写（等于白做）

## 四、"转动作条件"的定义与写法

### 4.1 什么是"转动作条件"

当研究结论出来后，在什么条件下，应该触发下月的执行动作？写清楚这个条件，研究结论才不会永远停在"结论"状态。

### 4.2 转动作条件的结构

每个转动作条件，必须包含：
1. **触发事实**：研究结论中出现的具体事实/数据
2. **判断门槛**：达到什么标准就触发
3. **对应动作**：触发后，下月应该执行什么动作

**标准格式**：
```
若[触发事实]达到[判断门槛]，则下月启动[对应动作]
```

### 4.3 转动作条件示例

**示例1（华夏存款研究）**：
```
转动作条件：
  若华夏银行3年定存利率持续高于我行15bp以上，
  且网点扩张月均新增>5个，
  则下月启动"华夏银行客户挖转风险排查"主动作
```

**示例2（招行联动研究）**：
```
转动作条件：
  若本期扫描发现3户以上重点客户反映"近期有招行联系过"，
  则下月启动"招行贷存款联动客户流失风险排查"主动作
```

**示例3（定期存款占比研究）**：
```
转动作条件：
  若样本行中3家以上银行定期存款占比月增>2个百分点，
  则下月启动"我行定期存款产品竞争力评估"主动作
```

### 4.4 转动作条件的降调处理

如果研究结论不确定，转动作条件也应该降调：

```
转动作条件（降调版）：
  目前数据样本不足以给出确定判断。
  预计下月可形成结论，届时根据结论类型触发对应动作。
  - 若华夏增速持续>5%：启动客户挖转排查
  - 若华夏增速回落至<3%：本轮研究归档，不再触发动作
```

## 五、研究准备动作的标准格式（V1）

```
▸ 研究准备：[研究主题]
    研究目的：[搞清楚这个问题是为了支持什么判断]
    研究责任人： [部门] / [角色]（如：分行公司部 / 行业研究员）
    研究内容：
      1. [具体要拿到的证据/数据/结论 1]
      2. [具体要拿到的证据/数据/结论 2]
    产出形式：[报告 / 表格 / 备注]（如：一页书面判断报告）
    接收方：[部门] / [角色]（如：分行公司部负责人）
    转动作条件：
      若[触发事实]达到[判断门槛]，则下月启动[对应动作]
      若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    预计完成时点：XXXX年XX月XX日
    注意：此动作本月不产生直接业务动作，研究结论是给下月用的
```

## 六、"立即执行"动作的闭环要素

立即执行动作不需要"接收方"和"转动作条件"，但必须有明确的"完成时点"和"验证结果"：

| 要素 | 说明 |
|------|------|
| 动作主题 | 要做的具体行动 |
| 目的 | 防范什么风险 / 抓住什么机会 |
| 责任人 | 部门 + 角色 |
| 验证指标 | 直接指标 + 间接指标 + 预警阈值 |
| 完成时点 | 具体日期（不能只写"本月"）|
| 完成结果 | 完成后需要确认"达到/未达到"验证指标 |

---

## 七、动作升级 V2 的两个核心改变

| V1 版本 | V2 版本 |
|---------|---------|
| 研究准备无"接收方" | 研究准备必须有"接收方" |
| 研究准备无"转动作条件" | 研究准备必须有"转动作条件" |
| 责任只到部门 | 责任到"部门+角色" |

---

## 八、检查表：研究准备动作有没有闭环？

写完一条研究准备动作后，用以下 7 问自检：

1. **研究主题是否具体？** → 不是"研究一下"，而是"搞清楚XXX"
2. **研究目的是否清楚？** → 不是"了解情况"，而是"为下月判断XXX提供依据"
3. **研究责任人是否到部门+角色？** → 不能只写部门
4. **研究内容是否具体？** → 不能只写"做分析"，要写清楚要拿什么数据/结论
5. **产出形式是否明确？** → 不能只写"报告"，要写清楚"几页/什么格式/给谁看"
6. **接收方是否写了？** → 不能留空
7. **转动作条件是否写了？** → 不能留空；若研究结论不确定，至少写"降调版"
