## 一、本包定位

**本包不是**：
- ❌ 配置管理系统
- ❌ 后台编辑页面
- ❌ 自动调度系统
- ❌ 正式运营平台

**本包是**：
- ✅ 只读运营入口页（single-chain-ops.html）
- ✅ 最小运营包文档（本文档）
- ✅ 双入口架构：config-status.html（配置） + single-chain-ops.html（运营）

**本包解决什么问题**：
- 运营者不知道当前项目处于什么状态
- 运营者不知道"现在能不能跑"
- 运营者不知道"下一步该做什么"
- 运营者不知道"什么情况下不能乱动"

## 二、当前统一状态口径

```
V7 基线冻结 / RUN-01 已完成 / 等待下一触发条件 / 第二链路冻结 / V7 基线冻结
```

**禁止表述**：
- ❌ "继续运行中"（漂移）
- ❌ "随时可以跑第二轮"（无触发条件）
- ❌ "第二链路待解冻后继续"（无来源）
- ❌ "阶段2未完成"（误导）

**正确表述**：
- ✅ "RUN-01 已完成，等待下一触发条件，第二链路继续冻结"
- ✅ "第一条链路成立，第二条链路冻结，解冻需用户提供候选来源"

## 四、为什么不立即进入 RUN-02

| 原因 | 说明 |
|------|------|
| 无新数据驱动 | usable 池已确认，无显著新增需要 |
| 无异常信号 | 无集中负向反馈，无维度缺口信号 |
| 无时间窗口触发 | 月度节奏未到（距 RUN-01 完成不足1天）|
| 无用户要求 | 用户未发出"开始下一轮"指令 |
| RUN-01 本身已完整 | 首轮从"跑通"到"收口"，结论清晰，无遗留问题 |

## 五、次轮触发规则（v1.0）

### 触发条件

| 触发类型 | 条件 | 触发动作 |
|---------|------|---------|
| **正常时间触发（主）** | 下一月度运行窗口到达 | 用户发出"开始 RUN-next"指令 |
| **数据触发（辅）** | 曾刚连续2个月0贡献 | 用户评估是否触发 RUN-next |
| **数据触发（辅）** | 月度 usable<3条 | 用户评估是否触发 RUN-next |
| **数据触发（辅）** | 出现集中负向反馈（≥2人同维度C/D）| 用户评估是否触发 RUN-next |
| **用户主动触发（辅）** | 用户明确要求提前运行 | 用户发出指令 |
| **用户主动触发（辅）** | 用户提供债券货币来源候选 | 触发第二链路解冻评估 |

### 不构成触发

| 情形 | 为什么不触发 |
|------|-------------|
| "想再跑一次看看" | 无新数据、无新反馈、无时间驱动 |
| 刚跑完 RUN-01 就想跑 RUN-02 | RUN-01 结论清晰，无遗留问题 |
| Usable 池有小幅波动 | 32条充足，小幅波动正常 |
| 某分析师某月无新文章 | 单月断档不构成触发（需连续2个月）|
| 主观感觉应该再跑 | 感觉类想法不构成触发条件 |

### 辅触发 ≠ 自动执行

辅触发是"评估通道"，不是"立即执行 RUN-02"：
触发辅条件 → agent 评估 → 用户决定是否触发 RUN-next
