## 一、段落分类体系

**段落类型：**
- `解释型段`：说明"为什么"，不参与判断
- `判断型段`：直接提供决策依据
- `训练型段`：防止误用，培养正确心智
- `路径型段`：引导下一步去哪
- `留痕型段`：提供最小输出模板
- `治理型段`：给维护者看的治理说明

**重复类型：**
- `完全重复`：同一段在多处出现且完全相同
- `近重复`：同一意思在多处表达，高度相似但有差异
- `角色性重复`：同一意思因页面角色不同而有不同厚度，有意设计
- `功能重复`：看似重复但功能定位不同

## 五、段落维护原则

**原则1：角色性重复保留**
同一意思因页面角色不同而有不同厚度，这是合理设计，不压缩。

**原则2：完全重复段必须同步维护**
P-SH-001~004 值班模板等完全重复段，维护时必须全站同步。

**原则3：S0/S1句所在段落优先保护**
涉及 S0/S1 句的段落（P-LC/P-2L-004/P-ES等）在改动前必须完成全站评估。

**原则4：近重复段按需整合**
P-SC（场景判例）类近重复段建议建立"全站场景库"，各页引用而非独立维护，但本轮不实施。
