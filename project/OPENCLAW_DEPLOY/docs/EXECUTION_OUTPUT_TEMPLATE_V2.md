## 三、双门槛试行表达规则（V2 新增）

### 3.1 为什么是双门槛

Q2反馈（FB-P2-20260410-001）指出：单一门槛5000万偏高，建议降到1000万。

但直接改成1000万会使执行量扩大5倍，月均触达量对执行层承载力的影响未知。

**V2折中方案：双门槛表达**
- 第一层（1000万以上）：纳入快速扫描范围，不需要全部联系，重点是摸底有没有异常信号
- 第二层（5000万以上）：纳入重点排查，必须联系，确认续做意向，防范挖转风险

### 3.2 双门槛格式

```
▶ 主动作1（本 月 必 做）：[动作名称]
    目的：[做这件事是为了防范什么/抓住什么机会]
    建议责任人：分行公司部 / 客户经理
    验证指标：
        重点排查：联系5000万以上存款到期客户，确认续做意向；已有转投迹象的，立即上报
        快速扫描：排查1000万以上存款客户有无异常信号（不必全部联系，重点是有没有异常）
        预警阈值：存量客户中单笔转投金额≥1亿元，上报分行公司部负责人
    完成时点：XXXX年XX月XX日
```

**注（V2加注）**：双门槛为试行表达。执行层实际触达量取决于各行存量客户数量。试行期间若发现1000万扫描范围过宽或过窄，下月反馈中注明，模板据此调整。

### 3.3 双门槛的适用范围

| 动作类型 | 是否适用双门槛 |
|---------|-------------|
| 客户挖转风险排查 | ✅ 适用（排查类动作天然有轻重之分）|
| 客户到期续做提醒 | ✅ 适用（淡旺季差异大，门槛有弹性）|
| 招行联动扫描 | ❌ 不适用（这是基准建立，不需要双门槛）|
| 建立机制类动作 | ❌ 不适用（机制类不是本月完成）|

## 四、研究准备责任归口调整说明（V2 新增）

### 4.1 为什么调整

Q1反馈（FB-P2-20260410-001）：研究准备责任人应归"企金部（对公业务部）"，而非"战略研究部"。

这反映的是不同银行之间组织架构差异：
- 企金部（对公业务部）：更贴近市场一线，最早感知竞品动态
- 战略研究部：更偏向宏观策略，行业研究能力更强但时效性可能偏弱

### 4.2 V2折中写法

**不直接替换，而是联合主办：**

```
建议责任人：企金部（主办）+ 战略研究部（支持）
角色建议：企金部行业研究员（主办）+ 战略研究部宏观研究员（支持）
```

**为什么这样写：**
- 企金部主办：反映Q1反馈的核心诉求（一线业务视角主导研究）
- 战略研究部支持：保留V1的宏观分析能力托底
- 不是"替换"，而是"联合"

### 4.3 组织差异降调写法

若某银行没有战略研究部，或企金部明确表示不承接，可简化为：

```
建议责任人：企金部 / 对公业务部（注：若无专门研究支持，可由企金部产品经理兼任）
```
