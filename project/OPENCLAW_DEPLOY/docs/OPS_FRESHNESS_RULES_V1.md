## 一、本包定位

**本包是什么：**
- 不是动态 freshness 系统（无实时刷新/自动提醒）
- 不是健康度系统（无评分/告警/趋势）
- 不是 reminder 系统（无定时提醒/推送）
- 是：**只读新鲜度分类与有效窗口模块**（嵌入现有页面）

**本包解决什么问题：**
- 不知道当前信息是否仍然新鲜
- 不知道哪些结论是当前有效、哪些只是最近一次确认
- 不知道多久不更新仍算正常等待
- 不知道页面显示的是最新状态还是最近确认状态

## 二、新鲜度四分类

### 2.1 A 类：当前有效

| 项 | 内容 |
|----|------|
| 定义 | 当前已确认且仍在有效窗口内 |
| 举例 | RUN-01 已完成（结论A）/ 第二链路冻结口径 / V7 基线冻结口径 / 十页结构 |
| 回访用户需要知道 | 这些结论当前仍然有效，不需要重新验证 |

### 2.2 B 类：正常等待

| 项 | 内容 |
|----|------|
| 定义 | 没有新变化，但仍处于合理等待窗口内 |
| 举例 | RUN-next 未触发 / 无新数据但未超复核阈值 |
| 回访用户需要知道 | 这是正常节奏，不是失效，不需要采取动作 |

### 2.3 C 类：待下次检查

| 项 | 内容 |
|----|------|
| 定义 | 当前结论依然可用，但下一个窗口应优先检查 |
| 举例 | 回访用户关注项 / 下个月度窗口 / 例行复核项 |
| 回访用户需要知道 | 下次回来时先检查这些项 |

### 2.4 D 类：偏旧需复核

| 项 | 内容 |
|----|------|
| 定义 | 超过建议窗口，虽然未证明失效，但需要优先复核 |
| 举例 | 当前状态：此类为空（无超窗口情况）|
| 回访用户需要知道 | 静态快照不代表自动刷新，下次回访时按 A/B/C 判断 |

## 四、如何向回访用户解释"为什么没变仍然有效"

**核心口径**：没变 ≠ 过期。当前有效不是"永久有效"，而是"在冻结/等待状态下，当前结论仍然成立，不需要重新验证"。

| 情况 | 为什么没变仍然有效 |
|------|-----------------|
| 冻结状态 | 冻结边界明确，在解冻条件满足前，冻结结论持续有效 |
| 等待触发 | 触发条件未命中，在命中前等待是正确节奏 |
| 无新数据 | 没有新输入，强行运行无意义，正常等待 |

---

## 五、何时应视为需要复核

| 触发条件 |
|---------|
| 到达下个月度运行窗口 |
| 收到新数据 / 新输入信号 |
| 有人提出新触发建议 |
| 出现异常信号（usable 异常下滑等）|

## 六、不应误解的事项

| 误解 | 实际 |
|------|------|
| 没更新 = 过期失效 | ❌ 没更新 ≠ 过期，冻结/等待状态下结论仍然有效 |
| 等待态 = 不可信 | ❌ 等待是正常节奏，不是系统问题 |
| 页面版本更新 = 状态刷新 | ❌ 页面版本号更新 = 结构完善，非状态刷新 |
| 上次对过 = 永久有效 | ❌ 当前有效是有条件的，下次回访应按 A/B/C 重新判断 |
| 静态快照 = 实时数据 | ❌ 页面是"最近一次确认结果"的展示，不代表自动刷新 |
