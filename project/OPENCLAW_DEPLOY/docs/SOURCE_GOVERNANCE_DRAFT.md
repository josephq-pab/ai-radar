## 数据源治理目标

在 MVP 阶段，以**最小管理成本**保持数据源的可信度和有效性，避免无效来源持续占用抓取资源。

---

## 数据源分类

| 分类 | 说明 | 可直接进入结论 | 需人工复核 | 当前处理方式 |
|------|------|--------------|-----------|------------|
| 核心源 | 与维度高度相关，作者背景强，更新频率稳定 | 是 | 否 | 直接使用 |
| 补充源 | 与维度相关，有一定价值，更新频率一般 | 否 | 是 | review 后使用 |
| 参考源 | 与维度间接相关，或数据质量不稳定 | 否 | 是 | 降级使用（仅摘要） |
| 隔离源 | 纯舆情/市场类，与维度关联弱 | 否 | 是 | 需明确标注非结论性 |

## 启用/停用规则

> ⚠️ 以下规则为候选初稿，v1.0 正式版发布后生效。

| 条件 | 动作 |
|------|------|
| 连续 3 次抓取成功 0 篇 | 降级为参考源 |
| 连续 5 次抓取成功 0 篇 | 暂停来源，标注原因 |
| URL 连续 3 次 404/403 | 标记为失效，更新 analyst_sources.json |
| 作者主动停更 | 移入隔离源，人工确认是否移除 |

---

## 可信等级规则

| 等级 | 定义 | 使用范围 |
|------|------|---------|
| L1 | 机构官网 / 研究员专栏，背景可验证 | 结论可直接引用 |
| L2 | 财经媒体 / 平台专栏，编辑审核 | 结论需标注来源 |
| L3 | 社交媒体 / 自媒体，背景不明 | 降级使用，不单独作为结论支撑 |

---

## 周期适配规则

- **每次周度抓取后**：检查是否有新出现的来源平台，记录到 fetch-run-log.json
- **每月 review 时**：按启用/停用规则检查来源有效性
- **来源变更时**：立即更新 analyst_sources.json，不追溯修改历史 JSON

---

## 当前阶段暂不纳入的后台能力

| 能力 | 原因 |
|------|------|
| 数据源评分系统 | MVP 阶段人工判断足够 |
| 自动发现新来源 | 边界不可控，需人工审核 |
| 来源质量可视化 dashboard | 超过 MVP 范围 |
| 多来源交叉验证 | 需要足够数据量才能做 |
| 来源贡献度分析 | P1-1 追踪表建立后再评估 |
