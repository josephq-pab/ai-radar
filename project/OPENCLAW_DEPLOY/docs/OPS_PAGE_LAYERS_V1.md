## 背景

当前已形成十页结构（P3-11~P3-25），内容相对完整，但主次层级不清晰：
- 用户容易误以为十页都需要全看
- 导航全量展示，缺少优先级和分层感
- 索引页被当作日常先看页使用
- 核心问题和"信息不够"无关，而和"主次不清"有关

本功能包不是新增入口页，而是在现有十页基础上做分层说明和优先级定义。

---

## 页面三层分类

### A. 核心层（Core Pages）— 优先看

| 页面 | 文件名 | 定位 |
|------|--------|------|
| 首页 | radar-home.html | 总入口，状态总览，能做什么/不能做什么 |
| 快报 | ops-brief.html | 状态快报，快速了解项目现状，对外/对内口径 |
| 运营 | single-chain-ops.html | 运营状态，触发规则，冻结边界 |

**使用标准：**
- 适合：所有人/多数场景，优先进入
- 何时看：首次访问 / 想快速了解现状 / 不知道从哪开始
- 何时不必先看：已知道当前状态，只需要查具体场景时
- 最小必看集：只看核心层3页即可掌握全貌

---

### B. 辅助层（Support Pages）— 需要再看

| 页面 | 文件名 | 定位 |
|------|--------|------|
| 决策助手 | ops-decision.html | 场景化决策指引，该做什么，该查哪页 |
| 执行流程 | ops-playbook.html | 命中触发后8步执行顺序，停点，产出 |
| 配置入口 | config-status.html | analyst_sources.json 只读快照，Gate状态 |
| 证据入口 | ops-evidence.html | 状态为何成立，链路冻结依据，历史节点 |

**使用标准：**
- 适合：需要做判断/执行/查配置/查证据时
- 何时看：核心层看完后 / 需要做具体判断 / 需要看执行步骤 / 需要查依据
- 何时不必先看：没有具体场景，不需要做判断时
- 按需进入，不需要全部都看

---

### C. 索引层（Index Pages）— 辅助查

| 页面 | 文件名 | 定位 |
|------|--------|------|
| 术语口径 | ops-glossary.html | 高频术语统一解释，关系澄清 |
| 真源登记 | ops-registry.html | 元信息索引，真源归属，优先级说明 |
| 角色路径 | ops-routes.html | 角色化阅读路径入口，交接指引 |

**使用标准：**
- 适合：特定目的时查，不适合作为默认入口
- 何时看：遇到术语不明确 / 不知道该信哪个文档 / 不知道按什么顺序读
- 何时不必先看：刚接触项目时 / 日常查看状态时
- 索引页不是起点，是辅助定位工具

## 最小必看集

| 场景 | 最少只需看 | 想更深入时再看 |
|------|-----------|--------------|
| 只想快速了解现状 | 核心层3页 | 辅助层按需 |
| 想判断能不能跑 | 首页+运营 | 决策助手+执行流程 |
| 遇到冻结/阻断 | 快报+运营 | 证据+真源登记 |
| 想完整理解全貌 | 核心层3页+角色路径 | 全部十页 |

## 当前不应误解的事项

| 误解 | 正解 |
|------|------|
| 十页都同样重要 | 否，核心层优先，辅助层/索引层按需 |
| 索引页是日常先看页 | 否，索引页是辅助定位工具，不是起点 |
| 证据页是所有人必读 | 否，证据页主要给需要理解依据的人 |
| 真源登记是第一步 | 否，真源登记是索引辅助，不是起点 |
| 有页面=有后台能力 | 否，所有页面只提供只读展示层 |
| 页面数量多=信息量大=必须全看 | 否，十页是分工结构，不需要全部看完 |

## 推荐默认阅读起点

1. **首次访问**：radar-home.html（首页）→ ops-brief.html（快报）
2. **想判断现状**：ops-brief.html（快报）→ single-chain-ops.html（运营）
3. **想完整理解**：ops-routes.html（角色路径）→ 按角色推荐顺序看
4. **遇到场景不知该干啥**：ops-decision.html（决策助手）

---

## 两种阅读方式对比

| 阅读方式 | 适合谁 | 核心问题 |
|---------|--------|---------|
| 按角色路径 | 已知自己是哪类角色 | 我看完能回答什么 |
| 按层级路径 | 时间有限，想快速定位 | 我现在该看什么 |

两种方式可以结合：先用时间预算找到入口（核心层），再用角色路径确认完整阅读范围（辅助层/索引层按需）。
