## 一、本包定位

**本包是什么：**
- 不是告警系统（无实时监控/告警触发）
- 不是工单系统（无分配/流转/状态追踪）
- 不是 SRE 台（无 SLA/错误预算/异常告警）
- 是：**只读异常分类与升级规则模块**（嵌入现有页面）

**本包解决什么问题：**
- 出现异常时不知道是观察、停止、修正口径，还是升级讨论
- 误判：口径不一致=系统异常 / 冻结=故障 / 无新数据=要重跑 / 页面差异=真源冲突

## 二、异常分类总览

### 2.1 信息类异常

| 项 | 内容 |
|----|------|
| 举例 | 页面版本标识未更新 / 口径轻微不一致 |
| 是否构成真源冲突 | ❌ 否（展示层问题）|
| 默认动作 | 修正口径 / 不升级 |
| 优先级 | 日常维护 |

### 2.2 数据类异常

| 项 | 内容 |
|----|------|
| 举例 | usable 明显下滑 / 连续 0 贡献 / 池异常缩小 / 新数据结构异常 |
| 是否构成真源冲突 | ❌ 否（数据波动）|
| 默认动作 | 观察 or 进入触发评估（不自动重跑）|
| 优先级 | 触发评估 |

### 2.3 触发类异常

| 项 | 内容 |
|----|------|
| 举例 | 条件是否命中不明确 / 介于建议与触发之间 / 月度窗口边界模糊 |
| 是否构成真源冲突 | ⚠️ 视情况 |
| 默认动作 | 停止 + 判断，不自动执行 |
| 优先级 | 高 |

### 2.4 边界类异常

| 项 | 内容 |
|----|------|
| 举例 | 有人建议动冻结项 / 改 V7 / 推进第二链路 |
| 是否构成真源冲突 | ⚠️ 视建议内容 |
| 默认动作 | 先看边界，不自动执行 |
| 优先级 | 边界优先 |

### 2.5 真源类异常

| 项 | 内容 |
|----|------|
| 举例 | 页面与文档不一致 / 文档间口径冲突 |
| 是否构成真源冲突 | ✅ 是 |
| 默认动作 | 按优先级处理，必要时修正登记 |
| 优先级 | 高 |

---

## 三、升级讨论条件

**必须同时满足才进入升级讨论：**

1. 属于边界类或真源类异常
2. 涉及冻结项
3. 无明确处理规则
4. 需要人工决策

**不构成升级讨论理由：**
- ❌ 单次轻微信号
- ❌ 有人建议动冻结项（先看边界）
- ❌ 页面口径轻微不一致
- ❌ 无新数据

---

## 四、不构成异常的情况

| 情况 | 不是异常，是... |
|------|---------------|
| 冻结状态 | 有依据地暂停 |
| 无新数据 | 正常等待，非故障 |
| 页面差异（轻微）| 展示层问题，日常维护 |
| 有人建议 | 仅是建议，不自动执行 |
| 触发条件未命中 | 正常节奏，非异常 |

---

## 五、异常与真源冲突处理

| 异常类型 | 是否构成真源冲突 | 处理方式 |
|---------|---------------|---------|
| 页面版本标识未更新 | ❌ 否 | 修正页面标识 |
| 页面与文档表述不一致 | ⚠️ 视情况 | 以文档为准，修正页面 |
| 文档间口径冲突 | ✅ 是 | 查优先级规则，修正登记 |
| 有人建议动冻结项 | ⚠️ 视建议内容 | 先看边界，不自动执行 |
| usable 明显下滑 | ❌ 否 | 观察，必要时进入触发评估 |

## 六、容易误判的关系

| 误解 | 实际 |
|------|------|
| 口径不一致 = 系统异常 | ❌ 口径不一致 ≠ 系统异常 |
| 冻结 = 故障 | ❌ 冻结 ≠ 故障 |
| 无新数据 = 要重跑 | ❌ 无新数据 ≠ 要重跑 |
| 页面差异 = 真源冲突 | ❌ 页面差异 ≠ 真源冲突 |
| 单次轻微信号 = 要升级 | ❌ 单次轻微信号 ≠ 要升级 |
| 有人建议 = 必须执行 | ❌ 有人建议 ≠ 必须执行 |

---

## 七、异常时在执行流程中的停点

| 异常类型 | 触发阶段 | 默认动作 | 能否跳过停点 |
|---------|---------|---------|------------|
| 信息类异常 | 任意阶段 | 修正口径，继续执行 | 可以 |
| 触发条件不明确 | Step 1 触发检查 | 停止，不进入 Step 2 | ❌ 不能跳过 |
| 输入数据明显异常 | Step 3 输入校验 | 停止，不进入 Step 4 | ❌ 不能跳过 |
| 边界类异常 | 任意阶段 | 停止，先看边界规则 | ❌ 不能绕过 |
| 运行结果偏离预期 | Step 6 结果判断 | 进入结论判断，不自动推进 | ❌ 不能绕过 |

**核心原则：不自动跳过停止点。出现异常时，先判断异常类型，再决定是否继续。**
