## 背景

P3-26 已完成十页三层分类（核心层/辅助层/索引层），但核心三页之间没有形成明确的闭环路径：
- 用户不知道三步之间"看完后去哪"
- 用户不知道三步走完后"何时可以停"
- 用户不知道"什么情况下才需要跳辅助页"

本功能包不是新增入口页，而是在现有核心三页中嵌入闭环说明和停止条件。

---

## 核心三页各自回答什么

| 步骤 | 页面 | 核心问题 |
|------|------|---------|
| 第一步 | radar-home.html（首页）| 这套系统是做什么的？现在处于什么状态？有哪些能力/边界？入口在哪？ |
| 第二步 | ops-brief.html（快报）| 当前状态是什么？最重要的一件事是什么？值得深读吗？ |
| 第三步 | single-chain-ops.html（运营）| 现在能不能跑？为什么不能/什么时候能？我现在该做什么？ |

---

## 默认阅读顺序

```
第一步：radar-home.html（首页）
  ↓ 看完这里：知道自己在什么系统里，知道有没有需要立即处理的事
第二步：ops-brief.html（快报）
  ↓ 看完这里：能用最短文字总结当前状态，知道是否需要继续
第三步：single-chain-ops.html（运营）
  ↓ 看完这里：知道运营上是否可以停止，还是需要采取行动
```

---

## 看完三页后：何时可以停止

### 正常态（可停止）

同时满足以下条件时，不需要再看其他辅助页：

| 条件 | 当前状态 |
|------|---------|
| RUN-next 未触发 | ✅ 未命中，条件未满足 |
| 第二链路冻结 | ✅ 结构性来源缺失 |
| V7 基线冻结 | ✅ 需连续2月正向反馈才解冻 |

**结论**：当前处于正常等待态，不需要任何动作，可停止。

### 异常态（必须继续深查）

出现以下信号时，必须继续判断：

| 信号 | 说明 | 跳转到 |
|------|------|-------|
| 触发条件已命中 | 正常触发（到月度窗口）或异常触发（曾刚断档/集中负向反馈）| ops-decision.html |
| usable 池异常下滑 | &lt;5条或集中负向反馈 | ops-decision.html |
| 收到用户强烈信号 | 曾刚连续断档/集中负向反馈 | ops-decision.html |

## 默认闭环 vs 角色路径

| 使用方式 | 什么时候用 | 覆盖页面 | 什么时候停 |
|---------|-----------|---------|---------|
| 🟢 **默认闭环路径** | 不知道自己是哪类角色 / 只想快速了解现状 / 日常状态查看 | 首页→快报→运营页（3步）| 三步走完，当前为等待态 → 可停止 |
| 🔵 **角色路径** | 已知自己是哪类角色 / 需要完整交接 / 需要做具体判断 | ops-routes.html→按角色推荐顺序读 | 角色任务完成或遇到阻断项 |

**推荐默认方式**：对大多数人，先走默认闭环路径（三步）。如果你是特定角色（执行/管理/领导），再走角色路径。

## 当前不应误解的事项

| 误解 | 正解 |
|------|------|
| 看完快报就够了，不需要看运营页 | 否，快报不能替代运营页的"能否跑"判断 |
| 看完运营页后必须再去看辅助页 | 否，当前为等待态时，三步走完即可停止 |
| 核心三页没有明确停点 | 否，RUN-next未触发+第二链路冻结+V7冻结=可停止 |
| 想理解"为什么"就必须去看证据页 | 否，理解"为什么不能跑"是可选的，不是必须的 |
