## 一、本包定位

**本包是什么：**
- 不是动态 changelog（无时间线/版本流）
- 不是更新日志系统（无自动版本刷新）
- 不是页面版本追踪（页面永远只是展示层）
- 是：**只读增量变化摘要模块**（嵌入现有页面）

**本包解决什么问题：**
- 回访用户不知道上次之后哪些变了
- 没变时不知道为什么还没变
- 变了时不知道变更幅度和影响
- 返回用户仍要重新阅读过多内容

## 二、变化类型定义

### 2.1 A类：实质状态变化

| 项 | 内容 |
|----|------|
| 定义 | RUN执行/usable变化/冻结状态变化 |
| 回访用户需要知道 | 这次真的变了什么，变化幅度多大 |
| 看什么 | ops-playbook.html + ops-evidence.html |
| 真源文档 | STAGE2_SINGLE_CHAIN_RUN_CLOSE_01.md |

### 2.2 B类：结构变化

| 项 | 内容 |
|----|------|
| 定义 | 入口数量变化/模块新增/角色路径新增/阅读模式新增 |
| 回访用户需要知道 | 这次新增了哪些入口/模块/能力 |
| 看什么 | radar-home.html + 对应入口页 |
| 真源文档 | STAGE2_SINGLE_CHAIN_STATUS.md 各功能包节 |

### 2.3 C类：口径变化

| 项 | 内容 |
|----|------|
| 定义 | 术语统一/边界澄清/异常规则补充/真源登记补充 |
| 回访用户需要知道 | 这次澄清了什么，表述是否更准确 |
| 看什么 | ops-glossary.html + ops-registry.html |
| 真源文档 | 对应功能包文档 |

### 2.4 D类：无变化但需解释

| 项 | 内容 |
|----|------|
| 举例 | RUN-next仍冻结/第二链路仍冻结/V7仍冻结/无新数据 |
| 回访用户需要知道 | 为什么没变，这是正常的吗 |
| 看什么 | ops-brief.html 首页横幅 + radar-home.html 状态横幅 |
| 真源文档 | STAGE2_SINGLE_CHAIN_TRIGGER_RULES_V1.md / STAGE2_SINGLE_CHAIN_STATUS.md |

## 四、如何向回访用户说明"为什么没变"

**核心口径**：没变 ≠ 没做。冻结是主动选择，停在等待态是正常节奏，不是失败。

| 情况 | 为什么没变 |
|------|-----------|
| RUN-next 未触发 | 触发条件未命中，等待是正常节奏 |
| 第二链路仍冻结 | 结构性来源缺失，等用户提供候选是正确做法 |
| V7 仍冻结 | 需连续2月正向试用反馈才解冻，当前是冻结状态 |
| 无新数据 | 正常等待，非系统失效或数据问题 |

## 六、不应误解的事项

| 误解 | 实际 |
|------|------|
| 无新数据 = 系统失效 | ❌ 正常等待，非系统问题 |
| 状态没变 = 没有进展 | ❌ 结构完善是进展，口径澄清是进展 |
| 文案修正 = 实质变化 | ❌ C类口径变化，不是A类实质状态变化 |
| 页面版本更新 = 项目状态变化 | ❌ P3编号变更=结构完善，非实质状态变化 |
| 第二链路冻结 = 项目失败 | ❌ 结构性来源缺失，有依据地冻结 |
| V7冻结 = 内容不可用 | ❌ 冻结的是改版动作，内容仍可使用 |
