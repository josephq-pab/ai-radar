## 背景

P3-30 已完成默认停点与不再深查规则，定义了什么时候应该停止。

但仍存在问题：
- 停下后遇到新输入，不知道该最小怎么重新启动
- 有新输入就默认整套流程重跑
- 有新问题就全链路重查
- 有新来源候选就自动解冻第二链路

本功能包定义新输入分类与最小响应路径。

---

## 新输入四分类

| 类型 | 说明 | 默认动作 |
|------|------|---------|
| 🅰 轻响应输入 | 只影响表述/理解/解释，不影响状态判断 | 更新理解，看快报或术语页，不进入评估 |
| 🅱 评估型输入 | 可能影响"要不要动"，但还不构成运行准备 | 去决策页做判断，不自动执行 |
| 🅲 运行准备型输入 | 已经接近或满足运行前条件 | 运营页→决策页→执行准备，不自动触发 |
| 🅳 结构性输入 | 影响开放项/阻断项/未来方向，但不直接进入本轮运行 | 归入未决事项，继续冻结第二链路，不自动推进 |

---

## 各类型详细说明

### 🅰 轻响应输入

**定义**：只影响表述、理解、解释，不影响状态判断的新输入。

**典型例子**：
- 口径澄清（"上次说的不对，正确的是……"）
- 术语理解（"这个词在我们这里指……"）
- 回访问答（"你之前说的X，现在情况是……"）
- 页面解释需求

**默认动作**：
1. 更新理解
2. 看快报或术语页
3. 不进入触发评估
4. 不重启本轮

**不应误判为**：需要立即重跑 / 需要触发评估 / 需要进入运行准备

---

### 🅱 评估型输入

**定义**：可能影响"要不要动"，但还不构成自动运行的新输入。

**典型例子**：
- "这算不算触发"（不确定是否命中触发条件）
- 轻微信号变化（usable池轻微波动）
- 有人提议提前跑
- 收到集中负向反馈（但不构成明确触发）

**默认动作**：
1. 去决策页做判断
2. 不自动执行
3. 不自动触发运行准备
4. 判断结论为"不动"则回到等待态

**不应误判为**：立即触发 / 自动进入运行准备 / 需要全链路重跑

---

### 🅲 运行准备型输入

**定义**：已经接近或满足运行前条件的新输入。

**典型例子**：
- 明确新数据到达（分析师池出现新可用来源）
- 明显新触发信号（usable池明显异常下滑）
- 连续异常（三次以上同类信号）
- 用户明确要求进入下一轮检查

**默认动作**：
1. 运营页→决策页→执行准备
2. 不自动触发
3. 需要决策页做判断确认
4. 确认触发后才进入执行

**不应误判为**：自动触发 / 不需要判断直接跑 / 立即全链路重跑

---

### 🅳 结构性输入

**定义**：影响开放项/阻断项/未来方向，但不直接进入本轮运行的新输入。

**典型例子**：
- 第二链路来源候选（有人说"某行可能有数据"）
- 架构建议（"能不能加上X来源"）
- 边界调整建议（"能不能扩展到另一类客户"）
- 人员调整建议

**默认动作**：
1. 归入未决事项或结构性评估
2. 不自动解冻第二链路
3. 继续等待，不推进运行
4. 结构性评估结论另作记录，不影响本期等待态

**不应误判为**：立即解冻第二链路 / 有人提了就自动进入 / 自动影响本期运行

---

## 最容易被误判的新输入关系

| 误判 | 正解 |
|------|------|
| 有新问题 → 立即重跑 | 新问题先判断类型，大多数只需要轻响应，不需要重跑 |
| 有新来源候选 → 立即解冻第二链路 | 来源候选归入结构性输入，不自动解冻 |
| 有人说"再看一下" → 触发运行 | 没有触发条件不触发，不需要重跑 |
| 有新数据片段 → 一定进入 RUN-next | 新数据需先判断类型，轻响应不触发，评估后才决定 |
| 文案纠正 → 实质状态变化 | 口径澄清只是轻响应，不改变状态判断 |
| 有人提议提前跑 → 立即进入准备 | 提议属于评估型输入，需决策页判断 |

---

## 新输入最小响应路径

```
停下后遇到新输入
  ↓
判断属于哪类
  ↓
┌─ 轻响应输入 → 更新理解，看快报/术语 → 停止
├─ 评估型输入 → 去决策页判断 → 停止或继续
├─ 运行准备型输入 → 运营→决策→准备 → 按判断执行
└─ 结构性输入 → 归入未决事项 → 继续等待
```

## 各页面新输入响应入口

### radar-home.html

- 新输入来了先判断类型
- 大多数新输入不需要全链路重跑
- 先分流再决定是否进入评估/准备

### ops-brief.html

- 轻响应输入：看快报或术语页即可
- 评估型输入：去决策页
- 结构性输入：归入未决，不自动推进

### single-chain-ops.html

- 只需记录/解释：更新理解，不需要重启
- 回到决策页评估：判断是否触发
- 接近运行准备：运营→决策→准备

### ops-decision.html

- 适合先来：判断类输入（"这算不算触发"）
- 不需要先来：口径/术语/配置/执行类

### ops-registry.html

- 轻响应输入归属：口径/术语层
- 评估型输入归属：评估层
- 运行准备型输入归属：运行准备层
- 结构性输入归属：未决事项/结构性评估

## 当前不应误解的事项

| 误解 | 正解 |
|------|------|
| 有新输入就应重新跑整套流程 | 否，大多数新输入只需要轻响应，不需要重跑 |
| 用户提到一个新点就要全链路重查 | 否，先判断输入类型，大多数只需要更新理解 |
| 任何新来源候选都自动进入第二链路 | 否，来源候选归入结构性输入，不自动解冻 |
| 轻微信息修正也要重新走流程 | 否，口径澄清属于轻响应，不需要触发评估 |
| 有人说"再看一下"就等于触发运行 | 否，没有触发条件不触发，不需要重跑 |
