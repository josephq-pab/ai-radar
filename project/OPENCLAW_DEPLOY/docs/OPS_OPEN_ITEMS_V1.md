## 二、未决事项五分类

### 2.1 A 类：已决但冻结

| 项 | 内容 |
|----|------|
| 定义 | 已有明确结论，当前不再推进 |
| 举例 | V7 基线冻结 / RUN-next 未触发不执行 / 第二链路冻结 |
| 回访用户需要知道 | 这些不是悬案，已有结论，只是冻结 |

### 2.2 B 类：未决待输入

| 项 | 内容 |
|----|------|
| 定义 | 需要用户提供新输入后才可能推进 |
| 举例 | 第二链路来源候选（必须用户提供）|
| 回访用户需要知道 | 这是真正需要用户行动的事项，自寻无效 |

### 2.3 C 类：未决待触发

| 项 | 内容 |
|----|------|
| 定义 | 条件成熟时才进入下一步判断 |
| 举例 | 下次月度窗口 / usable 异常下滑 / 集中负向反馈 |
| 回访用户需要知道 | 现在不需要动作，等条件成熟 |

### 2.4 D 类：阻断但暂不处理

| 项 | 内容 |
|----|------|
| 定义 | 当前知道它是阻断，但没有必要继续消耗 |
| 举例 | 自动运行/调度未做 / 后台配置系统未做 |
| 回访用户需要知道 | 这些是知道但暂时搁置的事项，不是被遗忘 |

### 2.5 E 类：当前明确不做

| 项 | 内容 |
|----|------|
| 定义 | 有意不做，而非未决 |
| 举例 | 配置编辑 / 动态版本系统 / 培训平台 |
| 回访用户需要知道 | 这些是项目边界，不是遗漏，不是将来要做 |

## 六、不应误解的事项

| 误解 | 实际 |
|------|------|
| 冻结项 = 悬而未决 | ❌ 冻结项是已有明确结论的主动暂停 |
| 阻断项 = 现在必须处理 | ❌ 阻断但暂不处理是知道但不继续消耗 |
| 等待触发 = 悬而未决 | ❌ 等待触发是有条件判断，到条件自然进入 |
| 当前不做 = 将来一定做 | ❌ 明确不做是项目边界，有意不做 |
| 未决 = 没有结论 | ❌ 已决但冻结≠未决，有结论只是不推进 |
