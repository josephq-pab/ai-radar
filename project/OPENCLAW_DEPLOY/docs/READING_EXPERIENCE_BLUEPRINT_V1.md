## 一、当前阅读体验痛点

### 痛点清单

| # | 痛点 | 表现 | 影响 |
|---|------|------|------|
| R1 | 视图不分层 | 所有内容平铺在一个页面 | 阅读者无法聚焦，容易走失 |
| R2 | 证据不可追 | 动作建议没有对应到具体分析师文章 | 执行者无法核验，无法判断是否采纳 |
| R3 | 变化不可比较 | 月度报告之间无法对比差异 | 无法快速看出"本月重点是什么" |
| R4 | 结论/动作切换不顺 | 结论和动作混在一起 | 决策者想看结论，执行者想看动作，互相干扰 |
| R5 | 无统一阅读入口 | 各页面分散（deposit/loan/peer-monitor）| 新用户不知道从哪开始 |
| R6 | 展示内容陈旧 | 部分页面数据仍是2026-04-06快照 | 影响信任度 |

## 二、目标阅读层次

本期阅读体验的目标是支持"五层阅读"：

### 五层定义

| 层次 | 阅读者 | 核心问题 | 内容特征 |
|------|-------|---------|---------|
| **结论层** | 领导/决策者 | "本月最重要的一件事是什么？" | 3~5条 bullet，结论先行 |
| **动作层** | 执行者 | "我今天要做什么？" | 动词打头三步式，可直接部署 |
| **证据层** | 分析复核者 | "这个结论是哪个来源支撑的？" | 每个动作对应分析师+文章 |
| **来源层** | 运营管理者 | "当前哪些来源在贡献？" | 来源活跃度+置信度 |
| **变化层** | 复盘参与者 | "本月比上月多了/少了什么？" | 月度 delta，可对比 |

## 三、最小阅读入口定义

### 主入口：index.html（五层切换）

```
index.html
├── 结论层（默认视图）
│   └── 月度最重要变化（bullet list）
├── 动作层
│   └── 执行版核心动作（三步式）
├── 证据层
│   └── 每个动作 → 分析师来源映射
├── 来源层
│   └── 来源活跃度 / 置信度 / 上次更新时间
└── 变化层
    └── 本月 vs 上月 delta
```

**不做**：不做新的 URL 路由，不做 SPA 架构，在现有 HTML 基础上增加视图切换。

### 各页面定位

| 页面 | 定位 | 是否改动 |
|------|------|---------|
| index.html | 主入口 + 结论层/动作层 | ✅ 升级为五层切换 |
| deposit.html | 存款分析详情 | ✅ 增加证据可追 |
| loan.html | 贷款分析详情 | ✅ 增加证据可追 |
| peer-monitor.html | 同业监测详情 | ✅ 保持现状 |
| weekly-report.html | 周报视图 | ✅ 补充月度对比入口 |
| tracking.html | 追踪项 | ✅ 保持现状 |

## 四、"阅读体验已过 MVP 线"的判断标准

**本期 MVP 阅读体验验收线**：

| 标准 | 判断方法 |
|------|---------|
| 执行者能在3秒内找到"今天要做什么" | 从 index.html 出发，3步内到达动作层 |
| 每个动作背后有分析师文章可查 | 动作层有点击展开，能看到来源名称 |
| 本月相比上月有明确差异说明 | 变化层有 bullet 对比，不是全文平铺 |
| 领导版/执行版明确分离 | 两套模板从不同入口可达，不混在一起 |
| 配置状态可见 | 首页有 Gate 状态/来源数量/上次更新时间 |

## 五、本期阅读体验最小改动清单

### 最小改动（必须做）

| 改动 | 目标 | 不做 |
|------|------|------|
| index.html 增加视图切换 tab | 五层阅读 | 不做 SPA 路由，不做动态加载 |
| deposit.html 增加"证据"展开 | 证据可追 | 不做全文引用，做法：点击动作展开分析师来源 |
| loan.html 增加"证据"展开 | 证据可追 | 同上 |
| change-summary.md 月度 delta 明确 | 变化可比较 | 不做多版本对比工具 |
| 首页增加 Gate 状态展示 | 配置可见 | 不做实时告警 |

### 本期不做（明确）

| 不做 | 原因 |
|------|------|
| 全文搜索功能 | MVP 阶段不需要 |
| 多版本对比工具 | change-summary.md 可满足最小对比需求 |
| 移动端适配 | MVP 阶段不需要 |
| 离线下载 | MVP 阶段不需要 |
| 评论/标注功能 | 多角色需求不明确 |
