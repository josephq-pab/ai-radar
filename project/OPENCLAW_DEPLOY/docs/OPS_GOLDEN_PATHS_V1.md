## 二、黄金路径：MT-01 改状态横幅 S0 句

**典型对象：** radar-home M01 / ops-brief M01 / single-chain-ops M01 banner 中的 V7 冻结/第二链路冻结/等待触发句

**风险等级：** 🔴 No-touch Zone

**黄金路径（必须按顺序执行）：**

```
Step GP-01-1：确认 canonical 版本
  → 查 OPS_SENTENCE_REGISTRY_V1.md，找到该 S0 句的 canonical 版本
  → 对照 canonical，确认改动目标是否与 canonical 一致
  → 如 canonical 本身有误，走 CC-xxx 变更流程

Step GP-01-2：列出所有出现位置
  → grep 全站搜索该句，确认所有出现位置（通常 ≥5）
  → 区分：真源层（STAGE2）vs 展示层（各页面）
  → 确认 banner/footer/内联引用全部位置

Step GP-01-3：真源层优先更新
  → 先更新 STAGE2_SINGLE_CHAIN_STATUS.md（真源）
  → 真源层未同步完成，禁止推进展示层

Step GP-01-4：同步所有展示层镜像
  → 按 OPS_MODULE_DEPENDENCY_MAP_V1.md 确认哪些页面有该模块镜像
  → 从 radar-home → ops-brief → single-chain-ops → ops-evidence → ops-decision → ops-glossary → ops-registry → ops-routes 依次同步
  → 每同步一个页面，记录该页面的版本更新

Step GP-01-5：全站一致性验证
  → grep 全站，确认所有位置已同步
  → 确认 banner/footer 与主体内容一致

Step GP-01-6：更新 P3 版本标注
  → 所有被改动页面更新 footer 版本为 P3-45
  → OPS_SENTENCE_REGISTRY_V1.md（如需要）

Step GP-01-7：完整回归（FC-1~FC-7）
  → 完成 OPS_REGRESSION_PLAYBOOK_V1.md 完整回归步骤
  → 白天人工验收（禁止夜间单独执行）
```

**最低回归要求：** FC-1~FC-7 全套 + 白天人工确认

**停止条件：**
- 真源层与展示层未全部同步 → 停止，强制回退
- 改动改变了 canonical 原意 → 立即回退，撤销所有改动
- 改动涉及 ≥5 页但当前是深夜（00:00~06:00）→ 停止，等白天

**适用文档：** OPS_SENTENCE_REGISTRY_V1.md / OPS_MODULE_DEPENDENCY_MAP_V1.md / OPS_REGRESSION_PLAYBOOK_V1.md / OPS_SAFE_EDIT_ZONES_V1.md

## 三、黄金路径：MT-05 改升级说明句（S1，跨 ≥3 页）

**典型对象：** 升级最小5字段 / 先分类再停 / B类值班结果才升级 / 次轮触发规则摘要

**风险等级：** 🟡 Controlled Zone

**黄金路径：**

```
Step GP-05-1：确认 S1 句身份
  → 查 OPS_SENTENCE_REGISTRY_V1.md，确认该句属于 S1（高风险同步句）
  → 确认该句跨多少页（≥3 页联动）

Step GP-05-2：列出所有出现位置
  → grep 全站，确认所有出现位置
  → 记录每页的模块 ID（如 brief-M09 / decision-M07 / ops-M06）

Step GP-05-3：确认统一目标版本
  → 确定"最终改成什么"
  → 确认该版本在全站各页面都适用

Step GP-05-4：批量同步所有页面
  → 按依赖关系从核心页（如 single-chain-ops）向外围页（如 radar-home / ops-decision / ops-routes）依次更新
  → 禁止只改一个页面不同步其他页面

Step GP-05-5：QC-1~QC-3 回归
  → 改后自查：页面加载正常 / 上下文通顺
  → 最近邻引用检查：相关页面 ≤2 个引用已同步
  → 升级判断：≥3 页联动？ → 是：升级到 FC-1~FC-7

Step GP-05-6：版本标注更新
  → 所有被改动页面更新 footer 版本

Step GP-05-7：记录本次改动
  → 在 OPS_SENTENCE_DIFF_MATRIX_V1.md 记录本次统一版本
```

**最低回归要求：** QC-1~QC-3（≥3 页联动升级到 FC-1~FC-7）

**停止条件：**
- 改动涉及 ≥3 页但只有夜间时间 → 停止，等白天
- 只改了一个页面，其他页面未同步 → 停止，补全同步

## 十一、黄金路径通用原则

1. **先查 canonical，再动手** — 任何 S0/S1 句改动，先对照 OPS_SENTENCE_REGISTRY_V1.md
2. **先改真源，再改展示** — 真源层未同步完成，禁止推进展示层
3. **先列影响面，再决定回归深度** — 1~2 页联动 QC / ≥3 页联动 FC
4. **先同步，再验收** — 所有镜像副本同步完成后再做人工验收
5. **版本跟着改动走** — 任何页面内容改动，必须更新 footer 版本标注
6. **深夜不碰 No-touch** — 深夜（00:00~06:00）只做 Safe Zone 改动
7. **停止条件先于改动** — 每一步都先判断是否触发停止条件
