# OPS_CANONICAL_LANGUAGE_RULES_V1.md — 全站 Canonical 语言规则

> 文档版本：v1.0（P3-43）
> 创建时间：2026-04-21
> 对应阶段：Phase 3 — P3-43 全站句子级 canonical registry、段落级比对与残余表达收束功能包第一期
> 目的：建立句子级和段落级的 canonical 语言规范，明确哪些句式必须统一，哪些允许差异

## 一、核心原则

**原则1：S0 句优先保护**
S0 硬约束句（SENTENCE_REGISTRY 中标注为 S0 的句子）不得轻易改动含义。涉及 S0 句的段落，改动前必须全站评估。

**原则2：先改真源，再改展示**
句子的 canonical 版本存储在真源层（STAGE2/OPS_COPY_BASELINE_V1.md）。先确认真源层的句子，再同步展示层。

**原则3：页面角色优先于句式整齐**
不同页面因受众不同，需要不同深度的解释。句式不要求完全一致，但 S0/S1 句的含义必须一致。

**原则4：训练句不得侵入判断句**
训练型语句（如"应该如何"）不应出现在判断型模块中，避免混淆判断依据和训练提示。

**原则5：解释句不得反向改写真源句**
解释型语句（如"为什么冻结"）不能作为判断依据使用。

## 二、主状态句语言规则

### 2.1 状态主线句规范

| 场景 | 标准写法 | 禁止写法 | 原因 |
|------|---------|---------|------|
| 主状态句（完整版） | V7 基线冻结 / 小范围月度试用等待触发 / 第二链路冻结 | V7 已冻结 / 基线已锁定 | "已"字暗示完成，与"冻结"冲突 |
| 快读版（banner/badge） | ⏳ 等待下一触发 / 🔒 第二链路冻结 | 等待中 / 第二链路暂停 | "中"字暗示进行中，"暂停"暗示会恢复 |
| 数字型（KPI卡片） | 可用分析师 X 条 / usable 充足 / 冻结 X 条 | 可用 X 条 / usable X 条 | "可用"与"冻结"必须区分 |

### 2.2 V7 基线句规范

| 场景 | 标准写法 | 禁止写法 | 原因 |
|------|---------|---------|------|
| 冻结状态 | V7 基线冻结（需连续2月正向反馈才解冻） | V7 基线锁定 / V7 已完成 | "锁定"含义模糊，"已完成"与冻结冲突 |
| 对外说明 | 冻结的是"改版动作"，不是"内容使用"。V7 仍可正常使用和发送 | V7 可以用 / V7 已失效 | 冻结≠不可用 |
| 非改稿时机 | 非改稿决策时机，需等连续2月正向反馈 | V7 需要调整 / V7 不适用 | 冻结≠需要调整 |

### 2.3 第二链路句规范

| 场景 | 标准写法 | 禁止写法 | 原因 |
|------|---------|---------|------|
| 冻结原因 | 第二链路冻结（结构性来源缺失，不自动解冻） | 第二链路暂停 / 悬而未决 / 暂时关闭 | 结构性缺失≠暂停/悬而未决 |
| 与第一链路关系 | 第一链路正常运行，第二链路冻结是因为结构性来源缺失 | 第二链路坏了 / 第二链路未接入 | 结构性≠坏了/未接入 |
| 解冻路径 | 解冻唯一路径：用户提供债券货币来源候选（名称+URL+dimension明确+正文可抓+与对公相关） | 有来源就能解冻 / 自己去找来源 | 必须用户提供，非自行发现 |
| 来源候选状态 | 来源候选≠解冻，候选必须经过评估才能用 | 来源候选就是解冻 / 有候选立刻解冻 | 候选≠已确认 |

## 三、生命周期句语言规则

### 3.1 七阶段标准写法

| 阶段 | 标准写法 | 标准代码 | 禁止写成 |
|------|---------|---------|---------|
| 等待态 | 等待态(W)是默认停点和起点 | W | 停止态 / 初始态 / 无状态 |
| 轻响应态 | 轻响应态(L) | L | 等待态 / 记录态 |
| 评估态 | 评估态(A)：边界不清/触发待判/异常待判，尚未准备 | A | 判断态 / 分析态 |
| 准备态 | 准备态(P)：接近下一步条件，但尚未进入 RUN-next | P | 运行态 / 执行态 |
| 升级态 | 升级态(E)：需要人工拍板，不是自动推进 | E | 决策态 / 拍板态 |
| 回复吸收态 | 回复吸收态(R) | R | 回复态 / 反馈态 |
| 回收停点 | 回收停点(S) | S | 结束态 / 终止态 |

### 3.2 流转约束句规范

| 场景 | 标准写法 | 禁止写法 | 原因 |
|------|---------|---------|------|
| 自动流转禁止 | 不允许自动流转：等待→评估→准备→RUN-next 必须由人工确认，V7 基线冻结 | 可以自动推进 / 满足条件就跑 | 违反 V7 冻结原则 |
| 准备≠RUN-next | 准备≠RUN-next已开启，准备只是等确认 | 准备好就触发 / 准备态可以跑 | 准备≠RUN-next |
| 评估≠升级 | 本地能判断就不升级，先在 ops-decision 评估 | 直接升级 / 评估=升级 | 评估≠升级 |
| 升级后停点 | 升级后应停在等待方结论，不继续扩写材料 | 升级后继续推进 / 升级=完成 | 升级≠推进 |

## 四、高频误判澄清语言规则

### 4.1 S0 澄清句（硬约束）

以下澄清句必须保持"≠"句式结构，不得改写成其他形式：

| 标准写法 | 含义 | 禁止改写成 |
|---------|------|---------|
| 轻微信号 ≠ 评估项 | 轻微信号留痕即可，不进评估 | 轻微信号可以不管 / 轻微信号要记录 |
| 边界不清 ≠ 直接升级 | 本地先尝试判断，判断不了才升级 | 边界不清先升级 / 不确定就升级 |
| 准备 ≠ RUN-next 已开启 | 准备只是等确认，RUN-next 仍需人工确认 | 准备好就触发 / 准备态可以跑 |
| 暂不推进 ≠ 否决 | 暂不推进可再提；否决关闭了可能性 | 暂不推进=算了 / 否决=结束 |
| 补材料 ≠ 否决 | 补材料=信息不够，补完可继续 | 补材料=拖延 / 补材料=不行 |
| 来源候选 ≠ 解冻 | 候选必须经过评估才能用，不自动解冻 | 有来源就解冻 / 候选=已确认 |
| 同意准备 ≠ RUN-next 已开启 | 同意准备=进入准备，RUN-next 仍需确认 | 同意准备=可以跑 / 准备=确认 |
| 无信号时不需要深查 | 没有新信号时，停在等待态 | 无信号要继续看 / 没变化也要查 |
| 先观察 ≠ 同意推进 | 暂不推进=维持等待，不是推进 | 先观察=可以动 / 等待=不反对 |
| 冻结 ≠ 没做 | 冻结是主动选择，有依据地暂停 | 冻结=没做 / 冻结=放弃 |
| 阻断 ≠ 放弃 | 有阻断项但整体仍在运营 | 阻断=停摆 / 阻断=不做 |
| 有页面 ≠ 有后台能力 | 只读展示系统，无后台编辑能力 | 有页面=可以改 / 有页面=能操作 |
| 证据 ≠ 判断依据 | 证据页只证明为什么冻结，不证明应解冻 | 证据=可以解冻 / 证据=判断 |
| 评估 ≠ 升级 | 本地能判断就不升级 | 评估=要升级 / 评估完就升级 |

### 4.2 澄清句使用规范

- 澄清句"≠"结构用于训练型/防误用模块，不得出现在判断型模块中
- 澄清句所在段落不得被改写为判断句（如"因为证据≠判断依据，所以不应该解冻"）
- 澄清句用于建立心智，不用于决策

## 五、页面角色句语言规则

### 5.1 页面角色边界句

| 页面 | 角色句标准写法 | 禁止写成 |
|------|-------------|---------|
| radar-home | 本页是只读信息展示页，不是运行按钮页 | 本页可以操作 / 本页是入口 |
| ops-brief | 快报页不是业务判断页，是状态快读 | 快报就是结论 / 快报=判断 |
| single-chain-ops | 运营页是当前是否有动作的主判断页 | 运营页是分析页 / 运营=规则 |
| ops-decision | 决策页不是每次必看页，是边界不清/触发待判指引 | 决策页要先看 / 每次都去决策 |
| ops-playbook | 执行页不是思考页，先判断再执行，不是先想步骤 | 执行页先想步骤 / 执行=判断 |
| config-status | 配置页是配置快照，不是可编辑后台，乱改 analyst_sources.json 无效 | 可以改配置 / 配置=运营参数 |
| ops-evidence | 证据页不是默认入口页，不是判断依据页 | 先看证据 / 证据=依据 |
| ops-glossary | 术语页不要当教材从头读，是查疑工具 | 按顺序学习 / 术语=课程 |
| ops-registry | registry页不是业务页，是真源索引入口 | 从这里开始 / registry=总览 |
| ops-routes | 路径页不是规则原文，是按角色/时间的阅读路径选择工具 | 按路径执行 / 路径=流程 |

### 5.2 页面层级句规范

| 层级 | 标准写法 | 禁止写成 |
|------|---------|---------|
| 核心层 | 核心层（radar-home / ops-brief / single-chain-ops） | 主层 / 第一层 / 必看层 |
| 辅助层 | 辅助层（ops-decision / ops-playbook / ops-evidence / config-status） | 次层 / 第二层 / 可选层 |
| 索引层 | 索引层（ops-glossary / ops-registry / ops-routes） | 第三层 / 工具层 / 辅助层 |

## 六、句式风格规范

### 6.1 允许差异的句式（按角色保留）

| 句式类型 | 允许差异原因 | 示例 |
|---------|------------|------|
| 总览句（首页） | 首页需要全局视角，可稍长 | "已完成 RUN-01...等待下一触发..." |
| 快读句（快报） | 快报需要最简短，可极度压缩 | "⏳ 等待触发" |
| 判断句（运营/决策） | 需要精确条件和依据 | "正常触发：月度窗口到达+用户发出指令" |
| 解释句（证据） | 需要详细说明原因和背景 | "第二链路冻结是因为结构性来源缺失..." |
| 导航句（路径） | 需要按场景给出推荐 | "首次访问：首页→快报→运营" |
| 训练句（各页） | 需要建立正确心智，防误用 | "≠"澄清句系列 |

### 6.2 禁止混用的句式

| 混用类型 | 禁止示例 | 正确做法 |
|---------|---------|---------|
| 训练句混入判断模块 | "你应该…"出现在判断依据区 | 训练句只出现在训练型/P3-40区块 |
| 判断句混入解释模块 | "必须满足X条件"出现在证据说明区 | 判断句只出现在判断型/P3-38区块 |
| 路径句混入状态模块 | "先看A再去B"出现在状态横幅区 | 路径句只出现在路径型/P3-28区块 |
| 留痕句混入训练模块 | "格式：…"出现在训练提示区 | 留痕句只出现在留痕型/P3-33区块 |

### 6.3 句式长度规范

| 场景 | 最大字数 | 原因 |
|------|---------|------|
| Banner/badge标签 | ≤8字 | 视觉空间限制 |
| 卡片快读句 | ≤20字 | 快读需要简洁 |
| 模块内说明句 | ≤50字 | 防止过长 |
| 段落级说明 | 无上限但建议≤100字 | 需要完整说明时 |
| 模板/留痕句 | 无上限 | 需要完整字段时 |

## 七、维护时的语言规则检查步骤

**Step 1：识别句子类型**
```
这个句子是 S0/S1/S2/S3 中的哪类？
查 OPS_SENTENCE_REGISTRY_V1.md 确认
```

**Step 2：检查页面角色**
```
这个句子出现在哪个页面/模块类型？
判断层/解释层/训练层/路径引导层？
是否出现了句式混用？
```

**Step 3：检查禁止写法**
```
这个句子是否使用了禁止句式？
是否出现了≠句式滥用？
是否出现了角色边界句错位？
```

**Step 4：检查允许差异**
```
这个句子是否属于允许差异的类型？
是快读版/解释版/训练版/总览版？
还是应该使用 canonical 版本？
```
