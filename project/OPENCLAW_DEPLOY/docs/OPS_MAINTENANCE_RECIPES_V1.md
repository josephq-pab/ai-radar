## 二、R-01：改 S0 横幅句配方

### 适用范围
改 V7 冻结状态句 / 第二链路冻结句 / 等待触发句 / 状态横幅 banner

### 改前检查
```
[ ] 查 OPS_SENTENCE_REGISTRY_V1.md，确认 canonical 版本
[ ] grep 全站列出该句所有出现位置（≥5）
[ ] 确认涉及真源层（STAGE2）还是只有展示层
[ ] 评估：当前是深夜（00:00~06:00）吗？ → 是：停止，等白天
[ ] 确认改动不改变 canonical 含义
```

### 改动顺序
```
1. 先更新 STAGE2（真源层）
2. 再按 radar-home → ops-brief → single-chain-ops → ops-evidence → ops-decision → ops-glossary → ops-registry → ops-routes 依次同步
3. 每同步一个页面记录该页面版本更新
```

### 改后回归
```
[ ] grep 全站，确认所有位置已同步
[ ] 确认 banner 与主体内容一致
[ ] 完成 FC-1~FC-7 全部步骤
[ ] 白天人工验收（必须有人在实际页面确认）
```

### 停止条件
- 真源层未同步完成就改了展示层 → 停止，回退
- 改动改变了 canonical 含义 → 立即回退，恢复 canonical
- 深夜改动 → 停止，等白天

### 夜间适用
❌ 禁止夜间执行

## 三、R-05：改跨页 S1 句配方

### 适用范围
升级最小5字段 / 先分类再停 / B类值班结果才升级 / 次轮触发规则摘要（≥3 页联动）

### 改前检查
```
[ ] 查 OPS_SENTENCE_REGISTRY_V1.md，确认该句属于 S1
[ ] grep 全站列出该句所有出现位置
[ ] 记录每页的模块 ID
[ ] 确定"最终改成什么"版本
[ ] 评估：≥3 页联动，当前是深夜？ → 是：停止，等白天
```

### 改动顺序
```
1. 从核心页（single-chain-ops）开始改
2. 再改 ops-decision / radar-home / ops-routes 等外围页
3. 禁止只改一个页面不同步其他页面
```

### 改后回归
```

[ ] 改后自查（QC-1）+ 最近邻检查（QC-2）
[ ] ≥3 页联动：升级到 FC-1~FC-7
```

### 停止条件
- 只改了一个页面就停止 → 强制补全同步
- 深夜改动且 ≥3 页联动 → 等白天

### 夜间适用
⚠️ ≥3 页联动不建议夜间

## 十二、通用停止条件（所有配方共用）

以下任一条件触发，立即停止：

| # | 停止条件 | 原因 |
|---|---------|------|
| 1 | 改动涉及 S0 句，但未完成全站同步检查 | S0 未同步是高风险漂移 |
| 2 | 真源层与展示层不同步 | 真源未同步，展示层迟早会乱 |
| 3 | 改动后人工验收发现语义变化 | 语义变化意味着误判风险增加 |
| 4 | 改动影响 ≥5 页，但当前是深夜 | 深夜大范围影响 = 高风险 |
| 5 | 改动涉及 V7 基线/第二链路/RUN-next 冻结相关内容 | No-touch Zone，建议白天确认 |
| 6 | 改动让系统变得更复杂 | 简单改复杂，禁止 |
| 7 | 没有明确改动理由 | 不知道改了解决什么问题，禁止 |
| 8 | 改动收益不明显 | 改了没大用，禁止 |

## 十三、配方案例：升级最小5字段统一（MT-05 示例）

**场景：** 把 ops-brief / single-chain-ops / ops-decision / ops-routes 四页的"升级最小5字段"措辞统一为最新版本

```
改前检查：
  [ ] 查 OPS_SENTENCE_REGISTRY_V1.md，确认 S1 句 canonical 版本
  [ ] grep 四页，确认各自当前版本
  [ ] 确定"最终改成什么"
  [ ] 当前是深夜？ → 是，建议等白天（≥3页联动）

改动顺序：
  [1] 从 single-chain-ops（核心页）开始改
  [2] 再改 ops-decision
  [3] 再改 ops-brief
  [4] 最后改 ops-routes
  [5] 每改一页记录版本更新

改后回归：
  [ ] grep 四页，确认全部已同步为最新版本
  [ ] QC-1（各页加载正常）
  [ ] QC-2（上下文通顺）
  [ ] FC-3 升级：≥3 页联动 → 完整回归
  [ ] 更新 footer 版本为 P3-45

停止条件：
  [ ] 只改了 1~2 页就停止 → 强制补全
  [ ] 某页出现语义变化 → 回退该页
  [ ] 深夜改动 → 补 FC，白天重新验收
```
