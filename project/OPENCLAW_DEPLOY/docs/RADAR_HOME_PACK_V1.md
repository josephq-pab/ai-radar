## 一、本包定位

**本包不是**：
- ❌ 门户系统
- ❌ 后台管理框架
- ❌ 运行操作页面
- ❌ 配置编辑系统

**本包是**：
- ✅ 统一总入口首页（radar-home.html）
- ✅ 四页统一导航一致性
- ✅ 首页说明文档（本文档）
- ✅ 四入口架构：首页 + 配置 + 运营 + 证据

**本包解决什么问题**：
- 运营者不知道"先看哪一页"
- 三入口之间彼此孤立，无统一导航
- 运营者无法一眼看到"当前能做什么/不能做什么"
- 没有默认着陆页，项目首页即配置页

---

## 二、四页分工（统一版）

| 页面 | 文件 | 定位 | 推荐访问场景 |
|------|------|------|------------|
| **首页（总入口）** | radar-home.html | 项目总体状态 + 导航总览 + 能做/不能做 | **首次访问从这里开始** |
| **配置入口** | config-status.html | 来源配置 / Gate状态 / 运行参数 / 规则配置 | 需要查配置时 |
| **运营入口** | single-chain-ops.html | 当前运行状态 / 触发规则 / 不能做 / 下一步 | 需要看运行判断时 |
| **证据入口** | ops-evidence.html | 状态为何成立 / 链路冻结依据 / 触发未触发证据 | 需要看证据链时 |

**推荐访问路径**：首次访问 radar-home.html → 按需进入运营入口或证据入口 → 配置入口备用。

---

## 三、首页核心能力

radar-home.html 提供以下六个能力：

| # | 能力 | 内容 |
|---|------|------|
| 1 | 项目状态总览 | V7冻结 / RUN-01完成 / RUN-next未触发 / 第二链路冻结 |
| 2 | 三入口直接跳转 | 配置入口 / 运营入口 / 证据入口，带描述说明 |
| 3 | 当前能做什么（5项）| 查看状态/结果/触发规则/等待触发/用户提供来源 |
| 4 | 当前不能做什么（7项）| 不能跑RUN-02/不能找来源/不能改基线等 |
| 5 | 最近关键结论 | 结论A / usable池32条 / 曾刚1条 |
| 6 | 下一步动作 | 等待触发 + 触发条件说明 |

## 四、当前准确状态口径

```
V7 基线冻结 / RUN-01 已完成 / 等待下一触发 / 第二链路冻结
```

**首页不是运行按钮页**：radar-home.html 不提供"开始运行"操作能力，是只读信息展示页。

**禁止表述**：
- ❌ "首页可以直接开始运行"（错误，无运行按钮）
- ❌ "四页可以互相替代"（各有分工）
- ❌ "当前可以跑 RUN-02"（未触发）

**正确表述**：
- ✅ "首次访问从首页开始，了解当前状态后按需进入运营/证据/配置入口"
- ✅ "RUN-next 需要命中触发条件，用户发出指令后才能进入"

## 五、四页导航一致性

所有四个页面底部均包含统一底部导航区：

```
首页 · 配置入口 · 运营入口 · 证据入口
```

每个页面顶部 header 均包含四页导航链接。

## 六、当前明确不能做的事项

| 事项 | 为什么 |
|------|--------|
| 不能直接点击"开始运行"按钮 | 本页不提供此按钮，只读展示 |
| 不能在未触发时强行进入 RUN-02 | P3-10/P3-LT 确认未命中 |
| 不能继续找第二链路来源 | 结构性来源缺失 🔒 冻结 |
| 不能修改 V7 基线 | 非改稿时机 🔒 冻结 |
| 不能修改 analyst_sources.json | 配置冻结 |
| 不能修改采集脚本 | 当前能力已够用 |
| 不能把只读入口当配置真源 | 只读快照，不支持写入 |

## 七、触发规则速查

| 类型 | 条件 |
|------|------|
| 正常触发（主）| 下一月度窗口到达 → 用户发出指令 |
| 数据触发（辅）| 曾刚连续断档 / usable<3条 / 集中负向反馈 |
| 用户主动触发 | 用户明确要求提前运行 / 提供债券货币来源候选 |

辅触发 ≠ 自动执行：触发 → agent 评估 → 用户决定。

---

## 八、首页使用指南

**首次访问**：打开 radar-home.html → 了解四个状态 → 按需进入运营/证据入口

**需要查配置时**：从首页点击配置入口，或直接打开 config-status.html

**需要看运行判断时**：从首页点击运营入口

**需要看证据链时**：从首页点击证据入口

**日常访问**：可直接访问运营入口（single-chain-ops.html），首页作为背景状态参考

---

## 九、本包边界

- ✅ 只读：不能编辑任何配置或运行任何操作
- ✅ 静态：不需要动态脚本
- ✅ 最小：只做必要的导航和状态展示
- ✅ 四页导航一致：每个页面都可到达其他页面
- ❌ 不做后台配置系统
- ❌ 不做自动调度
- ❌ 不做运行操作能力
- ❌ 不做门户产品化工程
