# OPS_DRILL_PACK_V1.md — 演练题包

> 文档版本：v1.0（P3-40）
> 创建时间：2026-04-21
> 对应阶段：Phase 3 — 试点运营化 / P3-40 使用者上手训练、误用防护与演练题包功能包第一期
> 目的：通过真实场景输入，把抽象规则套进可操作判断；不做打分系统，不做自动判题

---

## 演练说明

- 每题包含：输入 / 正确归类 / 应看页面 / 正确路径 / 正确停点 / 常见误判
- 题目不是考试，而是"把规则套进真实场景"的练习
- 不做打分系统，不做自动判题，做完只是对照检查

---

## 难度分层

| 层级 | 适用对象 | 特点 |
|------|---------|------|
| 入门（题1~4）| 新接手者 | 单一输入，单一判断，不涉及升级/回复 |
| 值班（题5~8）| 日常运营者 | 多种输入类型，涉及留痕/停点 |
| 升级/回复（题9~12）| 值班+升级处理者 | 涉及拍板/补材料/暂不推进等回复场景 |

## 入门题（题1~4）

### 题目1

**输入**：你是运营者，打开系统后，看到 ops-brief 显示"等待态(W)，无新输入"。

**问题**：你应该继续深查吗？

- **正确归类**：W（等待态，无新信号）
- **应看页面**：ops-brief → single-chain-ops → 停
- **正确路径**：快报无信号 → 运营页确认无新动作 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：深查 ops-evidence / ops-decision（不必要，应该先判断有没有新信号）

---

### 题目2

**输入**：你收到一条消息："某分析师提到同业存款利率上周上升5bp"。

**问题**：你该怎么处理？

- **正确归类**：L（轻响应态，有轻微信号）
- **应看页面**：ops-brief → single-chain-ops → ops-glossary（如需查"轻响应"定义）→ ops-brief 留痕 → 停
- **正确路径**：快报判断类型 → 运营页确认是轻微信号 → 1~2句话留痕 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：直接去 ops-decision 要求评估（轻微信号留痕即可，不进评估）；直接去 ops-evidence 翻证据（不需要）

---

### 题目3

**输入**：你看到 ops-brief 显示"有新输入：某分析师上调某企业评级，理由是Q1营收超预期"。

**问题**：这是轻微信号还是需要评估的信号？

- **正确归类**：A（评估态，有触发条件）
- **应看页面**：ops-brief → single-chain-ops → ops-decision（如边界不清）→ ops-evidence（如需依据）→ 停
- **正确路径**：快报判断类型 → 运营页确认有触发条件 → 决策页（如需要本地判断）→ 或直接升级
- **正确停点**：A（评估态）或 E（升级态）
- **常见误判**：当成轻微信号忽略（上调评级通常构成触发条件）；直接去 ops-playbook（还没到执行阶段）

---

### 题目4

**输入**：你收到一条消息："某分析师提到某城投债可能要提评级"。

**问题**：你应该怎么处理？

- **正确归类**：L（轻响应态，有信号但不构成触发条件）
- **应看页面**：ops-brief → single-chain-ops → ops-glossary（查触发条件定义）→ ops-brief 留痕 → 停
- **正确路径**：快报判断类型 → 运营页判断"可能要提"≠"确定提" → 轻微信号留痕 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：当成评估项（"可能要提"不构成触发条件，不需要进评估）；直接去 ops-decision（不需要）

---

## 值班题（题5~8）

### 题目5

**输入**：你在值班时，收到运营者转来的消息："某分析师更新了某企业信用评级展望为正面"。

**问题**：这是新输入吗？应该留痕还是进评估？

- **正确归类**：A（评估态，评级展望正面构成触发条件）
- **应看页面**：ops-brief → single-chain-ops → ops-decision（判断进升级还是本地评估）→ ops-evidence（如需依据）
- **正确路径**：快报判断 → 运营页判断触发条件（评级展望正面=升级信号）→ 决策页判断（是否需要升级）→ 进 E（升级态）
- **正确停点**：E（升级态）
- **常见误判**：当成轻微信号留痕（评级展望变化通常构成触发条件）；不去 single-chain-ops 直接在 ops-brief 判断

---

### 题目6

**输入**：你收到一句新消息，但没有新信号（分析师常规月度报告，无新观点）。

**问题**：你应该继续做什么？

- **正确归类**：W（等待态，无新信号）
- **应看页面**：ops-brief → single-chain-ops → ops-brief 确认无变化 → 停
- **正确路径**：快报判断 → 运营页确认无新动作 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：去 ops-decision（没有触发条件不需要）；去翻 ops-evidence（不需要）；去翻 config-status（不需要）

---

### 题目7

**输入**：你收到一句新消息："某分析师提到债券市场情绪好转"。

**问题**：这是留痕还是进评估？

- **正确归类**：L（轻响应态，泛泛的市场情绪描述不构成具体触发条件）
- **应看页面**：ops-brief → single-chain-ops → ops-glossary（查触发条件定义）→ ops-brief 留痕 → 停
- **正确路径**：快报判断类型 → 运营页判断"情绪好转"过于泛泛 → 留痕 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：当成评估项（"情绪好转"没有具体触发条件）；去 ops-decision（不需要）

---

### 题目8

**输入**：你在 ops-brief 看到当前处于"评估态(A)"。

**问题**：你下一步应该做什么？

- **正确归类**：A（评估态）
- **应看页面**：single-chain-ops → ops-decision（如边界不清）→ ops-evidence（如需补材料）→ ops-decision（最终判断）
- **正确路径**：运营页确认当前在评估态 → 判断本地能否决策 → 如边界不清进决策页 → 最终判断为 W（可本地决策）或 E（需升级）
- **正确停点**：W 或 E
- **常见误判**：继续翻 ops-brief（快报只是状态展示，不是判断工具）；直接去 ops-playbook（还没到执行阶段）

## 升级/回复题（题9~12）

### 题目9

**输入**：你发出了升级申请，收到回复："暂不推进，维持等待态(W)"。

**问题**：下一步该做什么？

- **正确归类**：R（回复吸收态，收到"暂不推进"）
- **应看页面**：ops-decision（处理回复）→ single-chain-ops（确认停点）→ ops-brief（留痕更新状态）→ 停
- **正确路径**：收到回复 → 判断类型（暂不推进=维持W）→ 回写状态 → 停在等待态(W)
- **正确停点**：W（等待态）
- **常见误判**：当成否定放弃（暂不推进≠否定，条件变可再提）；去翻 ops-evidence（不需要）

---

### 题目10

**输入**：你发出了升级申请，收到回复："补材料：需要具体说明营收超预期的幅度和同业对比"。

**问题**：这是被拒绝了吗？你应该做什么？

- **正确归类**：R（回复吸收态，收到"补材料"）
- **应看页面**：ops-decision（处理回复）→ ops-evidence（如需补材料）→ ops-decision（重新发起评估）→ ops-brief（留痕）
- **正确路径**：收到回复 → 判断类型（补材料≠否决）→ 补充具体依据 → 重新发起评估(A) → 最终判断
- **正确停点**：A（评估态，等待最终判断）
- **常见误判**：当成否决放弃（补材料=信息不够，补完可继续）；不补材料直接停止（错过了本可以推进的输入）

---

### 题目11

**输入**：你发出了升级申请，收到回复："同意进入准备态(P)"。

**问题**：RUN-next 是否已经自动启动了？

- **正确归类**：P（准备态，收到"同意准备"）
- **应看页面**：ops-decision（处理回复）→ single-chain-ops（确认停点为P）→ ops-playbook（如需执行步骤）→ 停（等待人工确认RUN-next）
- **正确路径**：收到回复 → 判断类型（同意准备=进入P）→ 回写状态为 P（准备态）→ RUN-next 仍需人工确认（V7 基线冻结）
- **正确停点**：P（准备态）—— RUN-next 仍需人工确认，不是自动触发
- **常见误判**：以为 RUN-next 已自动开启（准备态P ≠ RUN-next 已触发）；直接去翻 ops-playbook（可以开始看，但 RUN-next 仍未开）

---

### 题目12

**输入**：你收到一条新消息，涉及某分析师的债券来源候选。

**问题**：你能直接用这个候选来源作为判断依据吗？

- **正确归类**：W（等待态，来源候选≠可用）
- **应看页面**：ops-registry（查真源归属）→ single-chain-ops（判断候选不能直接用）→ ops-decision（如需正式评估）→ 留痕"候选待评估"
- **正确路径**：收到候选 → 判断这是结构性输入候选 → 不能直接用 → 留痕"候选待评估" → 停在等待态(W)；第二链路解冻需人工评估候选
- **正确停点**：W（等待态）
- **常见误判**：直接当成可用来源（候选≠可用，违反口径一致性）；以为第二链路会自动解冻（第二链路冻结中）

## 演练后最该记住的10条规则

1. **没有新信号时，不要深查辅助页**（等待态W是默认停点）
2. **轻微信号留痕即可，不进评估**（1~2句话，不需要整套评估）
3. **快报（ops-brief）是快读工具，不是规则学习材料**（1~2句状态，不是完整分析）
4. **运营页（single-chain-ops）的核心问题是"现在要不要动"，不是"有什么"**（判断动作，不是清单）
5. **决策页（ops-decision）只在边界不清/触发待判/异常待判时才来**（不是默认入口）
6. **证据页（ops-evidence）只在解释/补材料/成立依据时才进**（不是默认入口）
7. **准备态(P) ≠ RUN-next 已触发**（RUN-next 仍需人工确认，V7 基线冻结）
8. **补材料 ≠ 否决**（信息不够，补完可继续）
9. **暂不推进 ≠ 否决**（维持等待态，条件变可再提）
10. **来源候选 ≠ 解冻**（候选必须经过评估才能用，第二链路冻结中）
