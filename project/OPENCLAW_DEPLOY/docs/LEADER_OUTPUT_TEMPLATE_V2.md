## 二、V2 四段结构（与V1完全相同）

```
━━━ 【报告标题】━━━━━━━━━━━━━━━━━━
月度同业存款/贷款分析
（数据截至 XXXX年XX月）

━━━ 【一、本月最重要变化】（≤3条）━━━━━━━━━━
[四字判断 + 支撑数据 + 一句话意义]
[格式：判断句（鲜明）+ 数据佐证 + 背景补充]

示例：
1. 【中信存款攻势超预期】
    中信银行当月新增存款2401亿，增量居样本行第一，超出预期。
    [背景：两会后项目融资落地，支撑其对公存款增长]

2. 【华夏存款增速领先但待验证】
    华夏银行年增+6.3%，增速全行业最高。
    [但样本期仅3个月，尚无法判断是否趋势性变化，见下方注]

3. 【平安外币存款流入加快】
    平安银行外币存款月增+121亿，年增+19.7%，外币资金流入提速。

━━━ 【二、原因判断与对我行意义】（合并段）━━━━━━━━━
[机制推断 + 置信度标注 + 对我行业务含义；三段式]

针对变化1（中信攻势）：
    机制：两会后稳投资政策落地，对公结算存款同步增加
    置信度：【中】逻辑通顺，但缺中信贷款投向直接数据
    对我行意义：中信对公存款攻势若持续，可能分流我行重点客户存款
    ←注：若我行在中信有重点客户，需排查是否有挖转迹象

针对变化2（华夏提速）：
    机制：定期储蓄产品利率竞争力较强，叠加网点扩张
    置信度：【低】样本期仅3个月，尚无法判断是趋势还是季节性波动
    对我行意义：华夏存款定价策略持续性待观察，下月增速是否回落是关键信号
    ←注：本判断置信度低，已降调处理，结论待验证

针对变化3（平安外币流入）：
    机制：美联储降息预期下，客户锁汇意愿下降，部分资金从理财回流本币
    置信度：【低】样本期短，需持续跟踪
    对我行意义：外币存款的分流效应可能同步影响我行外币业务，需关注
    ←注：招行贷款利率已降至2.54%，若以贷促存，可能同步带动其存款争夺

━━━━━━━ 【三、三条建议】（行动段）━━━━━━━━━━
[结论鲜明，可立即安排]

1. 【立即排查中信挖转风险】
    目的：确认中信存款增量中是否有来自我行的转移存款
    建议责任人：分行客户管理部门
    验证指标：本月内存量重点客户（大额存单到期客户）中，是否有单笔转投金额≥1亿元

2. 【监测华夏定价策略走向】
    目的：判断华夏存款增长是价格驱动（不可持续）还是战略扩张（可持续）
    建议责任人：企金部门（注：企金部/战略研究部均可承接，视各行组织架构定）
    验证指标：下月华夏存款增速是否回落至3%以内；若持续>5%，则判断为战略驱动

3. 【启动招行贷存款联动预警】
    目的：招行贷款利率2.54%已是行业最低，若以贷促存对我行优质客户有效，需提前预警
    建议责任人：分行公司业务部门
    验证指标：我行优质客户（年结算量>1亿）近30天内是否有转招行贷款超过2户

━━━━━━━ 【四、两个观察项 ━━━━━━━━━━━━
[不是结论，是"睁大眼睛看着"的信号；
 与"三"（主动作）的区别：主动作是"本月要推动的事"，观察项是"要盯着但本月不推动的事"]

▷ 观察项A：外币存款流入是否持续
    观察什么：平安银行外币存款月增量（是否持续>100亿）
    频率：每月
    预警：若连续2个月>100亿且我行外币存款流失>10%，升级为主动作

▷ 观察项B：定期储蓄存款占比是否加速上升
    观察什么：各家银行定期存款占总存款的比例变化趋势
    频率：每月
    预警：样本行中3家以上月增占比>2个百分点，升级为主动作

━━━━━━━━━━━━━━━━━━━━━━━━

注（降调内嵌）：
【华夏银行增速】的判断置信度为"低"，证据样本期仅3个月，无法确认趋势性。
正式行文中表述为"年初提速值得关注，下月数据将是关键验证点"，不写确定性结论。
```

## 三、领导版 V2 与执行版 V2 的关系

| 维度 | 领导版V2 | 执行版V2 |
|------|---------|---------|
| 阅读对象 | 行领导、部门负责人 | 执行层（客户经理/分行公司部）|
| 行动指令 | 方向性建议（"立即排查"）| 具体动作+验证指标+完成时点 |
| 篇幅 | ≤1300字 | ≤800字 |
| 研究准备 | 建议方向（"监测华夏"）| 完整闭环（研究目的+责任人+产出+接收方+转动作条件）|

## 四、V2 确认的要点（来自Q3/Q4/Q5反馈）

1. **一页书面判断报告篇幅可行**：接收方（分行公司部负责人）愿意花2~3分钟看一页书面判断 → 领导版"三条建议"的写法保持不变

2. **部门+角色粒度合适**：执行层认为粒度够具体，能落到人 → 执行版的部门+角色写法V2不变

3. **三档转动作条件有必要**：执行层认为能避免结论悬而未决 → 执行版三档条件V2保留
