## 三、双门槛结构说明（V3 确认）

### 3.1 为什么保留双门槛

Q2冲突分析：
- FB-02明确认为5000万标准合理，不需要调整
- FB-01认为应降低到1000万
- **双门槛结构（1000万快速扫描+5000万重点排查）恰恰是两个分歧的折中方案**

V3处理：双门槛结构获多源支持，升级为"当前推荐表达"；具体数值标注弹性区间。

### 3.2 V3双门槛格式

```
验证指标：
    重点排查：联系5000万以上存款到期客户，确认续做意向
    快速扫描：排查1000万以上存款客户有无异常信号（重点是有无异常，不必全部联系）
    预警阈值：[超出多少需要立即上报]

【注】：门槛为建议参考值，各行可按实际存量客户数量和执行承载力调整。
```

### 3.3 门槛弹性反馈机制

| 执行层反馈 | V4调整方向 |
|-----------|----------|
| 1000万扫描范围过大，工作量过高 | 收紧为"重点关注有异常信号的客户" |
| 5000万重点排查漏掉2000~5000万区间的风险 | 评估后考虑扩大至3000万 |
| 5000万标准合理，保持不变 | 维持V3表达 |

## 四、报告打头结论鲜明要求（V3 新增）

### 4.1 Q3冲突的V4处理

Q3冲突：
- FB-01：A（会看，愿意花2~3分钟）
- FB-02：B（不一定，内容特别关键时才看）

**结论**："一页书面判断报告"的格式本身没被否定，但FB-02指出了一个合理要求——"内容特别关键时才看"意味着**打头第一条结论必须足够关键**。

### 4.2 打头结论鲜明要求

在ACTION_EXPRESSION_RULES_V4中新增一条：
- 研究准备产出格式："一页书面判断报告（3~5个要点）"，**打头第一条结论必须是最核心的判断，不能是背景铺垫**

### 4.3 典型错误

```
❌ 打头结论：背景—两会后项目融资逐步落地...
✅ 打头结论：判断—中信银行当月新增存款2401亿，攻势超预期
```
