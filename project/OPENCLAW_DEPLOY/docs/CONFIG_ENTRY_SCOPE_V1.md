## 四、第一期配置入口目标

**不是**：
- ❌ 配置编辑器
- ❌ 配置后台
- ❌ 数据库
- ❌ 权限系统

**是**：
- ✅ 单页配置状态总览
- ✅ 只读展示（可刷新更新）
- ✅ 每项配置包含：当前值 / 来源文件 / 生效说明
- ✅ 页面直接在本地打开（纯 HTML + JavaScript，读取本地 JSON）

---

## 五、每项纳入配置的展示字段规范

每项配置在状态页中必须展示：

| 字段 | 说明 | 示例 |
|------|------|------|
| 配置名称 | 简洁名称 | top-k 默认值 |
| 当前值 | 当前生效的值 | 5 |
| 来源文件 | 配置值的来源路径 | build_analyst_opinions.py:429 |
| 生效说明 | 值的含义和使用场景 | 最多进入周报的观点数量 |
| 最近变更 | 变更时间和变更内容（若有）| 2026-04-08，从3改为5 |

**不要求**：配置历史全量（第一期只要求有记录即可，不要求完整的版本对比）

---

## 六、第一期边界约束

**可以做**：
- 纯 HTML + JavaScript 读取 JSON 文件
- 单页展示所有关键配置
- 手动刷新更新
- 配置变更时更新配置状态页的变更记录

**不可以做**：
- 在页面里直接编辑配置
- 后端/API/数据库
- 自动化告警
- 多环境配置分离
