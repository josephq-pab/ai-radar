## 二、模板 T-HOME：首页模板

**代表页面**：radar-home.html

**页面角色**：任务触发入口。不是功能导航集，而是"有没有事"的判断首页。

### 2.1 必备 Blocks（固定顺序）

```
[1] 首屏判定区（T-HOME 专用四问）
    → 现在有没有事
    → 值不值得继续读
    → 当前最重要的结论是什么
    → 下一眼该看哪里
[2] 主任务入口区（触发条件展示）
[3] Footer Navigation
[4] P4-6 治理折叠区
```

### 2.2 可选 Blocks
- 无

### 2.3 模板约束
- 首屏判定区与主任务入口紧邻，不得被其他区块隔开
- 不承载独立操作能力
- 不做成工具箱或仪表盘

### 2.4 与其他模板的差异
- 无主动作/路径区（主链路页有）
- 无停止条件区（主链路/决策页有）
- 无"离开输出"语法（支援页有）

---

## 三、模板 T-MAINCHAIN：主链路页模板

**代表页面**：single-chain-ops.html

**页面角色**：主链路运营入口，是值班人员进入后的主要工作页面。

### 3.1 必备 Blocks（固定顺序）

```
[1] 首屏判定区
    → 现在有没有事（等待态判断）
    → 值不值得继续读
    → 当前最重要的结论是什么（当前阶段）
    → 下一眼该看哪里（配置/证据/决策/流程）
[2] 当前阶段状态区（L1 主结论）
[3] 配置/Gate 确认入口（L2 主动作）
[4] 证据入口（L3 必要依据）
[5] 决策入口（L2 主动作）
[6] 流程入口（L2 主动作）
[7] Footer Navigation
[8] P4-6 治理折叠区
```

### 3.2 可选 Blocks
- 例外情形说明区（L3/L4）
- 等待态正常确认区（L2）

### 3.3 模板约束
- 首屏判定区必须包含"当前阶段 + 是否卡住 + 是否需要继续"
- 当前阶段状态必须一眼可判（L1 默认展开）
- Footer 离开语法：`等待态正常 → 值班结果已给出 → 即可停止`

---

## 四、模板 T-DECISION：决策页模板

**代表页面**：ops-decision.html

**页面角色**：场景决策支持。不做执行，只给判断结论（执行/等待/升级）。

### 4.1 必备 Blocks（固定顺序）

```
[1] 首屏判定区
    → 现在有没有事（决策必要性）
    → 值不值得继续读
    → 当前最重要的结论是什么（当前倾向）
    → 下一眼该看哪里
[2] 场景判断区（L1 主结论：执行/等待/升级）
[3] 决策说明区（L3 必要依据）
[4] 停止条件区
[5] Footer Navigation
[6] P4-6 治理折叠区
```

### 4.2 模板约束
- 首屏判定区必须前置"当前倾向：执行/等待/升级"
- 决策结论（L1）不得被证据说明稀释
- Footer 离开语法：`判断已完成 → [执行/等待/升级] 结论已给出 → 即可停止`

---

## 五、模板 T-PLAYBOOK：执行页模板

**代表页面**：ops-playbook.html

**页面角色**：执行流程指引。不是监控页，是操作步骤清单。

### 5.1 必备 Blocks（固定顺序）

```
[1] 首屏判定区
    → 现在有没有事（是否需要立即执行）
    → 值不值得继续读
    → 当前最重要的结论是什么（当前应执行步骤）
    → 下一眼该看哪里
[2] 当前应执行步骤（L1 主结论）
[3] 执行步骤序列（L2 主动作）
[4] 非当前步骤折叠区（L4 参考细节）
[5] 停止条件区
[6] Footer Navigation
[7] P4-6 治理折叠区
```

### 5.2 模板约束
- 当前应执行步骤默认展开
- 非当前步骤默认折叠
- Footer 离开语法：`步骤执行完成 → 即可停止`

---

## 六、模板 T-EVIDENCE + T-BRIEF：证据/快报页模板

**代表页面**：ops-evidence.html（T-E），ops-brief.html（T-B）

**页面角色**：
- T-E：证据核查（依据成立/不成立/待补充）
- T-B：快查快报（等待态正常/异常）

### 6.1 T-EVIDENCE 必备 Blocks

```
[1] 首屏判定区（当前倾向：依据成立/不成立/待补充）
[2] 冻结依据区（L1 主结论）
[3] 证据详情区（L3/L4）
[4] 停止条件区
[5] Footer Navigation
[6] P4-6 治理折叠区
```

### 6.2 T-BRIEF 必备 Blocks

```
[1] 首屏判定区（快报够用/不够用）
[2] 快报核心结论（L1 主结论）
[3] 补充说明（L3/L4）
[4] 快报不够用时的跳转指引
[5] Footer Navigation
[6] P4-6 治理折叠区
```

### 6.3 共同约束
- 首屏必须前置主结论
- 证据/说明细节不得与主结论同层竞争
- Footer 离开语法：T-E 为"即可停止"，T-B 为"读完即可停止"

## 七、模板 T-SUPPORT：支援页模板

**代表页面**：ops-routes.html、ops-glossary.html、ops-registry.html、config-status.html

**页面角色**：支援工具页，不是独立阅读中心。找到答案即返回。

### 7.1 必备 Blocks（统一顺序）

```
[1] 首屏判定区
    → 现在有没有事（是否需要在此停留）
    → 值不值得继续读
    → 当前最重要的结论是什么
    → 下一眼该看哪里（回哪个源页面）
[2] 查询/导航结果区（L1 主结论）
[3] 支援定位说明区（支援化表达）
[4] Footer Navigation（可选，支援页可省略）
[5] P4-6 治理折叠区
```

### 7.2 支援化表达标准

| 支援页 | 标准停止语 |
|--------|---------|
| ops-routes | 选路完成即可返回 |
| ops-glossary | 查到定义即可返回 |
| ops-registry | 查到真源即可返回 |
| config-status | 核对配置即可停止 |

### 7.3 模板约束
- 支援化表达必须位于页面明显位置，不得埋在 details 内
- 不得承载延伸任务（不得变成帮助中心/文档库）
- 不得出现"建议通读"等停留诱导语
