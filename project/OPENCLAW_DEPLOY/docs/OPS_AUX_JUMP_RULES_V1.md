## 背景

P3-27 已完成核心三页默认闭环（首页→快报→运营），并定义了停止条件和例外条件。

但当用户离开核心三页后，仍存在问题：
- 用户不知道应该先跳哪个辅助页
- 容易把4个辅助页都打开
- 没有"一个问题 → 一个最优辅助页"的单跳规则

本功能包定义四个辅助页的问题归口，建立单跳优先原则。

---

## 四个辅助页各自回答什么问题

| 辅助页 | 最先承接的问题 | 不需要先来的问题 |
|--------|-------------|---------------|
| ops-evidence.html（证据页）| "为什么这个状态成立？"、"为什么第二链路冻结？"、"依据是什么？" | "要不要触发？"、"要开始执行？"、"配置是什么？" |
| ops-decision.html（决策助手）| "这是不是异常？"、"要不要触发？"、"现在该不该拍板？" | "为什么状态是这样？"、"步骤是什么？"、"配置是什么？" |
| ops-playbook.html（执行流程）| "如果要执行，步骤是什么？"、"先做哪一步？" | "要不要触发？"、"为什么是这样？"、"配置是什么？" |
| config-status.html（配置入口）| "当前配置/Gate/参数是什么？"、"配置真源一致吗？" | "要不要触发？"、"为什么是这样？" |

## 单跳优先原则

**核心原则：一问题 → 一最优辅助页 → 非必要不连续多跳**

离开核心三页后的标准路径：
```
核心三页闭环（首页→快报→运营）
  ↓ 出现具体问题
单跳一个最优辅助页 → 找到答案 → 停止
  ↓（仅在必要时）
二跳（如果有新的独立问题）
```

---

## 单跳规则表

| 离开核心闭环后的问题 | 单跳最优辅助页 | 原因 |
|------------------|-------------|------|
| 想理解"为什么这个状态成立" | ops-evidence.html（证据页）| 冻结依据/结论来源 |
| 想理解"为什么第二链路冻结" | ops-evidence.html（证据页）| 结构性来源缺失说明 |
| 想理解"为什么现在还是等待态" | ops-evidence.html（证据页）| 等待条件分析 |
| 不知道"这是不是异常" | ops-decision.html（决策助手）| 场景化判断指引 |
| 不知道"要不要触发 RUN-next" | ops-decision.html（决策助手）| 触发条件判断 |
| 不知道"现在该不该拍板" | ops-decision.html（决策助手）| 决策时机指引 |
| 触发已命中，要开始执行 | ops-playbook.html（执行流程）| 8步操作说明 |
| 想确认执行步骤和顺序 | ops-playbook.html（执行流程）| 步骤分解 |
| 想核对"当前配置/Gate/来源数" | config-status.html（配置入口）| 参数快照 |
| 想核对"配置真源是否一致" | config-status.html（配置入口）| 配置对比 |

## 允许二跳的条件

以下情况允许在第一跳之后再跳第二跳（但仍应尽量避免）：

| 第一跳 | 可能需要的第二跳 | 条件 |
|-------|--------------|------|
| 证据页（理解依据后）| 决策助手 | 理解了"为什么"之后需要判断"怎么办" |
| 决策助手（做出判断后）| 执行流程 | 判断"要执行"之后才去执行页 |
| 执行流程（执行完成后）| 证据页 | 想回溯执行依据时（可选）|

**不建议三跳以上**：连续跳多个辅助页说明问题还没想清楚，应该先回到核心三页重新梳理。

---

## 不建议的深查方式

| 错误做法 | 正确做法 |
|---------|---------|
| 离开核心闭环后，同时打开4个辅助页 | 先想清楚具体问题，再单跳一个 |
| 先跳证据页，再跳决策页，再跳执行页，再跳配置页 | 一个问题走完一跳，不连续多跳 |
| 不确定该跳哪个，先去配置页看看 | 先判断问题类型，再决定跳哪个 |
| 想核对配置，先去看证据页有没有 | 核对配置直接去 config-status.html |

---

## 当前推荐的最小深查路径

```
第一步：核心三页闭环（首页→快报→运营）
  ↓
第二步（仅在有具体问题时）：
  · "为什么"类问题 → 证据页（1跳）
  · "要不要/该不该"类问题 → 决策助手（1跳）
  · "怎么执行"类问题 → 执行流程（1跳）
  · "配置是什么"类问题 → 配置入口（1跳）
  ↓
第三步（仅在必要时）：二跳
  ↓
找到答案即停止，不连续多跳
```

---

## 当前不应误解的事项

| 误解 | 正解 |
|------|------|
| 离开核心闭环后，应该把辅助页都看一遍 | 否，带着具体问题单跳一个，找到答案即停止 |
| 不确定该跳哪个时，先跳证据页 | 否，先判断问题类型再决定跳哪个 |
| 跳完一个辅助页后必须再跳另一个 | 否，找到答案即可停止，不强制多跳 |
| 四个辅助页没有优先级/先后顺序 | 否，有明确的问题归口和问题类型对应关系 |
| 想核对配置应该先去证据页看依据 | 否，核对配置直接去 config-status.html |
