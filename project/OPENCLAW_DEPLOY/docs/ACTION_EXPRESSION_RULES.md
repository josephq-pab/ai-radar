# ACTION_EXPRESSION_RULES.md — 建议表达规则：让建议更有部署感

> 文档版本：v1.0
> 编制日期：2026-04-09
> 对应阶段：P2-1（业务问题框架重建 + 输出模板升级实施包）
> 用途：明确什么叫"建议可部署"，区分主动作/观察项/待核实项，给出标准表达格式

---

## 一、核心概念：什么叫"建议可部署"

**可部署的定义**：
一条建议，只有同时满足以下三个条件，才算"可部署"：
1. **动作明确**：读者知道要做什么（不是"加强关注"，而是"本周完成XX客户的XX动作"）
2. **责任到位**：读者知道由谁来做（不是"建议加强"，而是"建议XX部门/团队负责"）
3. **可验证**：读者知道怎么做算成功了（不是"持续关注"，而是"当XX指标超过X%时预警"）

**不可部署的典型（"泛建议"）**：
- "密切关注存款利率走势" → 怎么关注？看哪个数？关注到什么时候？
- "加强客户维护" → 哪些客户？维护什么？做到什么程度？
- "做好存款保险制度宣传工作" → 宣传什么？通过什么渠道？谁来做？

## 二、三类建议的明确区分

### 主动作（Active Actions）

**定义**：本月必须有人推动的事情，结果可追踪，失败有责任

**标志词**：推进/落实/完成/监测/启动/开展/确认/核实/摸底

**格式要求**：
```
动作名称：具体做什么（动宾结构）
目的：为什么做（防范什么/抓住什么）
建议责任人：部门/团队级别
验证指标：
  - 直接指标：[可量化的数字]
  - 间接指标：[可观察的行为]
  - 预警阈值：[超过多少需要上报]
观察频率：[每周/每两周/每月]
```

**禁止写法**：
- ❌ "持续关注"（改成"每周监测一次，持续至X月"）
- ❌ "加强"（改成具体行动）
- ❌ "建议做好"（改成"建议XX部门本周完成XX"）

---

### 观察项（Observations）

**定义**：值得监测但本月不需采取行动，信号出现后才升级

**标志词**：关注/监测/跟踪/留意

**格式要求**：
```
观察信号名称：[出现什么情况需要关注]
观察内容：具体盯哪个指标/行为/事件
观察频率：每周/每两周/每月
升级信号：[出现什么情况需要从观察升级为主动作或上报]
```

**禁止写法**：
- ❌ "继续观察"（改成"每周五记录一次，持续到X月"）
- ❌ "暂无结论"（改成"需要补充X才能下结论"）
- ❌ 观察项里写主动作

---

### 待核实项（Evidenced-Weak Claims）

**定义**：本轮已有初步判断，但证据不足，主动降调处理

**标志词**：信号不一致/样本不足/尚待验证

**格式要求**：
```
原始拟下结论：[本来想写的判断]
降调原因：[证据哪里不足]
需要补充什么：[下月需要什么才能形成结论]
降调用语：[标准降调表达之一（见第四章）]
预计下月能否形成结论：[是/否，原因]
```

**禁止写法**：
- ❌ 直接删除原始结论（降调不等于删除）
- ❌ 把"待核实"写成"已核实但不确定"（矛盾）
- ❌ 待核实项写成主动作（它不是要做的动作，是要补充的信息）

## 三、"泛建议"升级为"动作建议"的对照表

| 泛建议（不可部署） | 动作建议（可部署） | 升级方式 |
|----------------|----------------|---------|
| 密切关注招行存款变化 | 监测招行存款挖转率 | 加具体指标 |
| 加强客户维护 | 对XXX万元以上存款客户做到期前一周主动联系 | 加具体客户 |
| 做好利率宣传工作 | 在手机银行/柜面张贴存款保险宣传材料 | 加渠道+具体物 |
| 关注政策变化 | 跟踪人行/银保监近三月有无调整存款利率上浮区间 | 加时间+机构 |
| 强化结算存款营销 | 对公客户经理每日新增结算存款≥X万可计入KPI | 加数字+计法 |

## 四、标准降调用语

**适用场景一：样本不足**
- 标准表达："目前数据样本不足以形成结论，暂描述为[现象]，下月补充[X]后重新判断"
- 禁止表达："目前结论不确定"（模糊，不解决问题）

**适用场景二：信号不一致**
- 标准表达："当前信号不一致，[A现象]与[B现象]存在矛盾，原因待查，建议下月重点观察[X]"
- 禁止表达："目前无法判断"（不说明为什么不判断）

**适用场景三：时间窗口不足**
- 标准表达："趋势初现但仅[X]个月数据，建议再观察[X]个月再形成结论"
- 禁止表达："趋势仍需验证"（不写要验证多久）

**适用场景四：推断性质强**
- 标准表达："本观点为推断性质，[分析师]提供了[证据]，但缺乏直接数据支撑，建议[降级为观察项/补充X]"
- 禁止表达："该观点仅供参考"（不说明参考什么）

## 五、判断"建议鲜明度"的检查表

写完一条建议后，用以下5问自检：

1. **动作是否具体？** → 不能用"加强/关注/做好"开头
2. **责任是否到部门/团队？** → 不能只写"建议相关部门"
3. **是否有可观察/可量化的验证指标？** → 不能写"持续跟踪"
4. **是否区分了主动作和观察项？** → 主动作用动词，观察项用"关注/跟踪"
5. **是否区分了"要做"和"要补充信息"？** → 待核实项不是主动作

## 六、"存款为主、贷款辅助"在建议中的落地点

**存款主动作**：以客户存款行为、竞品存款利率/产品、负债成本为核心
**贷款辅助建议嵌入规则**：
- 只有当贷款利率变化**直接影响存款策略**时，才写进存款主动作
- 贷款利率单独变化（如某行利率下调但不影响存款）→ 在执行版"一"段一句话提及即可，不进主动作
- 贷款利率变化通过"贷款客户存款留存"间接影响存款 → 纳入主动作，但注明"联动影响"

**示例（贷款辅助在主动作中的正确嵌入）**：
```
▶ 主动作3：摸底贷款客户存款留存情况
    目的：光大银行近期贷款投放利率上调+12bp，若其一般性存款利率同步上调，
          可能引发其存量客户的存款资金重新配置，需提前摸底
    建议责任人：分行客户管理部门
    验证指标：
        - 光大银行同期一般性存款利率是否跟进上调
        - 其对公客户群在我行的存款是否有流失迹象
```

## 七、本轮强边界

- 本文档定义的是"如何写建议"，不定义"谁来执行"
- 不引入任务管理系统，不写成派工单
- 不写具体人名，只写部门/团队级
- 不写 deadline（具体日期），只写"本周/本月/每周"等频率
