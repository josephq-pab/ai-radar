## 一、登记页定位

**登记页是什么：**
- 不是文档中心（不建搜索/索引/标签化文档库）
- 不是后台管理系统
- 不是动态版本面板
- 是：**只读元信息索引入口**（ops-registry.html）

**登记页解决什么问题：**
- 运营者不知道某页是展示页还是依据页
- 运营者不知道页面状态以哪个文档为准
- 运营者不知道冲突时该信哪一个
- 入口越来越多后越来越容易混淆

## 三、页面性质说明

### 展示页（Display）
- 提供信息聚合和导航能力，本身不是最终判断依据
- 典型：首页 / 决策助手 / 执行流程
- 状态冲突时，以对应真源文档为准

### 依据页（Source）
- 展示配置快照或状态快照，是真实业务状态的只读呈现
- 真源是背后的配置文件或文档，页面仅用于方便查看
- 典型：配置页（展示 analyst_sources.json 快照）/ 运营页
- 配置页的真源是 analyst_sources.json，不是 config-status.html 本身

### 索引辅助页（Index）
- 不直接展示业务状态，而是提供元信息和导航指引
- 典型：术语口径页 / 真源登记页
- 不是业务判断依据，是辅助决策工具

## 五、冲突优先级规则

### 状态判断冲突
1. 优先查：STAGE2_SINGLE_CHAIN_STATUS.md
2. 其次查：STAGE2_SINGLE_CHAIN_RUN_CLOSE_01.md
3. 不要以页面展示的状态描述为最终判断

### 触发条件冲突
1. 唯一真源：STAGE2_SINGLE_CHAIN_TRIGGER_RULES_V1.md
2. 运营页/执行流程页的触发说明均为辅助展示
3. 有疑问时，直接查触发规则文档

### 配置口径冲突
1. 配置真源：analyst_sources.json
2. config-status.html是快照展示，不是配置编辑器
3. 配置变更请联系平台侧，当前不开放编辑

### 基线内容冲突
1. 唯一真源：TRIAL_BASELINE_FREEZE_V7.md
2. 证据页的基线说明为辅助展示
3. V7冻结期间，内容以本文档为准
