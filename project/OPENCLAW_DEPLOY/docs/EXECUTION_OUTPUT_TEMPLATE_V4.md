## 三、双门槛结构说明（V4 明确版）

### 3.1 双门槛结构（V4 当前推荐）

| 层级 | 门槛 | 目的 | 执行要求 |
|------|------|------|---------|
| 重点排查 | 5000万以上 | 防挖转 | 必须联系，确认续做意向 |
| 快速扫描 | 1000万以上 | 摸底异常 | 不必全部联系，重点是有无异常信号 |

### 3.2 门槛弹性标注（V4 明确）

```
验证指标：
    重点排查：联系5000万以上存款到期客户，确认续做意向
    快速扫描：排查1000万以上存款客户有无异常信号（不必全部联系）

【注】：5000万/1000万为参考值，各行可按实际存量客户数量和执行承载力上下浮动。
收到执行层反馈后，模板据此调整。
```

## 四、粒度弹性写法（V4 明确版）

### 4.1 粒度选择规则

```
责任粒度：
  建议：部门 + 角色两级（能让动作落到具体执行人）
  弹性：若本行组织架构扁平、可由部门统一部署，可简化为"部门"一级
  [试行·三份反馈存在分歧，各行可根据实际组织架构选择]
```

### 4.2 粒度选择判断

| 场景 | 推荐粒度 |
|------|---------|
| 部门内职责分工明确 | 部门 + 角色两级 |
| 部门内职责分工模糊/扁平 | 部门一级 |
| 跨部门联合动作 | 部门 + 角色（主办+支持）|
