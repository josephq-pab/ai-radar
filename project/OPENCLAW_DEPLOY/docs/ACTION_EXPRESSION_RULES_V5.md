## 一、核心概念：动作闭环三角

每一个动作建议都必须回答三个问题：

1. **谁来做？** → 责任人（部门 + 角色）
2. **做完交给谁？** → 接收方（仅研究准备动作需要）
3. **什么条件下要接着做？** → 转动作条件（仅研究准备动作需要）

## 三、动作类型及闭环要求（V5 完整版）

| 类型 | 定义 | 闭环终点 | 是否需要接收方 | 是否需要转动作条件 |
|------|------|---------|--------------|-----------------|
| **立即执行** | 本月必须有人推动 | 完成后确认验证指标 | ❌ 不需要 | ❌ 不需要 |
| **研究准备** | 本月需要搞清楚一件事，结果给下月提供判断依据 | 接收方看完结论并决定是否触发 | ✅ 需要 | ✅ 需要 |
| **建立机制** | 本月启动但本月完不成 | 机制进入常态运行 | ⚠️ 视情况 | ⚠️ 视情况 |

## 四、研究准备动作的标准格式（V5 完整版）

```
▸ 研究准备：[研究主题]
    研究目的：[搞清楚这个问题是为了支持什么判断]
    研究责任人：企金部门（注：企金部/战略研究部均可承接，视各行实际）[当前推荐·强确认]
    研究内容：
      1. [具体要拿到的证据/数据/结论 1]
      2. [具体要拿到的证据/数据/结论 2]
    产出形式：一页书面判断报告（3~5个要点）【打头第一条结论必须最核心】
    接收方：[部门] / [角色]
    转动作条件：
      若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    预计完成时点：XXXX年XX月XX日
```

## 五、责任归口要求（V5）

### 5.1 粒度要求

```
责任粒度：
  建议：部门 + 角色两级（能让动作落到具体执行人）
  弹性：若本行组织架构扁平、可由部门统一部署，可简化为"部门"一级
  [试行·三份反馈存在分歧，各行可根据实际组织架构选择]
```

### 5.2 企金部门归口（V5 强确认）

```
企金部门（注：企金部/战略研究部均可承接，视各行实际）[当前推荐·强确认]
若本行无战略研究部，可由企金部产品经理兼任研究职能
```

## 六、接收方的写法规则（V5 不变）

接收方 = 能决定"这个结论要不要触发下一步动作"的人，一般是部门负责人或分行长级别。

## 七、转动作条件的写法规则（V5 不变）

每个转动作条件必须包含三要素：触发事实 + 判断门槛 + 对应动作。

## 八、立即执行动作的格式（V5 明确版）

```
▶ 主动作1（本 月 必 做）：[动作名称]
    目的：[做这件事是为了防范什么/抓住什么机会]
    建议责任人：[部门] / [角色]
    验证指标：
        重点排查：联系5000万以上存款到期客户，确认续做意向
        快速扫描：排查1000万以上存款客户有无异常信号（重点是有无异常，不必全部联系）
        预警阈值：[超出多少需要立即上报]
    完成时点：XXXX年XX月XX日

【注】：5000万/1000万为参考值，各行可按实际存量客户数量和执行承载力上下浮动。
```

## 十、报告打头结论鲜明要求（V5 继承）

### 10.1 Q3冲突的V5处理

FB-01（Q3选A：会看）vs FB-02（Q3选B：不一定，关键时才看）

**结论**：
- "一页书面判断报告"的格式本身没被否定
- FB-02的要求合理：**打头第一条结论必须足够关键**
- V5在产出格式要求中保持：**打头第一条结论必须是最核心的判断**

### 10.2 判断标准

```
✅ 正确示范：判断句打头：华夏银行当月新增存款+558亿，增速领先但规模仍小
❌ 错误示范：背景铺垫打头：两会后项目融资逐步落地，对公结算存款同步增加
```

## 十一、"多源确认"表达定义（V5 延续）

### 11.1 标注层级

| 标注 | 含义 |
|------|------|
| [当前推荐·强确认] | 三源及以上一致，当前最稳表达 |
| [多源确认] | 两源及以上一致，可使用 |
| [当前推荐·结构确认] | 结构获多源支持，具体数值待定 |
| [试行·弹性] | 存在分歧，各行可根据实际情况调整 |

### 11.2 多源确认不等于最终定型

- 多源确认 = "已有2+份反馈一致支持，当前最稳的表达"
- 最终定型 = "经过多轮/多机构验证，已固化不再调整"
- V5的"当前推荐"仍是试用期的推荐，不是最终定型

## 十四、V5 检查表（11问）

写完一条动作后自检：

1. 动作类型是否明确？
2. 动作是否具体？
3. 责任是否到部门+角色（或已选择部门一级）？
4. 企金部门归口是否写为"当前推荐"？
5. 是否有可验证指标？
6. 是否区分了"立即执行/研究准备"？
7. 研究准备是否有接收方？
8. 研究准备是否有转动作条件？
9. 双门槛是否加注"参考值，各行可调整"？
10. 报告打头第一条结论是否够核心？
11. 是否有明确的完成时点？

## 十六、V5 试发验证目标（新增）

### 16.1 试发核心问题

| 优先级 | 问题 | 验证方式 |
|--------|------|---------|
| P1 | 值不值得看？ | 试发后问"是否觉得内容值得花时间看" |
| P2 | 看不看得懂？ | 试发后问"主动作是否知道怎么做" |
| P3 | 动作能不能用？ | 试发后问"验证指标是否可操作" |
| P4 | 愿不愿继续收？ | 试发后问"下月是否愿意继续收" |

### 16.2 试发成功标准（最低门槛）

- P1 有至少1人回答"值得看"
- P2 有至少1人回答"知道怎么做"
- P3 有至少1人回答"可操作"
- P4 不作强制要求（FB-01/02/03已选了B）
