## 一、核心概念：动作闭环三角（V6 不变）

每一个动作建议都必须回答三个问题：

1. **谁来做？** → 责任人（部门 + 角色）
2. **做完交给谁？** → 接收方（仅研究准备动作需要）
3. **什么条件下要接着做？** → 转动作条件（仅研究准备动作需要）

## 四、三档转动作条件规则（V6 压缩版）

### 4.1 V6 核心原则

**逻辑完全保留，表达压缩至少2/3。**

三档条件是为了回答"研究结论出来后，下月要怎么做"，是研究准备的闭环出口。不能删除逻辑，但可以用更少的字数表达同样的含义。

### 4.2 V5→V6 压缩对比

**V5 表达（偏重，约5行）**：
```
转动作条件：
    若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    若[触发事实]达到[判断门槛]，则下月启动[对应动作]
```

**V6 表达（压缩版，2行注语）**：
```
转动作条件（注语形式）：
    · 若[触发A]且[触发B] → 下月启动[对应动作]
    · 若[触发C] → 下月归档
    · 其余情况：结论留存，下月补充数据后再判断
```

### 4.3 压缩版写作规则

1. **触发条件和对应动作压缩为一行**："若[X]达到[Y] → 下月启动[Z]"
2. **归档条件单独一行**："若[X] → 下月归档，不再触发"
3. **待定情况一行**："其余情况：结论留存，下月补充数据后再判断"
4. **不使用条件连段落**，用注语符号"·"分行
5. **不写"判断门槛"等抽象词**，直接写具体事实

### 4.4 何时用完整版 vs 压缩版

| 场景 | 推荐表达 |
|------|---------|
| 执行版模板（V6）| **压缩版**（2行注语）|
| 领导版模板（V6）| 不写三档条件（领导版不涉及操作细节）|
| 样稿（V6）| **压缩版** |
| 规则文档（说明用）| 完整版（解释规则时使用）|

## 五、双门槛规则（V6 更新版）

### 5.1 双门槛结构（V6 确认）

| 层级 | 门槛 | 目的 | 执行要求 |
|------|------|------|---------|
| 重点排查 | 参考阈值 | 防挖转 | 必须联系，确认续做意向 |
| 快速扫描 | 参考阈值 | 摸底异常 | 不必全部联系，重点是有无异常信号 |

### 5.2 门槛弹性表达（V6 更新）

**V6 标注**：参考阈值（各行可按存量客户数量和执行承载力上下浮动）[参考值·可调]

**V5→V6 变化**：
- V5："[试行·弹性]"——表达仍偏"暂定"感
- V6："参考阈值"[参考值·可调]——表达更中性，给执行者更大主动调节的空间

**弹性参考值写法示例**：
```
验证指标：
    重点：联系5000万以上存款到期客户，确认续做意向
    扫描：排查1000万以上存款客户有无异常信号（不必全部联系）

【注】：5000万/1000万为参考阈值，各行可按存量客户数量和执行承载力上下浮动。
```

## 九、持续意愿信号处理规则（V6 新增）

### 9.1 Q4 反馈含义解析

Q4: 6/6 = B（愿意再看一版）

| B的含义 | 解读 |
|---------|------|
| 不是 A（强订阅）| 尚未达到"每月必看"状态 |
| 不是 C（不愿意）| 也未被否定，有继续看的意愿 |
| 是中间地带 | "顺手就看，不强求" |

### 9.2 V6 应对原则

1. **不下"已具备黏性"结论**：V6 不宣称"已验证使用黏性"
2. **以"顺手度提升"为目标**：V6 轻量化压缩是手段，目的是让 B 类在下一轮转化为 A
3. **下一轮才是验证时机**：V6 的真正验证是"下一轮 Q4 是否出现 A"

### 9.3 V6 样稿的顺手度设计

- 领导版篇幅：约900字（V5 约1300字）
- 执行版篇幅：约850字（V5 约1100字）
- 三档条件：2行注语（V5 完整段落）
- 动作语气：更直接，少用"制度感"表述

## 十、V6 检查表（11问）

写完一条动作后自检：

1. 动作类型是否明确？
2. 动作是否具体？
3. 责任是否到部门+角色（或已选择部门一级）？
4. 企金部门归口是否写为"[6源一致确认]"？
5. 是否有可验证指标？
6. 是否区分了"立即执行/研究准备"？
7. 研究准备是否有接收方？
8. 研究准备是否有转动作条件（注语形式，2行以内）？
9. 双门槛是否加注"参考阈值，各行可调"？
10. 报告打头第一条结论是否够核心？
11. 是否有明确的完成时点？
