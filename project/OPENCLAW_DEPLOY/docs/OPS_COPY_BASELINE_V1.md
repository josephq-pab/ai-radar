## 二、页面角色标准写法

### 十页角色定义

| 页面 | 角色层级 | 标准角色描述 | 不是什么 |
|------|---------|------------|---------|
| radar-home.html | 核心层-总入口 | 首页总入口 | 不是运行按钮页 / 不是证据页 / 不是规则页 |
| ops-brief.html | 核心层-快报 | 状态快报（面向快读用户） | 不是业务页 / 不是规则页 |
| single-chain-ops.html | 核心层-运营 | 单链路运营入口（当前是否有动作的主判断页）| 不是首页 / 不是证据页 |
| ops-decision.html | 辅助层-决策 | 运营决策助手（边界不清/触发待判/异常待判指引）| 不是留痕页 / 不是回写页 |
| ops-playbook.html | 辅助层-流程 | 运营执行流程（已进入执行/准备时的执行步骤）| 不是决策页 / 不是判断页 |
| config-status.html | 辅助层-配置 | 配置快照（V7 Gate/参数/Gate 核对）| 不是生命周期解释页 |
| ops-evidence.html | 辅助层-证据 | 运营证据与历史入口（解释/补材料/成立依据）| 不是默认入口页 |
| ops-glossary.html | 索引层-术语 | 术语与口径（统一术语定义）| 不是帮助中心 / 不是文档库 |
| ops-registry.html | 索引层-登记 | 版本与真源登记（元信息入口）| 不是业务页 |
| ops-routes.html | 索引层-路径 | 角色路径（按角色/按时间/按生命周期的阅读路径）| 不是业务页 |

## 四、核心概念统一短句

### V7 基线相关

| 概念 | 标准短句 |
|------|---------|
| V7 基线冻结 | V7 基线冻结 |
| 基线状态 | V7 基线冻结 / 小范围月度试用等待触发 |
| 第二链路冻结 | 第二链路冻结（结构性来源候选由人工评估，不自动解冻）|
| RUN-next 冻结 | RUN-next 由人工确认，V7 基线冻结 |

### 核心三页闭环

| 概念 | 标准短句 |
|------|---------|
| 核心三页 | radar-home（首页）+ ops-brief（快报）+ single-chain-ops（运营）|
| 默认闭环 | 核心三页默认闭环（首页→快报→运营→判断停点）|
| 辅助页单跳 | 辅助页单跳（从核心页跳转辅助页，辅助页不驱动核心流程）|
| 辅助页回收闭环 | 辅助页回收闭环（二跳约束：辅助页不跳转其他辅助页）|

### 停点规则

| 概念 | 标准短句 |
|------|---------|
| 默认停点 | 等待态(W)是默认停点 |
| 不再深查 | 没有新信号时，不需要深查辅助页 |
| 判断后停点 | 边界清晰时回到等待态(W)，边界不清时发起升级(E) |

### 新输入响应

| 概念 | 标准短句 |
|------|---------|
| 新输入最小响应 | 收到新输入 → 先归类 → 再走对应路径 → 判断停点 |
| 留痕最小原则 | 只需要 1~2 句话记录口径，不需要整套评估 |

### 升级与回复

| 概念 | 标准短句 |
|------|---------|
| 升级最小原则 | 只说清楚"需要判断什么"，不超过 3 句 |
| 拍板回复处理 | 收到回复 → 先分类（暂不推进/补材料/同意）→ 回写 → 对应停点 |
| 补材料最小原则 | 只需要 1~2 个具体依据，不需要整套证据 |

### 生命周期

| 概念 | 标准短句 |
|------|---------|
| 七阶段 | 等待态(W) / 轻响应态(L) / 评估态(A) / 准备态(P) / 升级态(E) / 回复吸收态(R) / 回收停点(S) |
| 不允许自动流转 | 等待→评估→准备→RUN-next 必须由人工确认，V7 基线冻结 |
| 默认停点 | 等待态(W)是默认停点和起点 |

## 五、高频误判澄清统一短句

| 澄清句（首页标准写法）| 含义 |
|---------------------|------|
| 轻微信号 ≠ 评估项 | 轻微信号留痕即可，不进评估 |
| 边界不清 ≠ 直接升级 | 本地先尝试判断，判断不了才升级 |
| 准备 ≠ RUN-next 已开启 | 准备只是等确认，RUN-next 仍需人工确认 |
| 暂不推进 ≠ 否决 | 暂不推进可再提；否决关闭了可能性 |
| 补材料 ≠ 否决 | 补材料=信息不够，补完可继续 |
| 来源候选 ≠ 解冻 | 来源候选必须经过评估才能用，不自动解冻 |
| 同意准备 ≠ RUN-next 已开启 | 同意准备=进入准备，RUN-next 仍需确认 |
| 无信号时不需要深查 | 没有新信号时，停在等待态，不需要翻辅助页 |
| 先观察 ≠ 同意推进 | 暂不推进=维持等待，不是推进 |

## 六、哪些句式必须统一

| 句式 | 必须统一的写法 | 不允许的写法 |
|------|-------------|-----------|
| V7 基线描述 | V7 基线冻结 / 小范围月度试用等待触发 | V7 已冻结 / 基线已锁定（含义接近但不标准）|
| 第二链路描述 | 第二链路冻结 | 第二链路暂停 / 第二链路不可用（含义不清）|
| RUN-next 描述 | RUN-next 由人工确认，V7 基线冻结 | RUN-next 已开放 / RUN-next 可触发（违反冻结原则）|
| 状态代码 | W/L/A/P/E/R/S | 状态1/状态2（不标准）|
| 页面角色 | 核心层/辅助层/索引层 | 主层/次层/第三层（不标准）|

## 七、哪些内容允许按页保留差异

以下内容允许在不同页面有不同的解释深度，不要求完全一致：

| 内容 | 允许差异原因 |
|------|------------|
| 场景判例的完整说明 | ops-brief 只需要一句话，single-chain-ops 需要完整说明 |
| 生命周期详细流转图 | radar-home 需要全图，ops-brief 只需要一句话位置说明 |
| 升级说明的完整模板 | ops-decision 需要完整模板，ops-brief 只需要一句话提示 |
| 术语的详细定义 | ops-glossary 需要完整定义，其他页只需要引用标准短句 |
