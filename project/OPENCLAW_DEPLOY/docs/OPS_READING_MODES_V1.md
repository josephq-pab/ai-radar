## 一、本包定位

**本包是什么：**
- 不是阅读中心（无内容索引/搜索/标签）
- 不是培训系统（无学习路径/测验/onboarding）
- 不是导览平台（无交互切换/个人化推荐）
- 是：**只读时间路径最短阅读模式模块**（嵌入现有页面）

**本包解决什么问题：**
- 页太多，时间少时不知道先看什么
- 只想知状态却读了太多
- 想判要不要动却绕太远
- 新接手者不知先看全局还是先看术语

## 二、三档时间模式

### 2.1 30秒模式

| 项 | 内容 |
|----|------|
| 目标 | 只知道当前状态和是否要动 |
| 产出 | 一句话状态 + 能/不能做 + 下一步一句话 |
| 先看 | 首页顶部状态横幅 + 【能力与边界看板】✅ 已可用 |
| 看完回答 | 项目停在哪 / 下一步要不要跑 RUN-next |

### 2.2 3分钟模式

| 项 | 内容 |
|----|------|
| 目标 | 判断现在是否应该推进 / 是否需要继续看 |
| 产出 | 状态 + 边界 + 触发条件 + 推荐后续页 |
| 看完路径 | 首页看板 → ops-brief.html 快报 → ops-decision.html 决策 |
| 看完回答 | 冻结是什么 / 阻断是什么 / 我现在该等还是该动 |

### 2.3 10分钟模式

| 项 | 内容 |
|----|------|
| 目标 | 完整理解当前阶段、边界、依据和下一步 |
| 产出 | 状态 + 依据 + 决策 + 执行路径 + 真源 |
| 看完路径 | ops-routes.html 角色路径 → ops-glossary.html 术语 → ops-decision.html 决策 → ops-playbook.html 执行 → ops-registry.html 真源 |
| 看完回答 | 全部依据 / 冻结原因 / 触发条件 / 执行步骤 / 交接时怎么说 |

---

## 三、两种阅读方式的对比

| 维度 | 按角色路径 | 按时间预算 |
|------|-----------|-----------|
| 适合谁 | 已知自己是哪类角色 | 时间有限，想快速定位 |
| 核心问题 | 我不知道看完能回答什么 | 我现在该看什么 / 能回答什么 |
| 推荐起始页 | ops-routes.html 角色路径 | radar-home.html 首页 / ops-brief.html 快报 |
| 结合方式 | 可先用时间预算找到入口，再用角色路径确认完整范围 |

---

## 四、不建议的阅读方式

- ❌ 无目标跳转十页
- ❌ 从第一页依次看到第十页
- ❌ 没有时间预算时把所有页都打开
- ❌ 一次性把十页全部看完
- ❌ 无差别把十页链接扔给对方
- ❌ 让领导从首页逐页看（信息量太大）
- ❌ 让协作者先看执行流程（不需要知道怎么跑）
- ❌ 让接手者自己摸索（应该有明确路径）
