## 背景

P3-29 已完成辅助页回收闭环与二跳约束，定义了跳进辅助页后何时停止/返回/二跳。

但仍存在问题：
- 不知道什么时候已经"看够了"
- 没有新信号却继续深查，想找"更多确定性"
- 觉得"再看一页更安全"而不是真的需要
- 决策页被当作每轮必看页，而不是有具体场景才来

本功能包定义默认停点与不再深查规则。

---

## 默认停点定义

| 停点 | 条件 |
|------|------|
| 首页默认停点 | 大多数用户看完首页即可停止，知道系统当前状态即可 |
| 快报默认停点 | 快报结论是"等待态正常"，且自己无待办，不需要去运营页 |
| 运营页默认停点 | 判断结论是"等待态正常"，且无新触发/异常/用户输入 |
| 辅助页单跳停点 | 问题已在该页得到直接回答，无自然转化为另一类问题 |

---

## 强制停点定义

| 停点 | 条件 |
|------|------|
| 无新数据停点 | 没有新输入时，反复阅读不会改变结论 |
| 无触发信号停点 | 触发条件未命中，不需要做决策判断 |
| 无异常停点 | 没有出现 usable 池异常下滑等信号，不需要决策 |
| 无用户输入停点 | 没有用户明确提出的新问题，不做泛泛浏览 |

---

## 可继续深查的必要条件

满足以下任一条件，才继续深查：

| 条件 | 说明 |
|------|------|
| 有新数据/新触发 | 出现了新的分析师数据或触发信号 |
| 有未完成待办 | 用户自己在运营页有待办未处理 |
| 有用户新输入 | 用户明确提出了新的具体问题 |
| 有异常信号 | usable 池异常下滑 / 收到集中负向反馈 / 曾刚连续断档 |

## 各页面默认停点

### radar-home.html（首页）

| 情形 | 动作 |
|------|------|
| 大多数用户 | ✅ 看完首页即可停止，知道系统状态即可 |
| 需要更多细节 | → 去快报（ops-brief.html）|
| 有具体问题 | → 带着问题单跳辅助页 |

### ops-brief.html（快报）

| 情形 | 动作 |
|------|------|
| 快报结论"等待态正常"，自己无待办 | ✅ 可停止，不需要去运营页 |
| 快报提示需去运营确认 | → 去运营页（single-chain-ops.html）|
| 有具体场景问题 | → 带着问题单跳辅助页 |

### single-chain-ops.html（运营页）

| 情形 | 动作 |
|------|------|
| 判断结论"等待态正常"，无新触发/异常 | ✅ 停止，不需要跳辅助页 |
| 有新触发信号 | → 单跳最合适的辅助页 |
| 自己在运营页有待办 | → 处理完待办再停止 |

### ops-evidence.html（证据页）

| 情形 | 动作 |
|------|------|
| 已回答"为什么"问题，无转化问题 | ✅ 停止或返回运营页 |
| 引出"是否触发"问题 | → 有条件二跳去决策页 |

### ops-decision.html（决策页）

| 情形 | 动作 |
|------|------|
| 无新信号/无触发/无异常 | ✅ 不需要来决策页 |
| 有触发信号/异常/强烈信号 | → 来决策页做具体判断 |

### ops-playbook.html（执行页）

| 情形 | 动作 |
|------|------|
| 执行完毕，问题已回答 | ✅ 停止或返回运营页 |
| 执行前需核参 | → 有条件二跳去配置页 |

### config-status.html（配置页）

| 情形 | 动作 |
|------|------|
| 核对配置完毕 | ✅ 停止或返回运营页 |
| 核参后仍需判断触发 | → 有条件二跳去决策页 |

---

## 不建议继续深查的情形

| 错误做法 | 正确做法 |
|---------|---------|
| 没有新数据，却想再确认一下 | → 停止，没有新输入再看一遍结论不变 |
| 没有触发信号，却预防性翻辅助页 | → 停止，预防性深查不改变等待态结论 |
| 已经得到答案，却觉得"再看一页更安全" | → 停止，惯性浏览不等于必要确认 |
| 决策页被当作每轮必看 | → 决策页不是每轮必看，有具体场景才来 |
| 四个辅助页都打开确认一遍 | → 带着具体问题单跳，不无序深查 |

---

## 当前推荐的默认阅读闭环

```
首页（看完可停）
  ↓
快报（结论等待态正常可停）
  ↓
运营页（判断等待态正常可停）
  ↓
有具体问题 → 单跳最合适的辅助页
  ↓
问题已回答 → 停止或返回运营页
  ↓
非必要不继续多跳
```

---

## 当前不应误解的事项

| 误解 | 正解 |
|------|------|
| 十页都需要全看一遍才算确认 | 否，大多数场景三页核心闭环已足够 |
| 看完一页就应该继续看下一页 | 否，有答案即停止，不强制继续 |
| 决策页是每轮必看的例行页面 | 否，决策页是有具体场景才需要来的页 |
| 没有新数据时再看一遍会更确定 | 否，没有新输入时结论不变 |
| 多跳几个辅助页更保险 | 否，惯性深查不等于必要确认 |
