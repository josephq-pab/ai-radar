## 一、本包定位

**本包不是**：
- ❌ 完整审计平台
- ❌ 后台管理系统
- ❌ 动态历史数据库
- ❌ 自动取数系统

**本包是**：
- ✅ 只读证据与历史入口页（ops-evidence.html）
- ✅ 最小证据包文档（本文档）
- ✅ 三入口架构：config-status.html（配置）+ single-chain-ops.html（运营）+ ops-evidence.html（证据）

**本包解决什么问题**：
- 运营者不知道"当前状态为什么成立"
- 运营者对"第二链路冻结"和"不跑 RUN-02"感到空洞无据
- 运营者需要快速追溯关键节点而不翻大量文档

### 触发条件

| 类型 | 条件 |
|------|------|
| 正常触发（主）| 下一月度运行窗口到达 → 用户发出"开始 RUN-next"指令 |
| 数据触发（辅）| 曾刚连续2月0贡献 → 用户评估 |
| 数据触发（辅）| usable<3条 → 用户评估 |
| 数据触发（辅）| 集中负向反馈（≥2人同维度C/D）→ 评估 |
| 用户主动触发 | 用户明确要求提前运行 |
| 用户主动触发 | 用户提供债券货币来源候选 → 解冻评估 |

### 辅触发 ≠ 自动执行

辅触发是"评估通道"，不是"立即执行 RUN-02"：
触发辅条件 → agent 评估 → 用户决定是否触发
