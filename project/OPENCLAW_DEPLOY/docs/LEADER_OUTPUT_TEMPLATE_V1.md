## 二、V1 四段结构

```

月度同业存款/贷款分析
（数据截至 XXXX年XX月）

━━━ 【一、本月最重要变化】（≤3条）━━━━━━━━━━
[四字判断 + 支撑数据 + 一句话意义]
[格式：判断句（鲜明）+ 数据佐证 + 背景补充]

示例：
1. 【中信存款攻势超预期】
    中信银行当月新增存款2401亿，增量居样本行第一，超出预期。
    [背景：两会后项目融资落地，支撑其对公存款增长]

2. 【华夏存款增速领先但待验证】
    华夏银行年增+6.3%，增速全行业最高。
    [但样本期仅3个月，尚无法判断是否趋势性变化，见下方注]

3. 【平安外币存款流入加快】
    平安银行外币存款月增+121亿，年增+19.7%，外币资金流入提速。

━━━ 【二、原因判断与对我行意义】（合并段）━━━━━━━━━
[机制推断 + 置信度标注 + 对我行业务含义；三段式]

针对变化1（中信攻势）：
    机制：两会后稳投资政策落地，对公结算存款同步增加
    置信度：【中】逻辑通顺，但缺中信贷款投向直接数据
    对我行意义：中信对公存款攻势若持续，可能分流我行重点客户存款
    ←注：若我行在中信有重点客户，需排查是否有挖转迹象

针对变化2（华夏提速）：
    机制：定期储蓄产品利率竞争力较强，叠加网点扩张
    置信度：【低】样本期仅3个月，尚无法判断是趋势还是季节性波动
    对我行意义：华夏存款定价策略持续性待观察，下月增速是否回落是关键信号
    ←注：本判断置信度低，已降调处理，结论待验证

针对变化3（平安外币流入）：
    机制：美联储降息预期下，客户锁汇意愿下降，部分资金从理财回流本币
    置信度：【低】样本期短，需持续跟踪
    对我行意义：外币存款的分流效应可能同步影响我行外币业务，需关注
    ←注：招行贷款利率已降至2.54%，若以贷促存，可能同步带动其存款争夺

━━━━━━━ 【三、三条建议】（行动段）━━━━━━━━━━
[结论鲜明，可立即安排]

1. 【立即排查中信挖转风险】
    目的：确认中信存款增量中是否有来自我行的转移存款
    建议责任人：分行客户管理部门
    验证指标：本月内存量重点客户（大额存单到期客户）中，是否有单笔转投金额≥1亿元

2. 【监测华夏定价策略走向】
    目的：判断华夏存款增长是价格驱动（不可持续）还是战略扩张（可持续）
    建议责任人：战略研究/企金部门
    验证指标：下月华夏存款增速是否回落至3%以内；若持续>5%，则判断为战略驱动

3. 【启动招行贷存款联动预警】
    目的：招行贷款利率2.54%已是行业最低，若以贷促存对我行优质客户有效，需提前预警
    建议责任人：分行公司业务部门
    验证指标：我行优质客户（年结算量>1亿）近30天内是否有转招行贷款超过2户

▷ 观察项A：外币存款流入是否持续
    观察什么：平安银行外币存款月增量（是否持续>100亿）
    频率：每月
    预警：若连续2个月>100亿且我行外币存款流失>10%，升级为主动作

▷ 观察项B：定期储蓄存款占比是否加速上升
    观察什么：各家银行定期存款占总存款的比例变化趋势
    频率：每月
    预警：样本行中3家以上月增占比>2个百分点，升级为主动作

━━━━━━━━━━━━━━━━━━━━━━━━

注（降调内嵌）：
【华夏银行增速】的判断置信度为"低"，证据样本期仅3个月，无法确认趋势性。
```

## 四、写作规范（V1 收紧版）

### 4.1 主结论格式（"一"段）

- **四字判断 + 数据 + 背景**：判断句必须是一个明确的结论句，不加"可能""或将"等模糊词
- **数据必须出现**：结论句后面直接跟数字，不允许只有判断没有数字
- **背景补充用括号**：与结论同段的补充信息用"【注】"或括号括起，与正文分开

### 4.2 判断段格式（"二"段）

- **三要素必须全**：机制 + 置信度 + 对我行意义
- **置信度必须标注**：高/中/低三档，不得省略
- **降调必须内嵌**：置信度为"低"的判断，在段内直接加注，不单独成段
- **"对我行意义"必须直接**：不允许只写"需关注"，必须写清楚"关注什么"

### 4.3 行动段格式（"三"段）

- **格式：结论句（判断先行）+ 目的 + 责任人 + 验证指标**
- **结论句必须是判断句**：如"立即排查中信挖转风险"，不是"建议关注是否有挖转"
- **验证指标必须可量化**：不允许"观察是否有异常"这种问句式指标

### 4.4 观察段格式（"四"段）

- **与"三"的区别必须在格式上体现**：观察项标题前用"▷"，主动作用"▶"
- **必须写执行人/执行部门**：观察项"谁来看"必须明确
- **预警信号必须量化**：不允许"明显异常时升级"，必须写具体阈值

## 五、版本变更说明（V0 → V1）

| 变更点 | V0 | V1 | 变更原因 |
|--------|----|----|---------|
| 段落数量 | 6段 | 4段 | 推断段与意义段重复，合并 |
| 降调处理 | 独立第六段 | 内嵌在变化2的注中 | 降调内容本质是附注，不应占据独立大段 |
| 主结论格式 | "xxx变化，xxx" | "【四字判断】：支撑数据" | 更鲜明，更像业务判断 |
| 推断格式 | "推断机制+依据+置信度" | "机制+置信度+意义+注" | 删除学术引用格式，保留业务判断核心 |
| 字数控制 | ≤1800字 | ≤1300字 | 压缩废话，提升阅读效率 |
| 置信度标注 | 有 | 必须有 | 缺了置信度的推断等于没有依据 |
