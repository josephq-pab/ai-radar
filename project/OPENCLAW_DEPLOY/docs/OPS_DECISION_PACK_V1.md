## 一、决策助手定位

**运营决策助手是什么：**
- 不是操作台（无运行按钮）
- 不是规则引擎（无动态判断逻辑）
- 不是后台配置系统（无编辑能力）
- 是：**只读场景化决策指引页**（ops-decision.html）

**决策助手解决什么问题：**
- 运营者遇到具体场景时，不知道该做什么、不该做什么、该看哪一页
- 状态信息已通过四页结构展示，但"遇到具体情况该跳去哪一页"仍需人工拼装
- 决策助手把分散在多份文档的判断逻辑，收敛成一个场景化入口

**决策助手不做什么：**
- 不提供动态触发判断能力
- 不替代触发规则文档（STAGE2_SINGLE_CHAIN_TRIGGER_RULES_V1.md）
- 不替代配置快照（config-status.html）
- 不替代证据链（ops-evidence.html）
- 不提供任何写入/编辑/自动化能力

## 五、当前通用决策原则

| # | 原则 | 含义 |
|---|------|------|
| P1 | 未触发不跑 | RUN-next 必须命中触发条件，不能"想跑就跑" |
| P2 | 无新输入不扩 | 没有用户提供的债券货币候选，不扩第二链路 |
| P3 | 冻结不是停摆 | 冻结是主动选择，意思是"当前不动，等条件" |
| P4 | 只读入口不是真源 | config-status.html 是快照，不等于 analyst_sources.json |
| P5 | 第二链路未解冻前不推进 | 在用户未提供候选来源的情况下，不能推进第二链路来源发现 |

## 八、为什么现在仍然停在等待态

**当前状态不是"卡住了"，而是正常等待：**

1. **RUN-01 已完成**：首轮月度运行已真实跑通，usable 池 32 条，足够支撑试用
2. **触发规则已冻结**：次轮触发规则 v1.0 已建立，未命中时不得强行跑
3. **第二链路冻结中**：结构性来源缺失，开放用户输入窗口，但 Agent 侧无能为力
4. **V7 基线冻结**：等连续2月正向试用反馈，非改稿时机
5. **决策助手已建立**：运营者现在有场景化决策工具，不再需要自己拼装

**等什么：**
- 下一月度运行窗口到达（正常触发）
- 或出现强烈异常信号（异常触发）
- 或用户提供债券货币来源候选（解冻触发）
- 或用户提供连续2月正向试用反馈（V7 解冻）

## 九、五页架构

| 页面 | 文件 | 定位 | 何时访问 |
|------|------|------|---------|
