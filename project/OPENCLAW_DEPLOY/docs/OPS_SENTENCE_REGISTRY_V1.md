## 一、Registry 结构说明

**句子分类体系：**
- **S0（硬约束句）**：全站硬约束，不得轻易改动，改动需要全站评估
- **S1（高风险同步句）**：改了需要同步多处，必须查依赖图
- **S2（可局部调整句）**：本页面/本模块内调整，不影响全局
- **S3（可压缩/可替换句）**：可优化或用 canonical 版本替换

**Canonical 版本原则：**
- Canonical 版本是最标准/最通用/最稳定的版本
- 页面变体句是 canonical 的快读版/解释版/训练版，不要求完全一致
- 但核心判断句和硬约束句必须在 canonical 版本控制下

## 九、句式风格规范

### 9.1 禁止使用的句式

| 禁止句式 | 替代句式 | 原因 |
|---------|---------|------|
| 基线已锁定 | V7 基线冻结 | "锁定"含义模糊 |
| 系统停摆 | 等待触发 | "停摆"含义负面且不准确 |
| 第二链路暂停 | 第二链路冻结 | "暂停"暗示会恢复，结构性缺失不是暂停 |
| 可触发RUN-next | RUN-next 由人工确认 | 不暗示自动触发 |
| 状态1/状态2 | W/L/A/P/E/R/S | 必须用标准状态代码 |
| 主层/次层 | 核心层/辅助层/索引层 | 必须用标准层级名 |
| 有页面=有功能 | 有页面=可查询（无后台） | 区分展示与操作 |

### 9.2 句式长度规范

| 场景 | 最大长度 | 原因 |
|------|---------|------|
| 状态 banner/badge | ≤10字 | 视觉空间限制 |
| 卡片/快读句 | ≤20字 | 快读需要简短 |
| 段落级说明 | ≤50字 | 防止过长影响阅读 |
| 模板句 | 无限制 | 但应尽量精简 |
