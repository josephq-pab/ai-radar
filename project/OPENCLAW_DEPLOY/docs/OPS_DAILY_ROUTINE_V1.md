## 背景

P3-31 已完成新输入最小响应路径，定义了停下后遇到新输入时如何最小重启。

但仍存在一个独立问题：

> **每次重新打开系统时，如何用一条固定巡检动作判断这次该停、该记、该解释、还是该进入评估？**

这个问题不需要新输入也会出现——即使没有任何新信号，运营者每次打开系统也需要一个**判断框架**，而不是凭感觉决定要不要继续看。

### 当前最常见的值班摩擦

| 摩擦 | 描述 |
|------|------|
| 每次都从头看很多页 | 没有最小路径，每次凭感觉决定看几页 |
| 不知道本次有没有新事 | 打开后无法快速判断"今天有没有动作" |
| 没有新输入却继续深查 | 感觉"再看几页更安全"，实际上不会改变结论 |
| 有一点信号就不知道该停该记还是该评估 | 判断框架缺失，信号来了无法分类 |

## 日常值班最小巡检路径（四步）

### 第一步：看状态横幅（首页顶部）

**做什么**：看统一状态横幅，确认当前四个冻结状态

**回答**：系统现在是等待态，还是有需要处理的事？

**判断**：
- RUN-next 未触发 + 第二链路冻结 + V7 冻结 → 等待态正常，无需动作
- 有任一状态异常 → 进入第二步进一步判断

---

### 第二步：看快报快读（ops-brief.html）

**做什么**：看快报核心结论，用3~6条口径快速判断本次是否有动作

**回答**：快报结论是"等待态正常"还是"需要关注"？

**判断**：
- 快报结论：等待态正常 + 无新增待办 → **本次值班无动作，可停止**
- 快报提示需要关注某项 → 进入第三步
- 不确定快报结论 → 进入第三步

---

### 第三步：看运营页判断（single-chain-ops.html）

**做什么**：看运营页核心判断区，确认本次是否有新触发/异常/未完成待办

**回答**：本次是否有新信号需要处理？

**判断**：

| 情形 | 结论 | 对应值班结果 |
|------|------|-------------|
| 无新信号 + 无待办 | **本次无动作** | 可停止 |
| 有轻微口径澄清/术语疑问 | **本次仅记录** | 看快报/术语即可 |
| 有新信号但不明确是否触发 | **本次进入评估** | 去决策页判断 |
| 有明确新数据/明显触发信号 | **本次进入准备** | 运营→决策→执行准备 |

---

### 第四步：按结果执行或停止

**做什么**：根据第三步的判断，执行对应最小动作

**执行路径**：

```
第三步判断为"无动作" → 直接停止，记录"本次值班无动作"
第三步判断为"仅记录" → 看快报/术语页，记录口径澄清，更新理解
第三步判断为"进入评估" → 去决策页判断是否触发，不自动执行
第三步判断为"进入准备" → 运营页→决策页→执行准备，不自动触发
```

---

## 值班四类结果定义

| 结果 | 定义 | 触发条件 | 典型动作 |
|------|------|---------|---------|
| **本次无动作** | 无新信号 + 无待办 + 等待态正常 | 快报结论"等待态正常" + 运营页判断"等待态正常" | 停止，不做任何记录 |
| **本次仅记录** | 口径/术语需要更新理解，但不改变状态判断 | 轻响应输入（口径澄清/术语疑问/回访确认） | 看快报/术语，更新理解，记录口径修正 |
| **本次进入评估** | 有新信号但是否触发不明确，需要判断 | 评估型输入（"这算不算触发"/轻微信号/提议提前跑）| 去决策页判断，不自动执行 |
| **本次进入准备** | 有明确新数据/明显信号，已接近或满足运行前条件 | 运行准备型输入（明确新数据/明显异常/连续异常）| 运营→决策→准备，需要确认才执行 |

## 本次可结束的条件

### 可直接结束巡检的条件（满足其一）

| 条件 | 说明 |
|------|------|
| 快报结论"等待态正常" | 无需进入运营页可直接停止 |
| 快报+运营均判断"等待态正常" | 确认无新信号，可停止 |
| 没有新输入 | 反复阅读不会改变结论 |
| 没有待办 | 自己在运营页无待办 |
| 无异常信号 | 无 usable 池异常/无集中负向反馈/无曾刚断档 |

### 仍需要继续深查的条件（必须同时满足才继续）

- 有新数据/新触发信号出现
- 有未完成待办
- 用户明确提出新的具体问题
- 出现异常信号

---

## 巡检输出最小口径

每次值班结束，记录以下最小口径：

```
值班结果：[无动作 / 仅记录 / 进入评估 / 进入准备]
判断依据：[快报结论 + 运营页判断]
日期：[YYYY-MM-DD]
备注：[如有口径修正/新理解]
```

**无动作时不需要写日报/记录**，只需要心里清楚"本次无动作"即可。
