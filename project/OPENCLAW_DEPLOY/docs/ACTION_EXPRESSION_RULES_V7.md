## V6→V7 核心变更

| 变更点 | V6 | V7 | 变更原因 |
|--------|----|----|---------|
| 三档条件表达 | 2行注语（压缩过头）| **V4/V5 完整段落** | Q2=B：逻辑正确，但压缩后指导性不足，需恢复数值背景 |
| 主动作格式 | 动作名+目的+做法 | **动词打头+具体操作标准（三步式）** | Q3=C：动作不像能部署，分析报告感太强 |
| "怎么做"细节层 | 偏分析描述 | **可部署的操作标准（三步式）** | Q1=D：执行者拿到的是摘要不是行动工具 |
| 研究准备"研究内容" | 3项选2项，无完成标准 | **明确完成标准和产出格式** | Q2=B：执行者不知如何算完成 |
| 执行版篇幅 | 约 850 字 | **约 950~1000 字** | 必要信息厚度回归 |
| 阶段验证重点 | V6 顺手度验证 | **V7 动作翻译验证** | Q3=C 直接指向动作表达失败 |

## 一、核心概念：动作闭环三角（V7 不变）

每一个动作建议都必须回答三个问题：

1. **谁来做？** → 责任人（部门 + 角色）
2. **做完交给谁？** → 接收方（仅研究准备动作需要）
3. **什么条件下要接着做？** → 转动作条件（仅研究准备动作需要）

## 四、主动作格式规则（V7 重写）

### 4.1 V7 核心原则

**动作建议必须看起来像可部署动作，而不是分析建议。**

检验标准：执行者读完这条动作，是否知道"今天/本周要做什么、联系谁、问什么、什么时候完成"。

### 4.2 V6→V7 格式对比

**V6 格式（分析报告感）**：
```
▶ 主动作X（本月必做）：[动作名称]

- 做这件事是为了：[防范什么/抓住什么机会，1句话]
- 建议责任人：[部门] / [角色] [确认标注]
- 怎么做：
    · 重点：[联系哪类客户，要确认什么]
    · 扫描：[快速扫哪类客户，重点是什么]
    · 预警：[超出什么阈值要立即上报]
```

**V7 格式（行动指令感，三步式）**：
```
▶ 主动作X（本月必做 / 完成时点）：[动作名称]

  做这件事是为了：[防范什么/抓住什么机会，1句话]
  建议责任人：[部门] / [角色] [确认标注]

  【具体怎么做】
  第一步：[第一个可执行的动作，具体到联系哪类客户/问什么问题]
    → 若[某种情况]，记录并立即上报[负责人]
  第二步：[第二个可执行的动作]
    → 若[某种情况]，立即上报[负责人]
  第三步：[整理什么结果，提交给谁]

  参考阈值：[参考值·可调]
  完成时点：XXXX年XX月XX日前
```

### 4.3 三步式写作规则

1. **每一步必须是可执行的具体动作**：不是"分析客户"，而是"联系5000万以上到期客户，确认续做意向"
2. **每一步必须跟结果判断**：→ 若[情况]，则[上报/记录/停止]
3. **第三步是整理提交**：执行结果整理成名单/报告，提交给接收方
4. **不使用"等措施"等模糊词**：具体到"问什么、联系谁、什么时候完成"

### 4.4 主动作最低标准（V7）

写完一条主动作后自检：

- [ ] 是否动词打头（"排查"而非"关于排查"）
- [ ] 执行者是否知道今天/本周要做什么
- [ ] 执行者是否知道联系谁
- [ ] 执行者是否知道问什么问题
- [ ] 执行者是否知道超出什么情况要上报
- [ ] 执行者是否知道什么时候完成
- [ ] 执行者是否知道完成后整理什么、交给谁

## 五、三档转动作条件规则（V7 恢复完整段落）

### 5.1 V7 核心原则

**逻辑完全保留，表达恢复 V4/V5 完整段落。**

三档条件是为了回答"研究结论出来后，下月要怎么做"，是研究准备的闭环出口。不能删除逻辑，不能只留结论，必须恢复数值背景。

### 5.2 V6→V7 格式对比

**V6（2行注语，问题：缺数值背景，执行者无法判断）**：
```
转动作条件（注语形式，压缩版）：
    · 若华夏3年定存利率高于我行15bp以上 且 月均新网点>5个 → 下月启动挖转排查
    · 若华夏月增连续2个月<500亿 → 下月归档
    · 其余情况：结论留存，下月补充数据后再判断
```

**V7（完整段落，恢复数值背景和三出口）**：
```
转动作条件（完整段落）：
    【条件A — 触发动作】
    若同时满足以下两个条件，则下月启动针对华夏的存款挖转排查：
      (1) 华夏3年定存利率高于我行15bp以上（价格驱动强）
      (2) 华夏近三月月均新网点>5个（渠道扩张驱动明显）
    → 下月启动挖转排查：重点联系有华夏网点覆盖区域的存量客户

    【条件B — 归档】
    若华夏月增连续2个月<500亿，说明增速回落后不再构成主要威胁
    → 下月归档，不再触发针对华夏的存款动作

    【条件C — 待定】
    其余情况（满足价格驱动但无渠道扩张；或满足渠道扩张但价格驱动不足）
    → 结论留存，下月补充数据后再判断
```

### 5.3 完整段落写作规则

1. **条件A（触发动作）**：说明两个条件的具体数值和判断逻辑
2. **条件B（归档）**：说明什么情况下不再构成威胁
3. **条件C（待定）**：说明什么情况下不下结论，下月继续补充
4. **每个条件必须说明"数值从哪里来"**（15bp、5个网点），或标注"各行可按实际调整后告知"

## 六、研究准备动作格式（V7 更新）

### 6.1 V7 核心原则

研究准备动作必须回答：
- 研究谁来做
- 做什么（3项选2项）
- 做完后产出是什么（完成标准 + 产出格式）
- 交给谁
- 什么条件下要接着做

### 6.2 V7 格式

```
▸ 研究准备（完成时点）：[研究主题]

  搞清楚这个问题是为了：为下月判断[什么]提供依据
  研究责任人：[部门]（注：企金部/战略研究部均可承接）[6源一致确认]
  接收方：[部门/角色]

  研究内容（3项选2项即可，不用全做）：
    1. [具体研究内容]
       完成标准：拿到[什么证据/数据/结论]
       产出格式：[具体格式，如：3~5行文字+1张对比表]
    2. [具体研究内容]
       完成标准：拿到[什么证据/数据/结论]
       产出格式：[具体格式]
    3. [具体研究内容]
       完成标准：拿到[什么证据/数据/结论]
       产出格式：[具体格式]

  产出形式：一页书面判断报告（3~5个要点）【打头第一条结论必须最核心】

  转动作条件（完整段落格式）：
    [同第五节]

  预计完成时点：XXXX年XX月XX日前
```

## 七、执行侧动作表达最低厚度（V7 新增）

### 7.1 不能再压掉的信息

以下信息在执行版中**不能再压缩**，必须保留：

| # | 信息类型 | V6处理 | V7最低要求 |
|---|---------|--------|-----------|
| 1 | "做这件事是为了"（目的句）| 有，但被压缩 | 必须有，且必须是1句话 |
| 2 | 责任人（部门+角色）| 有 | 必须有 |
| 3 | 具体操作步骤 | V6压缩为"重点/扫描/预警"三点 | V7必须展开为三步，每步具体到联系谁/问什么 |
| 4 | 三档条件数值来源 | V6只有"15bp"，不说来源 | V7说明"为什么是这个数"，或标注"各行可调" |
| 5 | 研究内容完成标准 | V6无 | V7必须写明每项研究的"完成标准"和"产出格式" |
| 6 | 完成时点 | 有但不强制 | 必须有，明确到"XX月XX日前" |

### 7.2 动作建议最低标准

| 检查项 | 标准 |
|--------|------|
| 可部署感 | 执行者读完知道"今天做什么"，不是"本月做什么" |
| 具体性 | 具体到联系哪类客户、问什么问题 |
| 可判断性 | 执行者知道"做到了"是什么样子 |
| 阈值透明 | 执行者知道阈值是怎么来的，可以判断是否需要调整 |

## 八、V7 检查表（13问）

写完一条动作后自检：

**主动作检查（9问）**
1. 动作类型是否明确（立即执行 / 研究准备）？
2. 是否动词打头？
3. 执行者是否知道今天/本周要做什么？
4. 执行者是否知道联系谁、问什么问题？
5. 责任是否到部门+角色？
6. 是否有三步操作，每步有结果判断？
7. 是否有"超出什么情况要立即上报"？
8. 是否有明确的完成时点？
9. 研究准备是否有接收方和转动作条件（完整段落）？

**三档条件检查（4问）**
10. 条件A是否有具体数值和来源说明？
11. 条件B是否有明确的归档条件？
12. 条件C是否有待定情况的处理方式？
13. 三档条件是否回答了"研究结论出来后下月怎么做"？
