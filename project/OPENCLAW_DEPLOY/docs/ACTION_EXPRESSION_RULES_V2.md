## 一、核心概念：动作闭环三角

V2 新增概念：**动作闭环三角**——每一个动作建议，都必须回答三个问题才算完整闭环：

1. **谁来做？** → 责任人（部门 + 角色）
2. **做完交给谁？** → 接收方（仅研究准备动作需要）
3. **什么条件下要接着做？** → 转动作条件（仅研究准备动作需要）

**立即执行动作**只需要回答第1问，闭环终点是"完成/未达到验证指标"。
**研究准备动作**需要回答全部三问，闭环终点是"接收方读完结论并决定是否触发下一步"。

---

## 二、动作类型及闭环要求（V2 完整版）

| 类型 | 定义 | 闭环终点 | 是否需要接收方 | 是否需要转动作条件 |
|------|------|---------|--------------|-----------------|
| **立即执行** | 本月必须有人推动，完成后直接影响业务结果 | 完成后确认验证指标（达到/未达到）| ❌ 不需要 | ❌ 不需要 |
| **研究准备** | 本月需要搞清楚一件事，结果给下月提供判断依据 | 研究结论被接收方阅读并决定是否触发动作 | ✅ 需要 | ✅ 需要 |
| **建立机制** | 本月启动但本月完不成，效果在下个月之后体现 | 机制进入常态运行，下月由谁运行待定义 | ⚠️ 视情况 | ⚠️ 视情况 |

## 三、研究准备动作的标准格式（V2 完整版）

V2 研究准备格式相比 V1，新增两个必填字段：**接收方**和**转动作条件**。

```
▸ 研究准备：[研究主题]
    研究目的：[搞清楚这个问题是为了支持什么判断]
    研究责任人： [部门] / [角色]
    研究内容：
      1. [具体要拿到的证据/数据/结论 1]
      2. [具体要拿到的证据/数据/结论 2]
    产出形式：[报告 / 表格 / 备注]
    接收方：[部门] / [角色]
    转动作条件：
      若[触发事实]达到[判断门槛]，则下月启动[对应动作]
    预计完成时点：XXXX年XX月XX日
    注意：此动作本月不产生直接业务动作
```

## 四、责任归口要求（V2 新增）

### 4.1 粒度要求

**V2 责任归口必须到"部门 + 角色"两级**，仅写部门级视为不完整：

| 正确写法 | 错误写法 |
|---------|---------|
| 分行公司部 / 客户经理 | 分行公司部（无角色）|
| 战略研究部 / 行业研究员 | 相关部门（无部门+角色）|
| 资产负债部 / 利率管理岗 | 资产负债部（无角色）|

### 4.2 联合行动写法

当动作需要多个部门/角色配合时，使用"主办 + 支持"写法：

```
研究责任人：战略研究部（主办）+ 分行公司部（支持）
角色建议：行业研究员（主办）+ 客户经理（支持）
```

**禁止写法**：
- ❌ 只写主办，不写支持（联合行动缺少一方信息不完整）
- ❌ 把"支持"写成"协助"（不明确支持什么）
- ❌ 写具体人名（具体人是派工单的事，不是分析稿的事）

### 4.3 降调写法（组织架构不确定时）

当动作责任部门/角色在各行之间可能存在差异时，使用降调写法：

```
建议责任人：分行公司部 / 客户管理部（注：各部门均可承接，视各行实际情况定）
```

## 五、接收方的写法规则（V2 新增）

### 5.1 接收方的定义

研究准备的产出（报告/表格/备注）完成后，需要交给"能决定要不要触发下月动作"的人阅读，这个人就是接收方。

**接收方的判断标准**：
- 有权决定"这个结论要不要触发下一步动作"
- 一般是部门负责人或分行长级别
- 不是实际做研究的角色（研究责任人≠接收方）

### 5.2 接收方的典型写法

| 研究主题 | 典型接收方 |
|---------|----------|
| 竞争对手存款策略研究 | 分行公司部负责人（或分行长）|
| 贷款利率走势判断 | 资产负债部负责人 |
| 客户异动深度研究 | 客户所属部门负责人 |
| 宏观政策对存款影响 | 战略研究部负责人 |
| 产品竞争力分析 | 企金部 / 产品部门负责人 |

### 5.3 接收方的禁止写法

- ❌ "留存备查"（等于没人看）
- ❌ "交由相关部门"（太泛）
- ❌ 不写（研究白做）

## 六、转动作条件的写法规则（V2 新增）

### 6.1 转动作条件的结构

每个转动作条件必须包含三要素：
1. **触发事实**：研究结论中出现的具体事实/数据
2. **判断门槛**：达到什么标准就触发
3. **对应动作**：触发后，下月应该执行什么动作

**标准格式**：
```
若[触发事实]达到[判断门槛]，则下月启动[对应动作]
```

### 6.2 转动作条件示例

**存款竞争研究**：
```
若华夏银行3年定存利率持续高于我行15bp以上，
且网点扩张月均新增>5个，
则下月启动"华夏银行重点客户挖转风险排查"主动作
```

**招行联动研究**：
```
若本期扫描发现3户以上重点客户反映"近期有招行联系过"，
则下月启动"招行贷存款联动流失风险客户排查"主动作
```

### 6.3 降调版转动作条件

当研究结论不确定时，使用降调版：
```
目前数据样本不足以给出确定判断。
预计下月可形成结论，届时根据结论类型触发对应动作：
  - 若[触发事实A]：启动[动作A]
  - 若[触发事实B]：启动[动作B]
  - 若[触发事实C]：本轮归档，不再触发动作
```

### 6.4 转动作条件的禁止写法

- ❌ "若研究结论支持，则触发"（触发事实不具体）
- ❌ "届时再决定"（等于没写转动作条件）
- ❌ 不写（研究结论永远停在结论)

## 七、立即执行动作的格式（V2 更新）

V2 的立即执行动作，相比 V1 增加了**完成时点**（具体日期）：

```
▶ 主动作（本 月 必 做）：[动作名称]
    目的：[做这件事是为了防范什么/抓住什么]
    建议责任人：[部门] / [角色]
    验证指标：
      直接指标：[可量化的具体数字或行为]
      间接指标：[可观察的相关信号]
      预警阈值：[超出多少需要立即上报]
    完成时点：XXXX年XX月XX日
```

**完成时点的重要性**：没有具体完成时点，"立即执行"动作就变成了"有空就做"的动作，月底不会被回顾。

## 八、V2 检查表（7问）

写完一条动作后，用以下7问自检：

1. **动作类型是否明确？** → 不能把研究准备写成立即执行
2. **动作是否具体？** → 不能用"加强/关注/做好"开头
3. **责任是否到部门+角色？** → 不能只写部门
4. **是否有可验证指标？** → 不能写"持续跟踪"
5. **是否区分了"立即执行/研究准备"？** → 两类动作不能混格式
6. **研究准备是否有接收方？** → 不能留空
7. **研究准备是否有转动作条件？** → 不能留空；若结论不确定是否写了降调版

## 九、版本变更说明（V1 → V2）

|--------|----|----|---------|
| 研究准备闭环要素 | 5项（缺接收方+转动作条件）| 7项（完整闭环）| RP-01/RP-02 P0问题解决 |
| 接收方 | 未定义 | 必须填写（研究准备动作）| RP-01 解决 |
| 转动作条件 | 未定义 | 必须填写（研究准备动作）| RP-02 解决 |

| 组织架构差异降调写法 | 未定义 | 降调用语 | RA-05 解决 |
