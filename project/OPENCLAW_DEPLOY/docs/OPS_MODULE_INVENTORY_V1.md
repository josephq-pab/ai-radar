# OPS_MODULE_INVENTORY_V1.md — 全站模块级 Inventory 总表

> 文档版本：v1.0（P3-42）
> 创建时间：2026-04-21
> 对应阶段：Phase 3 — P3-42 全站模块级 inventory、依赖关系、重复治理与模块维护底图功能包第一期
> 目的：建立十页体系的模块级清单，让任何人都能精确回答"这个模块叫什么、做什么、属于哪页、谁依赖它"

---

## 一、编制说明

**本清单覆盖范围：**
- 10 个 HTML 页面（展示层）
- 核心治理文档（规则/训练/生命周期/维护类）
- 不包括：analyst_sources.json、脚本、V7 基线内容

**模块唯一命名规则：**
- 格式：`{页面}-{模块序号}-{模块名}`
- 示例：`radar-home-M03-状态横幅` = radar-home 第3个模块"状态横幅"
- 页面前缀：home / brief / ops / evidence / decision / playbook / glossary / registry / routes / config

**模块类型分类：**
- `状态总览` = 直接反映当前系统状态（看板/横幅/KPI）
- `判断模块` = 提供决策判断依据（条件/规则/触发）
- `路径引导` = 告诉用户下一步去哪（导航/流程图）
- `训练提示` = 防止用户误用的说明（训练入口/误判澄清）
- `治理维护` = 给维护者看的治理提示（维护说明/不可改清单）
- `真源映射` = 将页面内容映射到真源文档
- `证据说明` = 解释为什么状态成立
- `场景判例` = 提供典型场景的分类与处理路径
- `生命周期` = 说明当前处在生命周期哪一段
- `留痕/升级/回复` = 提供最小输出模板
