## 变更控制原则

Phase 3 性质为**收敛稳定**，非扩展阶段。

**三类处理规则**：
- **当前必须做**：纳入 P0/P1 任务包，进入主线
- **后续可做**：记录在"延后处理事项"节，不进入当前主线
- **当前明确不做**：记录在"本阶段不做事项"节，说明理由

## 变更申请流程

> **硬化规则（C-03）**：任何新增变更必须先记录到本文档"新增建议变更记录"节并形成结论（批准/拒绝/延后），才能进入 TASK_PACKAGE。不得先进入 TASK_PACKAGE 再补记录。

当用户提出新想法时，按以下流程处理：

1. **判断是否属于 Phase 3 五项推进点**：是 → 进入下一步；否 → 归入延后处理
2. **判断是否阻断 MVP 闭环**：是 → 评估紧急程度；否 → 进入 backlog
3. **判断是否有外部依赖**：是 → 标注条件，不作为当前任务推进
4. **记录到本文档"新增建议变更记录"节，形成结论**（批准/拒绝/延后）
5. **DECISION_LOG 同步记录**（D-xx 编号对应到 CC-xx）
6. **结论为批准后，才可进入 TASK_PACKAGE**
