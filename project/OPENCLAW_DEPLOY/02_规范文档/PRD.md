# 对公 AI 雷达站 PRD v0.1

## 1. 产品概述

对公 AI 雷达站是面向平安银行公司业务管理部的经营情报、观点研判与策略建议系统。系统围绕对公业务整体、存款、贷款三大板块，持续监测同业、政策、行业变化，通过人工与大模型协同加工，形成趋势判断、经营建议与行动建议，为公司业务决策和策略调整提供支持。

## 2. 产品使命

- 让高层管理者更快看到真正重要的市场变化
- 让业务管理者获得高质量、可落地的经营情报
- 让同业、政策、行业信息不再停留在信息堆积，而是转化为经营判断
- 让人工经验、管理偏好和模型能力形成协同闭环，不断提升雷达大脑的准确性与适配性

## 3. 核心用户

- 对公行长
- 业务部门领导
- 经营分析小组
- 贷款业务决策人员
- 存款业务决策人员

报告最终由邱非确认定稿。

## 5. 输出机制

### 5.1 输出频率

- 固定输出：高质量周报
- 弹性输出：不定期专题报告（重大市场事件 / 监管新政策 / 重点行业变化）

### 5.2 输出形式

- 周报
- 专题报告
- 重点观点池
- 同业动态池
- 政策影响池
- 行业趋势池

### 5.3 输出风格

输出遵循三层结构：
1. 观察提示
2. 策略建议
3. 行动建议

要求从事实推演策略，再输出行动建议。

## 6. 处理机制（避免信息堆场）

### 6.1 信息准入

所有采集信息先进入候选池，不直接进入正式报告。每条信息至少补齐：
- 来源分类：同业 / 政策 / 行业
- 业务板块：整体 / 存款 / 贷款
- 影响对象：产品 / 行业 / 定价 / 竞争 / 策略
- 时效等级
- 重要程度
- 可信度

### 6.2 价值判断

只有能支持经营判断的信息才进入重点加工流程。判断问题包括：
- 是否影响竞争格局
- 是否影响存贷款结构
- 是否影响产品策略
- 是否影响定价
- 是否值得进入周报或专题报告

### 6.3 观点加工

重要信息必须转化为：
- 事实
- 含义
- 影响
- 初步判断
- 建议方向

### 6.4 观点博弈验证

重点观点需保留：
- 支持证据
- 反向证据
- 不确定点
- 人工待确认点

### 6.5 输出限量

高层输出坚持少而硬，按重要性排序，避免全量堆积。

## 7. 人机协同

### 7.1 机器负责

- 观点初稿

### 7.2 人工负责

- 报告定稿

## 8. 升级机制

系统通过以下方式持续升级：
- 记录原始生成内容与人工修改后内容
- 记录修改类型与修改原因
- 记录用户期望内容与表达偏好
- 复盘哪些判断被验证、哪些建议被采纳
- 学习对不同角色更有效的表达模板与结论结构

核心目标：让用户对生成结论的调整事项越来越少。
