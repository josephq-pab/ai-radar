# 对公AI雷达站｜OpenClaw 协作共识说明

## 1. 项目定位

“对公AI雷达站”是一个面向总行管理层和经营分析团队的信息管理与决策支持系统，不是营销大屏，也不是单纯资讯聚合页。

项目目标是围绕以下链路构建综合能力：
- 信息感知
- 判断加工
- 行动支持
- 闭环跟踪

系统最终输出形态为：
- 看板
- 周报
- 专题
- 跟踪台账

## 2. 服务对象

主要服务对象：
- 总行管理层
- 经营分析团队

协同对象包括：
- 科技团队
- 数据团队
- 业务经营团队
- 专家支持团队

## 3. 项目级固定约束（后续不得随意覆盖）

### 3.1 业务维度固定为
- 整体对公业务
- 存款
- 贷款

### 3.2 主题维度固定为
- 监管雷达
- 行业风险
- 同业动态
- 行业热点
- 分析师观点
- 对标分析

### 3.3 首期建设策略固定为
- 整体对公业务：轻量上线，承担总览、统筹和导航功能
- 存款：适度做深
- 贷款：适度做深

### 3.4 输出形态固定为
- 看板
- 周报
- 专题
- 跟踪台账

### 3.5 页面设计总原则固定为
- 企业级后台
- 高密度信息组织
- 表格优先
- 固定页面骨架
- 白名单组件
- 先结构后视觉
- 所有重点内容必须可钻取
- 强调闭环跟踪，不做只展示不行动的页面

## 4. 输入分层规则

后续所有输入请按以下层级理解，不得将低层修改误判为高层重定义：

### 第1层：项目宪法层
包括：
- 项目定位
- 服务对象
- 业务维度
- 主题维度
- 首期策略
- 输出形态
- 设计总原则

### 第2层：系统架构层
包括：
- 页面层级
- 页面职责
- 首页、看板、清单页、专题页、跟踪页之间的关系

### 第3层：页面规范层
包括：
- 首页规范
- 看板规范
- 交互规范
- 组件映射
- 筛选规范

### 第4层：案例内容层
包括：
- 存款维度的重点模块
- 贷款维度的重点模块
- 具体业务实践内容
- 对标分析内容

### 第5层：局部修改层
包括：
- 某个模块位置调整
- 某个字段增加
- 某个按钮改法
- 某段文案优化

规则：
- 高层约束优先于低层修改
- 低层输入不得覆盖高层结构
- 若冲突，优先保留项目宪法层和正式页面规范层

## 5. 修改执行规则

### 5.1 默认增量修改
除非明确要求“全量重写”，否则一律按增量修改处理。

### 5.2 修改前必须先说明
在执行任何修改前，必须先输出：
- 本次修改影响范围
- 本次修改不影响哪些内容
- 是否与既有规范冲突
- 如冲突，按什么优先级处理

### 5.3 页面生成规则
生成页面时，应先确定稳定模块结构，再填入业务内容，不得让零散业务内容冲散页面框架。

## 6. 当前阶段任务边界

当前阶段只允许处理以下任务：
1. 建立统一项目理解
2. 接收首页与看板规范
3. 接收首页 schema
4. 接收存款看板深版 schema

当前阶段暂不处理：
- 全系统完整视觉稿
- 多页面同时重写
- 零散的局部审美优化
- 无上位约束情况下的自由发挥

## 7. 当前协作目标

当前协作目标是：
- 先稳定首页 schema
- 再稳定存款看板深版
- 之后再进入贷款维度和更深案例
