# 对公AI雷达站｜首页与看板 OpenClaw 执行规范

## 1. 文档用途

本文件用于将“对公AI雷达站”的首页、看板模块和交互逻辑转化为可被 OpenClaw 长期复用的执行规范。

目标不是自由创意设计，而是让 OpenClaw 在遵守既有设计系统的前提下，稳定输出一致的页面结构、组件映射和交互行为。

## 2. 页面层级理解

OpenClaw 必须把“对公AI雷达站”理解为多层级 B 端管理后台，而不是单页 dashboard。

页面层级如下：

### 一级层：系统首页 / 总览入口
承担：
- 全局概览
- 重点变化发现
- 快速进入各业务维度
- 快速进入重点事项和专题
- 查看周报与跟踪状态

### 二级层：业务维度看板
业务维度：
- 整体对公业务
- 存款
- 贷款

每个维度下挂接主题：
- 监管雷达
- 行业风险
- 同业动态
- 行业热点
- 分析师观点
- 对标分析

### 三级层：主题详情 / 清单 / 专题 / 跟踪页
用于承载高密度筛选、表格、专题详情、事项闭环和对标分析深页。

## 3. 页面骨架统一规则

所有页面固定采用以下区域：
- topbar
- tabs
- sidebar
- main_content

### Tabs 建议
- 首页总览
- 整体对公业务
- 存款
- 贷款
- 重点事项跟踪
- 周报与专题

## 4. 首页定位

首页是管理型 dashboard overview page，不是数据大屏。

首页必须同时解决两个问题：
1. 快速看到最值得关注的变化
2. 快速进入可行动的后续页面

## 5. 首页固定模块

首页 main_content 必须包含以下模块：

1. 全局提示区 home_alert_strip
2. 今日/本周重点概览 home_focus_summary
3. 业务维度导航 home_dimension_navigation
4. 雷达主题矩阵 home_theme_radar_matrix
5. 重点事项与预测跟踪 home_tracking_panel
6. 周报与专题入口 home_report_panel
7. 待补充资料与待处理任务 home_pending_actions

## 6. 首页模块要求

### 6.1 全局提示区
目标：
- 放置高优先级提示
- 不堆叠过多

组件：
- alert
- button

### 6.2 今日/本周重点概览
目标：
- 让管理层一眼看到当前最重要变化与结论

建议按三列组织：
- 监管与政策
- 经营与同业
- 风险与跟踪

### 6.3 业务维度导航
必须平级展示：
- 整体对公业务
- 存款
- 贷款

其中：
- 整体对公业务偏总览与导航
- 存款和贷款可显示更深提示

### 6.4 雷达主题矩阵
必须用“业务维度 × 主题维度”表格呈现：

行：
- 整体对公业务
- 存款
- 贷款

列：
- 监管雷达
- 行业风险
- 同业动态
- 行业热点
- 分析师观点
- 对标分析

要求：
- 使用 table 思维组织
- 行列语义清晰
- 单元格支持钻取

### 6.5 重点事项与预测跟踪
必须体现闭环跟踪，不得只展示资讯。

需展示：
- 事项名称
- 业务维度
- 当前状态
- 最新进展
- 下一步动作

### 6.6 周报与专题入口
需并列展示：
- 最新周报
- 最新专题

### 6.7 待补充资料与待处理任务
需体现提问式优化与协同采集能力。

## 7. 维度看板设计原则

### 7.1 整体对公业务看板
定位：
- 轻量总览
- 导航统筹

### 7.2 存款看板
定位：
- 首期适度做深

优先深化主题：
- 同业动态
- 行业热点
- 分析师观点
- 对标分析

### 7.3 贷款看板
定位：
- 首期适度做深

优先深化主题：
- 监管雷达
- 行业风险
- 分析师观点
- 对标分析

## 8. 关键交互逻辑

### 8.1 首页到维度看板
点击维度入口后：
- 打开对应维度看板
- 保留最近一次时间筛选（如有）
- 默认滚动到维度摘要区

### 8.2 首页矩阵到主题清单页
点击矩阵单元格后：
- 自动注入维度和主题筛选条件
- 打开该主题列表页
- 默认显示最近一周数据

### 8.3 事项进入跟踪详情
点击事项后：
- 打开事项详情
- 展示状态流转、最新进展、相关专题、建议动作

### 8.4 从待补充任务进入上传页
点击补充动作后：
- 自动带入相关主题与维度
- 显示系统建议补充问题
- 上传后返回对应专题或事项

## 9. 筛选规范

首页只保留轻量筛选：
- 时间范围
- 业务维度
- 主题类型

复杂筛选放在维度看板和主题清单页。

## 10. 组件与视觉原则

必须遵守：
- 企业级后台
- 高密度
- 表格优先
- 固定骨架
- 不使用娱乐化视觉
- 不大面积卡片化
- 一个区域只保留一个高强调主按钮

## 11. OpenClaw 输出要求

当处理本项目页面时，必须按以下结构输出：
1. 需求解析
2. 页面类型判断
3. 规范映射表
4. 页面结构树
5. 模块清单
6. 组件布局说明
7. 交互事件流
8. 合规检查表
9. 最终页面方案
