## 一、项目背景

**对公 AI 雷达站**是面向平安银行公司业务管理部的经营情报、观点研判与策略建议系统。

系统定位：
- 围绕对公业务整体、存款、贷款三大板块
- 持续监测同业、政策、行业变化
- 通过人工与大模型协同加工
- 形成趋势判断、经营建议与行动建议

核心用户：对公行长、业务部门领导、经营分析小组、贷款和存款业务决策人员。

---

## 二、产品架构

### 2.1 三大业务板块
1. 对公业务整体
2. 存款板块
3. 贷款板块

### 2.2 信息监测优先级
1. 同业（重点监测对象：光大、华夏、民生、平安、浦发、兴业、招商、中信）
2. 政策（宏观、监管、金融五篇大文章相关产业）
3. 行业（五篇大文章涉及重点行业）

### 2.3 输出机制
- 固定输出：高质量周报
- 弹性输出：专题报告（重大市场事件 / 监管新政策 / 重点行业变化触发）

### 2.4 输出结构
三层递进：观察提示 → 策略建议 → 行动建议

## 四、当前系统能力

### 4.1 可用页面

| 页面 | 文件 | 说明 |
|------|------|------|
| 首页总览 | index.html | 主链路入口，展示 8 家同业对标数据 |
| 存款深案例 | deposit.html | 关键观察、建议动作、解释规则、跟踪事项摘要 |
| 贷款深案例 | loan.html | 关键观察、建议动作、利率观察、解释规则、跟踪事项摘要 |
| 重点事项跟踪 | tracking.html | 状态筛选、事项主表、详情预览、来源回溯 |
| 周报与专题 | weekly-report.html | 周报草稿、待确认事项、事项闭环摘要、规则承接 |
| 同业分析 | peer-monitor.html | 二级承接页 |

### 4.2 数据处理管道

```
原始Excel数据（存款/贷款/利率）
    ↓
parse_initial_data.py → 数据中间层（JSON/CSV）
    ↓
build_interpretation_rules.py → 解释规则
build_recommendation_rules.py → 建议生成规则
    ↓
generate_weekly_report.py → 周报草稿
generate_review_queue.py → 待确认事项队列
    ↓
build_review_status.py → review 状态汇总
build_tracking_items.py → tracking 中间层
    ↓
build_web_bundle.py → 前端数据包（app-data.js）
    ↓
前端页面展示
```

### 4.3 规则体系

- **偏好学习画像**：从真实 review 提炼主题偏好，动态调整生成风格
- **栏目模板**：按栏目（核心结论/同业动态/贷款观察/定价/策略建议/待确认）生成口径指引
- **解释规则**：存款/贷款/定价/同业板块的先看什么、怎么判断、追问什么
- **建议生成规则**：观察提示/策略建议/直接行动建议的分层标准与优先级

## 六、待解决问题与下一步

### 6.1 当前核心问题

**问题 1：外部信息源尚未接入**

当前原型主要依赖内部对标数据（月度更新），缺乏：
- 外部分析师观点（公众号文章）
- 监管政策动态
- 行业趋势信息

**问题 2：分析师观点来源**

计划在存款/贷款/整体对公各找 5-10 位资深分析师，定期爬取其公众号观点作为雷达信息输入，但尚未落地：
- 分析师名单待提供
- 抓取频率待确定
- 观点录入格式待设计

### 6.2 路线 A 下一步（让原型先跑起来）

1. 按 SOP 跑一次完整周节奏，验证全链路是否真正可跑
2. 用真实数据做分析，确认输出质量是否达到管理层标准
3. 收集真实使用反馈，迭代偏好画像和规则
4. 分析师观点接入：等名单确认后，搭建公众号爬取 + 观点池 + 跟踪闭环

### 6.3 路线 B（同时准备）

在现有存款/贷款数据基础上，补充政策标签和行业维度分析，为后续扩展做准备。
