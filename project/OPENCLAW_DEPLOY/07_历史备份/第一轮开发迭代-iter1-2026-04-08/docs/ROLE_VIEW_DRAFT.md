# ROLE_VIEW_DRAFT.md — Phase 3 角色视图草案

> 文档版本：v1.0（初建）
> 最近更新：2026-04-07
> 对应阶段：Phase 3 — 试点运营化 / MVP闭环

---

## 角色说明

本文档定义 Phase 3 MVP 阶段各角色的视图边界和能力范围。

**当前阶段原则**：MVP 阶段只保留最小角色能力，不做权限扩展。

---

## 角色一：系统维护者（AI雷达站 Agent）

- **核心目标**：确保抓取、生成、呈现链路稳定运行
- **关注信息**：
  - 抓取成功率 / 失败率
  - 数据 freshness（最近一次抓取时间）
  - 脚本执行日志
  - 异常告警
- **可执行动作**：
  - 运行 fetch_analyst_articles.py
  - 运行 run-pipeline.py
  - 检查 analyst_opinions_raw.json 内容
  - 维护 analyst_sources.json 来源配置
- **不需要看到的内容**：
  - 前端页面渲染细节（由运营角色负责）
  - 报告内容审核（由编辑角色负责）
- **当前阶段最小支持能力**：
  - 命令行执行入口
  - 日志输出可读
  - 错误信息可定位
  - 非交互式运行（cron / scheduled）

---

## 角色二：运营编辑者（业务使用者）

- **核心目标**：从抓取数据中确认可用的分析师观点，转化为报告输入
- **关注信息**：
  - 最近抓取的分析师观点列表（标题 / 作者 / 发布时间 / 来源）
  - 每条观点的 summary 和 keyViewpoints
  - 确认项分级（confirmable / referenceable / reportable）
  - 历史 review 记录
- **可执行动作**：
  - 查看 analyst_opinions.json（reviewed 层）
  - 对观点进行 confirm/reject 操作
  - 将确认后的观点移入 report-ready 层
- **不需要看到的内容**：
  - 爬虫脚本细节
  - raw JSON 原始内容（除非需要查错）
  - 数据库结构
- **当前阶段最小支持能力**：
  - 前端页面查看抓取结果摘要
  - 手动确认/驳回操作（当前为纸面流程，未系统化）
  - 运行日志可见

---

## 角色三：报告阅读者（最终用户）

- **核心目标**：获取对公业务领域的最新分析师观点和置信判断
- **关注信息**：
  - 高优先级确认项（P1）
  - 关键观点摘要
  - 数据 freshness 标注
  - 来源列表
- **可执行动作**：
  - 读取报告页面
  - 查看置信等级和判断依据
- **不需要看到的内容**：
  - 原始抓取内容
  - review 过程记录
  - 爬虫状态
- **当前阶段最小支持能力**：
  - 静态 HTML 页面
  - 关键数据有标注来源
  - 数据月份标注

---

## 当前阶段只保留的最小角色能力

| 角色 | MVP 最小能力 |
|------|------------|
| 系统维护者 | 命令行可执行 + 日志可读 + 非交互式运行 |
| 运营编辑者 | 查看 reviewed 层 + 手动 review 操作（纸面） |
| 报告阅读者 | 静态页面 + 数据有来源标注 |

---

## 后续可扩展但暂不纳入的内容

| 扩展方向 | 触发条件 |
|---------|---------|
| 运营编辑者在线确认操作界面 | review 工作量超过人工承受阈值 |
| 多级权限（编辑/审核/发布） | 有多人协作需求且产生权限冲突 |
| 系统维护者告警通知 | 运行失败频率超过阈值 |
| 报告阅读者反馈入口 | 需要收集阅读者满意度 |

---

|------|---------|
