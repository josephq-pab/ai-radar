# 对公AI雷达站｜原型页面与信息架构说明

## 1. 整体信息架构
原型采用“1 个总览入口 + 2 个业务深案例页 + 1 个统一跟踪页”的最小架构：
- 首页总览
- 存款深案例页
- 贷款深案例页
- 重点事项跟踪页

## 2. 页面层级关系
### 一级页面
1. 首页总览
2. 存款深案例页
3. 贷款深案例页
4. 重点事项跟踪页

### 二级承接内容
- 周报入口
- 专题入口
- 跟踪事项详情预览

## 3. 页面跳转主链路
- 首页总览 → 存款深案例页
- 首页总览 → 贷款深案例页
- 首页总览 → 重点事项跟踪页
- 存款深案例页 → 重点事项跟踪页
- 贷款深案例页 → 重点事项跟踪页
- 重点事项跟踪页 → 首页/存款页/贷款页

## 4. 页面定位
### 首页总览
承担总览、导航、提醒、入口承接。

### 存款深案例页
承接存款关键观察、对标变化、建议动作、跟踪事项、专题入口。

### 贷款深案例页
承接贷款关键观察、监管雷达、行业风险、分析师观点、对标变化、建议动作、跟踪事项、专题入口。

### 重点事项跟踪页
承接事项收口、状态更新、进展记录、下一步动作与来源页回溯。

## 5. 关键原则
- 不重写首页与既有深案例结构
- 周报/专题首期以承接位存在，不急于扩成独立页面体系
- 对标分析和跟踪主表优先依赖真实或半真实数据
