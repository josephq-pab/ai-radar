# 对公AI雷达站｜重点事项跟踪页原型说明

## 1. 页面定位
重点事项跟踪页是原型中的“闭环管理页”，不是普通事项列表页。其作用是把首页、存款页、贷款页产生的重点事项统一收口，并将“观察→跟踪→上报/行动→关闭”的过程显性化。

## 2. 服务对象
- 管理层：看重点事项状态与推进情况
- 经营分析/业务团队：维护事项进展与下一步动作
- 科技/产品团队：理解字段、状态与来源承接规则

## 3. 页面结构
- tracking_summary
- tracking_filter_panel
- tracking_main_table
- tracking_detail_preview
- related_entry_area

## 4. 核心字段
- 事项ID
- 事项名称
- 来源维度（存款/贷款）
- 来源主题
- 当前状态
- 最新进展
- 下一步动作
- 最近更新时间
- 关联页面/专题入口

## 5. 状态流转
统一沿用以下口径：
- 待研判
- 跟踪中
- 已上报
- 已行动
- 已关闭

建议最小流转：
待研判 → 跟踪中 → 已上报 / 已行动 → 已关闭

## 6. 首期最小操作
- 事项列表查看
- 按维度/状态/来源主题筛选
- 查看事项详情预览
- 跳转回来源页面
- 查看关联专题/周报入口

## 7. 首期暂不纳入
- 审批流
- 自动派单
- 多人协同编辑
- 权限系统
- 复杂分析与预测能力
