🔟 执行总流程（8步）
1
检查
触发检查（Preflight） 第一步，必须先做
对照 STAGE2_SINGLE_CHAIN_TRIGGER_RULES_V1.md，检查是否命中触发条件。
检查方法：
① 查看 analyst_opinions_raw.json 的 fetchedAt 时间戳，与上轮 ROUND_COMPARISON 记录对比
② 查看 raw 总数是否有显著新增（≥5条增量）
③ 检查距上次运行是否满14天
④ 检查是否有异常信号（曾刚断档/usable异常/集中负向反馈）
触发条件（三选一即可）：
A. 数量触发：raw 新增 ≥5 条
B. 时间触发：距上次满 14 天
C. 异常/用户触发：强烈信号或用户明确要求
2
停止
未触发 → 立即停止 未命中则停在此步
若 Step 1 检查结果为"未触发"，立即停止，不执行任何后续步骤。
⛔ 未触发时不得执行 Step 3 及后续任何步骤。
本步是强制停止点，不能绕过。
留痕动作：在 REVIEW_LOG 或对应检查记录中标注"本轮 preflight 未触发，维持分支 D"
3
执行
已触发 → 执行运行链路 触发命中后执行
命中触发条件后，按顺序执行以下命令（全部在 05_工具脚本/ 目录下）：
命令执行顺序（必须按顺序）：
cd /path/to/OPENCLAW_DEPLOY/05_工具脚本
./run-analyst-fetch.sh
→ 刷新 analyst_opinions_raw.json（fetchedAt 会变化）
./run-analyst.sh
→ build → generate_review_queue → tracker merge → tracker export
→ 产出：analyst_opinions.json / analyst-review-queue.json / pilot-tracking-ledger.csv
fetch → build → queue → tracker merge → export
4
核对
核对 raw / usable / 曾刚贡献 运行后必须核对
运行链路完成后，必须核对以下三个核心指标，与上轮对比判断本轮质量。
核对方法：
① raw 新增条数 → analyst_opinions_raw.json records.length
② usable 条数 → analyst_opinions.json usable_count
③ 曾刚贡献 → analyst_opinions.json 中 dimension=对公整体 且 author 含曾刚的记录
本轮参照（RUN-01）：raw 新增 6 条 / usable 池 32 条 / 曾刚 1 条
关注：若 usable <3 条 → 需评估是否触发异常辅触发
5
输出
形成本轮月度输出 核心产物
基于本轮 queue（analyst-review-queue.json）和 usable 池，形成月度试用稿。
输出结构（参考 V7 基线模板）：
① 领导版月度分析（基于usable池，按LEADER_OUTPUT_TEMPLATE_V7撰写）
② 执行版行动指引（按EXECUTION_OUTPUT_TEMPLATE_V7撰写）
③ 主动作 + 观察项（按ACTION_LOOP_RULES_V1格式）
关键：月度试用稿产出后，由运营编辑者发送给试用对象（赵总/曾总/邱非）
6
判断
做本轮状态判断 关键决策节点
基于 Step 4 核对结果，做本轮最终状态判断（参照 P3-7 三级判断框架）。
三级判断：
结论A：继续保持单链路 → 曾刚持续贡献 + usable 充足 + 无异常信号
结论B：第二链路讨论 → 曾刚断档≥2月 或 单一来源集中度恶化
结论C：V7 基线讨论 → 连续2月正向试用反馈积累
本轮参照（RUN-01）：结论A（usable 32条 >> 3条阈值，曾刚1条，第二链路/V7均未触发）
7
记录
留痕并收口 必须完成才能算本轮结束
本轮所有产出和判断结论必须记录到文档，留痕才算本轮正式结束。
必须留痕的文档（按顺序）：
① 新建 docs/ROUND_RECAP_YYYY-MM-DD.md（每轮新建，用模板）
② 更新 docs/ROUND_COMPARISON.md（追加本轮行）
③ 更新 pilot-tracking-ledger.csv（确认tracker状态）
④ 更新 docs/STAGE2_SINGLE_CHAIN_RUN_LOG_XX.md（如是新运行编号）
⑤ 更新 docs/REVIEW_LOG.md（追加本轮R-xx记录）
禁止跳过留痕："运行了但没记录"等于本轮未完成
8
停止
留痕完成 → 本轮结束，等待下一触发 终点：强制停止点
Step 7 完成后，本轮正式结束。不得以任何理由继续执行其他步骤。
⛔ 本轮结束后：不再执行任何采集/生成动作。
等待下一触发条件命中（正常时间触发 / 异常辅触发 / 用户主动触发）。
本轮参照（RUN-01）：RUN-01 → 结论A → 留痕完成 → 正式结束，等待下一触发

🚫 当前明确禁止动作
✗不未触发先跑：preflight 未命中 → 停在 Step 2，不能进入 Step 3
✗不跳过 preflight：不能"想跑就跑"，必须先做 Step 1 触发检查
✗不跳步执行：不能跳过 Step 4 核对直接形成输出，不能跳过 Step 7 直接结束
✗不留痕不算结束：Step 7 未完成，本轮不算结束，不能开始下一轮规划
✗不找第二链路来源：第二链路结构性冻结，不得以任何理由推进来源发现
✗不改 V7 基线：V7 冻结中，非改稿时机，不得修改
✗不改脚本：run-analyst.sh / run-analyst-fetch.sh 不得修改
