🚫 当前不建议的阅读方式
✗ 一次性把十页全部看完 — 十页是分工协作的结构，不是培训材料。按角色路径只看最少的几页即可。
✗ 无差别把十页链接扔给对方 — 对方不知道从哪开始。应该告诉对方"你是哪类角色，按这个顺序看这几页"。
✗ 让领导从首页开始逐页看 — 领导要快，首页信息量太大。先看快报（1页）再按需深入。
✗ 让协作者先看执行流程 — 协作者不需要知道怎么跑，只需要知道状态依据和信谁。
✗ 让接手者自己摸索 — 应该有明确的角色路径，按路径走可以少走弯路。

🔔 巡检结束后何时进入升级说明
巡检结果默认先留痕，只有命中升级条件时才进入升级说明。升级是"请求判断"，不是"汇报进展"。
✅ 不升级（默认）
何时：无动作/仅记录/轻微信号
按值班输出模板留痕即可，不需要升级
🔶 评估请判断
何时：边界不清/判断困难/触发不明确
给出最小升级说明，请人判断是否继续
🟣 准备请确认
何时：明显新信号/运行前条件接近满足
给出准备说明，请确认是否进入下一步
⚠️ 升级后停点：停在等待方结论，不继续扩写材料。升级后由接收方给出结论，运营者不应继续自主扩写。
📩 升级后回复的最小回写路径
升级说明发出后，收到回复先分类，再决定停在哪。不需要重新看所有页面。
✅ 同意推进
收到回复后：先分类
同意推进 → 回写"已获同意" → 进入下一步
🔵 暂不推进
收到回复后：先分类
暂不推进 → 回写"维持等待" → 停止，不继续扩写
🔶 需补材料
收到回复后：先分类
需补材料 → 回写"补X" → 补最小必要依据 → 重新升级
❌ 不同意本次
收到回复后：先分类

⚠️ 非必要不再扩写：收到回复后不需要重新看所有页面。需补材料时只补1~2个最小必要依据，不需要重新整理整套证据。
🔄 生命周期路径 vs 阅读路径
阅读路径和生命周期路径不是一回事，不要混淆。
📖 阅读路径（理解系统）
首页 → 快报 → 运营页 → 决策页/证据页/真源页 → 回到首页
用于理解系统当前状态、各页面职责、如何查找信息
是"看书"的路径
🔄 生命周期路径（系统运转）
等待(W) → 评估(A) → 准备(P) → 升级(E) → 回复(R) → 回收停点(S)
描述值班从开始到结束的完整流转过程
是"系统怎么动"的路径
⚠️ 不要混淆：阅读路径是用来理解系统的，生命周期路径是系统实际运转的。值班结束时停在哪个状态由生命周期决定，不是由阅读路径决定。
📋 典型场景 → 路径 → 停点
判例不是新路径，是现有路径的示例化。场景→对应生命周期路径→正确停点。
A. 无信号 → W→停
生命周期路径：W（等待）→ 本次无动作
B. 轻微信号 → L→停
生命周期路径：W→L（轻响应）→ 本次仅记录
C. "这算不算触发" → A→W或E
生命周期路径：W→A（评估）→ 边界清晰回W / 边界不清→E（升级）
D. 明显新数据 → P→E→R→停
生命周期路径：W→A→P（准备）→E（升级请确认）→R（收到回复）→停P
F. "先观察" → E→R→W
生命周期路径：E（升级）→R（收到回复）→回W（维持等待）
