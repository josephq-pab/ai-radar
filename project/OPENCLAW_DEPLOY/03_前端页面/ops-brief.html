🔄 当前处在生命周期哪一段
系统大多数时候停在等待态。不是所有值班都会推动状态前进。
等待态(W) — 本次无动作。系统默认停在等待态，无新信号时不需要深查。
轻响应态(L) — 本次仅记录。口径/术语/回访类输入，不进入评估。
评估态(A) — 进入评估。边界不清/触发待判/异常待判，需要在运营页和决策页判断。
准备/升级/回复 — 接近下一步/需要人工拍板/收到回复。详细在运营页。

📋 高频误判一句话对照

"先观察" ≠ 同意推进 → 暂不推进=维持等待
"补材料" ≠ 否决 → 补完可继续，不需要整套证据
"同意准备" ≠ RUN-next已开启 → 进入准备，RUN-next仍需确认
"来源候选" ≠ 解冻第二链路 → 必须评估后才能用
轻微信号 ≠ 评估项 → 留痕即可，不进评估
无信号时不需要深查 → 停在等待态，不需要翻辅助页

⚖️ 高频边界一句话对照
两个很像，但不是一回事。对照快速区分。
"暂不推进" ≠ 否决 → 暂不=可再提 / 否决=关闭可能
"补材料" ≠ 否决 → 补材料=信息不够可继续 / 否决=明确不采纳
"同意准备" ≠ 已开始执行 → 同意准备=等RUN-next确认
"来源候选" ≠ 解冻第二链路 → 候选≠可用，评估后才能用
"无动作" ≠ 仅记录 → 无动作=完全平静 / 仅记录=有变化但轻微
"轻响应" ≠ 评估项 → 轻响应=不影响触发 / 评估=边界不清需判断
