🏷 页面性质说明
📄 展示页（Display）
1. 提供信息聚合和导航能力，本身不是最终判断依据
2. 作用是让运营者"快速找到该看的文档"
3. 典型：首页 / 决策助手 / 执行流程
4. 状态冲突时，以对应真源文档为准
📋 依据页（Source）
1. 展示配置快照或状态快照，是真实业务状态的只读呈现
2. 典型：配置页（展示 analyst_sources.json 快照）/ 运营页
3. 真源是背后的配置文件或文档，页面仅用于方便查看
4. 配置页的真源是 analyst_sources.json，不是 config-status.html 本身
🔖 索引辅助页（Index）
1. 不直接展示业务状态，而是提供元信息和导航指引
2. 典型：术语口径页 / 真源登记页
3. 作用是帮助运营者"不混淆该信哪个"
4. 不是业务判断依据，是辅助决策工具
⚠️ 重要原则：页面 ≠ 真源
1. 所有页面都是"只读展示层"，不是配置编辑器
2. 页面和文档同时存在时，判断真源以文档为准
3. 页面可以不看文档，但判断必须以文档为最终依据
4. 冲突时，查本表 → 找对应文档 → 以文档为准
⚡ 冲突时的优先级规则
